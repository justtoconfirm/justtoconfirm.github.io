Introduction to WordPress

WordPress (WP) is an open-source Content Management System (CMS) written in PHP and using a MySQL or MariaDB database.

How to setup WordPress

WordPress can be downloaded from the WordPress.org website. The latest version of WordPress is version 5.8. It is recommended to never edit the core code of WordPress.

The recommended requirements:

  • Apache or Nginx web server
  • PHP version 7.4 or greater
  • MySQL version 5.6 or greater / MariaDB version 10.1 or greater

Install the web server

To setup the Apache or Nginx web server locally, install one of these recommended tools:

  • MAMP PRO (Mac)
  • MAMP PRO (Windows)
  • WampServer
  • XAMPP

WordPress houses lots of different types of content that are divided into something called Post Types. By default WordPress stores all post types in the database under the wp_posts table.

Pages

Pages are used for static content that is not affected by date. For example, an About page, Contact page, or Home page, are examples of pages. We may also have a Blog page used to display posts. Pages are typically listed in your site's navigation.

A page is an example of a Post Type in WordPress. This will be stored in the wp_posts table within the database.

To create your first page, log in to your WordPress dashboard and navigate to Pages/Add New.

Example pages:

  • Home
  • About
  • Contact
  • Blog

Pages do not use categories or tags, unlike posts. Instead, pages can be organised into parent and child page groups to group them hierarchically. For instance, you may have a parent page called "Apparel" which can then be split into "Women's Apparal" and "Men's Apparal".

Posts

Posts are entries listed in reverse chronological order on your site. Think of them as articles, or blog posts, that can be displayed on your site.

A post is an example of a Post Type in WordPress. This will be stored in the wp_posts table within the database.

To create your first post, log in to your WordPress dashboard and navigate to Posts/Add New.

When you create a post, you can assign it to a category and add tags, so your visitors can browse content by topic rather than chronologically by date.

  • Categories - are like a table of contents for your WordPress site. For instance, if you sell merchandise, one of your categories maybe "Shirts".
  • Tags - are granular terms that are specific to a piece of content. For instance, if you want to publish an announcement about a new Shirt, you may use tags like "Organic Cotton" or "Sports" as descriptors.

You may want to create a Blog page to display all your Blog posts, for example. It is often useful to create a custom template for displaying Blog posts.

Adding Posts to your pages

Custom Post Types (CPT)

WordPress houses lots of different types of content that are divided into something called Post Types. A single item is called a post which is a standard post type called posts. All post types are stored in the database under the wp_posts table.

A post in WordPress is a post type that is typically used for blog posts. We may want to extend the amount of post types to allow us to break things down into smaller categories. For example, if we want to have a section on Books, it would be better suited to creating a Custom Post Type (CPT) for Books. Creating a Custom Post Type (CPT) can be done using the register_post_type() function.

Creating a Custom Post Type as a plugin is recommended to ensure it can always be available if you decide to switch themes. Here, we're creating a Custom Post Type called Cars in the functions.php file:

    

function my_first_post_type() {

    $args = array(

        'labels' => array(
            'name' => 'Cars',
            'singular_name' => 'Car',
        ),
        'hierarchical' => true,
        'public' => true,
        'menu_icon' => 'dashicons-images-alt2',
        'has_achive' => true,
        'supports' => array('title', 'editor', 'thumbnail'),

    );

    register_post_type('cars', $args);

}

// WP fires this hook using 'init' hook just before the website loads
add_action('init', 'my_first_post_type');

When we save the functions.php file and refresh the WordPress admin page, the Cars Custom Post Type will now appear in the sidebar.

Now if we want to create sub-categories of the Cars, such as BMW, Audi, Nissan and others, we can use a taxonomy, which is much like a category and can be assigned to a post type.

Open the functions.php file and add the following code to create a taxonomy:

    

function my_first_taxonomy() {
    
    $args = array(
        'labels' => array(
            'name' => 'Brands',
            'singular_name' => 'Brand',
        ),
        'public' => true,
        'hierarchical' => true,
    );

    register_taxonomy('brands', array('cars'), $args);
}

add_action('init', 'my_first_taxonomy');

Now if we save the file and refresh the WordPress admin, you will see the Brands category (taxonomy) be added to the Cars Custom Post Type in the sidebar.

Next, create a new Car named Toyota Corolla in the admin and then in the Brands panel on the right of the admin page, click the + Add New Category link and create a new category called Toyota.

Click the Publish button.

In the Settings/Permalinks page, select the Post name option and click Save Changes.

Themes

WordPress sites can be styled using themes. We have the option to install an existing theme or we can choose to create our own custom theme. A list of available themes available to install can be found on the WordPress.org website.

Any theme installed will be inside the wp-content/themes/ directory of your WordPress installation. WordPress 5.8 comes pre-installed with the Twenty Twenty-One theme by default. This theme can be changed.

A WordPress theme needs two files to exist - style.css and index.php. Some themes will also have a functions.php file. This is known as a theme functions file and acts like a plugin for the WordPress site that's automatically activated within the current theme.

Develop a custom theme

Begin by creating a new directory in the wp-content/themes directory. This folder will contain our custom theme, and as mentioned, requires the following files:

  • index.php
  • style.css
  • functions.php (optional)

The index.php file is the main template. If your theme provides its own template, index.php must be present. This will apply to all our pages and posts by default, so this is where our head content is placed.

The index.php file is going to be rendering all of our pages and posts when the theme is activated. To begin, we create the head section. This will be written in standard HTML. The wp_head() action hook is included as this is core functionality of WordPress and allows both WordPress and third-party plugins to insert code into the header without modifying the template files.

Within your theme, update the index.php file to add a head section.

    

<!DOCTYPE html>
<html lang="en-GB">
<head>
    
	<meta charset="utf-8">
	<meta name="viewport" content="width=device-width, initial-scale=1">

	<?php wp_head(); ?>
</head>

We can update this further by using more WordPress hooks.

    

<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
	<meta charset="<?php bloginfo('charset'); ?>">
	<meta name="viewport" content="width=device-width, initial-scale=1">

	<?php wp_head(); ?>
</head>

There's no <title> tag here as WordPress uses the wp_head() hook to dynamically insert the page title.

Next, we have the body of the page:

    

<body <?php body_class(); ?>>

</body>
</html>

The body_class() is a helper function provided by WordPress that will output a list of useful CSS classes to describe the page being displayed. Remember to add a closing </body> tag.

We can add further classes to this helper function as it also accepts parameters.

    

<body <?php body_class('wide-template dark-background'); ?>>

Before the closing </body> and </html> tags, add the wp_footer() action hook used by WordPress and plugins to include scripts in the footer needed to render the page.

    

<?php wp_footer(); ?>

</body>
</html>

Title tag

All themes should use the built-in functionality of WordPress to generate the title tag. Within the functions.php file, add the following code:

    

<?php
	add_theme_support('title-tag');
?>

This will display the page/post title followed by the site title. To update the site title, open the Settings/General and update the text within the Site Title input field.

Template Parts

We can use template parts to allow us to reuse code across multiple templates. For example, as our header and footer will be the same on every page, they are perfect candidates for using template parts.

We can move the footer code into a template part named footer.php:

    

<?php wp_footer(); ?>

The index.php file can then be updated to include this template part using the get_template_part() function. When getting a template part, we omit the .php file extension.

    

<?php get_template_part('footer'); ?>

</body>
</html>

Repeat this process for the <head> - including the page title, meta tags etc. - and any other part of the code you may want to reuse.

Our index.php file will now look like this:

    

<?php get_template_part('head'); ?>

<body <?php body_class(); ?>>

	<!-- Page content goes here -->

	<?php get_template_part('footer'); ?>
</body>
</html>

Our theme folder will now look like this:

    

wp-content /
    themes /
        mytheme /
            footer.php
            functions.php
            head.php
            index.php
            style.css

This is only an example, so it's still best to use the core WordPress functionality of get_header() and get_footer() instead of the template parts for the header and footer.

Adding a Navigation Menu

To add a custom navigation menu, we must first register a menu by adding code to the functions.php file or our theme. Once the following code has been added, the Menus option will appear in the Appearance tab.

    

function wpb_custom_new_menu() {
	register_nav_menu('custom-menu',__('Custom Menu'));
}

add_action('init','wpb_custom_new_menu');

Now when we go to the Appearance/Menus page in the WordPress admin, we will see the Custom Menu listed in the Menu Settings section. Click the checkbox next to this menu and click the Save Menu button.

If you want to add more than one new navigation menu location, then use this code:

    

function wpb_custom_new_menu() {
	register_nav_menu(
		array(
			'custom-menu' => __('Custom Menu'),
			'extra-menu' => __('Extra Menu')
		)
	);
}

add_action('init','wpb_custom_new_menu');

Once the menu location has been added, we can add some menu items in the WordPress admin. Next, we need to display the menu in our theme.

Open the index.php file (or your theme's template file) and add the following code where you want to display your menu.

The theme location is the name of the menu that was given in the previous code. The container class is the CSS class that will be added to your navigation menu allowing you to style the menu using CSS.

This will initially render an unordered list without any styling for each of the menu items added on your pages that use the template.

Plugins

WordPress plugins are PHP scripts that extend the functionality or add new features to WordPress. Any plugin installed will be placed within the wp-content/plugins/ directory of your WordPress installation.

It's possible to install plugins manually by extracting and copying them into the plugins directory. Doing so will make them available for activation within the WordPress admin dashboard.

GraphQL

GraphQL can be installed using the WPGraphQL plugin from within the WordPress admin.

Advanced Custom Fields (ACF)

Advanced Custom Fields (ACF) is a WordPress plugin that allows you to add extra content fields, known as Custom Fields, to the edit screens within the admin dashboard of WordPress.

This plugin does not come pre-installed with WordPress so needs to be installed to be used. The plugin will be installed in the wp-content/plugins/ directory of your WordPress installation.

Once installed, we will see a Custom Fields tab in the sidebar of the WordPress admin dashboard.

Introduction to WordPress

WordPress (WP) is an open-source Content Management System (CMS) written in PHP and using a MySQL or MariaDB database.

How to setup WordPress

WordPress can be downloaded from the WordPress.org website.

Recommended requirements:

  • PHP version 7.4 or greater
  • MySQL version 5.6 / MariaDB version 10.1 or greater
  • Apache or Nginx

http://localhost:8888/MAMP/

The WordPress setup page should now be displayed. The host in this example is named test.local and so our URL to view the WordPress setup is: https://test.local:8890/wp-admin/setup-config.php.

The setup process will generate the wp-config.php file to connect to our database. The database credentials may be provided by your Web Host, but for local development we will create test credentials. Click the Let's go! button to continue.

At this stage we will want to create a database to store our WordPress data. If your web server has phpMyAdmin installed, you can create the local database here:

  • http://localhost:8888/phpMyAdmin5/

Set the language and encoding to be: utf8mb4_general_ci

https://wordpress.org/support/article/creating-database-for-wordpress/

For simplicity we will keep the default credentials (these need to match the database credentials used in phpMyAdmin) and click the Submit button. We can look to change these credentials in the wp-config.php file once it's been generated.

If successful, click the 'Run the installation' button to begin the installation.

Next, we setup our login credentials to login to the WordPress admin. Click the Install WordPress button when ready.

Your WordPress install should now be complete. Login to the dashboard using your username and password.

Themes

The latest WordPress version (5.8, September 2021) comes installed with the Twenty Twenty-One theme by default. This theme can be changed if required.

  • WordPress Themes documentation

To view the current theme installed:

  • Login to the WordPress admin dashboard (if not already logged in).
  • Select Appearance and Themes.

Advanced Custom Fields (ACF)

Advanced Custom Fields (ACF) is a WordPress plugin that allows you to add extra content fields, also referred to as Custom Fields, to the WordPress edit screens in the dashboard.

  • Advanced Custom Fields (ACF) documentation

The Advanced Custom Fields plugin does not come pre-installed with WordPress so needs to be installed as a plugin to be applied.

Installation

  • Login to the WordPress admin dashboard (if not already logged in).
  • Select Plugins and Add New.
  • Search for Advanced Custom Fields within the Add Plugins page.
  • Click the Install Now button to install the plugin.
  • Click the Activate button.
  • The Custom Fields menu option will now appear in the side bar.

Once installed, the plugin code can be located from within the wp-content/plugins directory of your WordPress installation. This is where WordPress stores plugins. It is possible to install plugins manually by extracting and copying them into this directory. Doing so will make them available for activation in the WordPress dashboard.

Create a Custom Field

  • Login to the WordPress admin dashboard (if not already logged in).
  • Select Custom Fields and Add New.
  • Add a title.
  • Add a Field.
  • Select the Rule, so we could set this to a specific page or post. As all our pages use the Default Template, we will set it to be available for all pages that use the Default Template.
  • Click Publish and Update.

Now any page using the Default Template will display the custom field when you edit the page from within the dashboard.

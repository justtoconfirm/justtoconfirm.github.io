CSS Units

Declaring a font size

Recommended to use the rem (Root Element) unit to declare font sizes. They are relative to the root element (The HTML element of your HTML document). The default size is 16px, so 1rem = 16px.

Using rem units allows the size to adapt to the browser preferences and screen sizes.

    

body {
    font-size: 1rem;
}

Setting the root font size on the HTML element to 62.5% which will set the base font size (root) to 10px and this will then make the rem value easier to calculate.

2.1rem x 10 = 21px

    

html {
    font-size: 62.5%;
}

body {
    font-size: 2.1rem;      // equal to 21px
}

Try avoiding the use of em for font sizes as this will use the parent element as the base font size and not the root.

Declaring a width

Use a percentage and a max-width property. We can also use the ch property to manage the number of characters in a line.

Padding & Margin

Use the rem or em unit for padding and margin where possible. Use px if you wish the margin and padding to be consistent, or rem units if you want it to adjust based on screen size and/or preferences.

Padding on buttons should use em units to allow the margin and padding to scale if the font-size changes.

Give all your text elements (list, heading, paragraphs) a margin-top of 1rem and it will be consistent across the entire HTML document. You could also change this to 1em and the headings will have greater spacing because they have a large font-size.

    

p {
    margin-top: 1rem;
}

h1, h2, h3, h4, h5, h6 {
    margin-top: 1rem;
}

Spacing

It is often better to provide spacing between elements determines by the element's parent element.

    

<ul class="nav">
    <li class="nav-item">Item one</li>
    <li class="nav-item">Item two</li>
</ul>

Our CSS will look like this:

    

.nav {
    display: flex;
    flex-direction: column;
    gap: 1em;
}

Media Queries

Set media queries using the em unit. This helps to provide consistency across all browsers.

GraphQL

GraphQL is an open-source data query language for APIs (Application Program Interface) and was developed internally by Facebook in 2012 before being publicly released in 2015. It is a syntax that describes how to ask for data and is generally used to load data from a server to a client.

REST is built on top of an existing architecture, which in the most common scenarios is HTTP. GraphQL is built on its own set of conventions.

In a traditional REST-based API approach, the client makes a request and the server dictates the response. But, in GraphQL, the client determines precisely the data it wants from the server. For example, the client may only want the user's name and email and no further information. GraphQL is less verbose, we only get what we ask for, which differs from REST, which gives you everything (not very efficient and lots of unnecessary data).

With GraphQL, the user is able to make a single call to fetch the required information rather than to construct multiple REST requests to fetch the same.

Within Node, tools such as express-graphql, can be used as a popular server middleware for integrating GraphQL with Node.js.

A single endpoint

GraphQL has only one endpoint where you send all your queries. With a REST approach, you create multiple endpoints and use HTTP verbs to distinguish read actions, such as GET, or write actions, such as POST, PUT and DELETE. GraphQL does not use HTTP verbs to determine the request type.

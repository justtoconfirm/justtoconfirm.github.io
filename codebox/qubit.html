Qubit

Qubit is a CRO (Conversion Rate Optimisation) tool that uses Node Package Manager (NPM) and the command-line to make building tests easier for developers.

https://www.qubit.com/

Qubit was founded by a small team who previously worked at Google. Based in London's Convent Garden and with offices in both London and Los Angeles.

Conversion Rate Optimisation, or CRO, is the practice of raising your conversion rate by compelling visitors to your site to take specific actions. It is used to make small, incrememntal improvements, often through the use of CRO tools, to write CRO tests to test the effect of any changes on visitors. These tests are often referred to as A/B or MVT tests. Tools such as Qubit, Google Optimize, VWO, Adobe Target, and Monetate are just some of the tools used for CRO.

Developers

Qubit has a Command Line Interface (CLI) which lets developers work inside their own repositories (repos) and push changes ot experiences up to Qubit. NPM packages are also available allowing code reuse.

The Qubit CLI also provides support inside a CI/CD environment.

The code can be pulled down to your local machine and checked into your main codebase. Development can take place in your code editor of choice and run locally using the Qubit hot-reloading build system and Chrome extension.

Qubit uses CommonJS modules so they can be managed like other NPM packages. You can also develop unit tests against these modules using a framework of choice. Qubit use Jest.

Support for both React and Angular is also provided.

A/B testing

On a simple level, A/B testing is a method of experimenting with two versions of a website or mobile app. By observing and comparing the behavior of visitors randomly allocated to each version, we can determine which performed better against defined goals such as conversion rate or click-through rate.

Qubit uses A/B testing as a framework to compare two versions of a website or mobile app. Visitors are randomly bucketed into one of the two versions - control or variation - and behaviors are observed over time.

  • Control - visitors see your website or mobile app without any changes.
  • Variant - visitors will see the changes you're testing.

A/B testing allows you to determine what impact your personalisation is having in your site. Usually there's a control (A) and a variant (B), with randomised allocation of visitors for each group based on traffic allocation.

Getting Started with React

What is React?

React is a UI (User Interface) component library developed by engineers at Facebook and intially released on May 29, 2013. React uses JavaScript to create components without the need to use a template language. This approach is creating composable UIs and is fundamental to the philosophy of React.

React allows components to be created that are similar to reusable HTML elements, to quickly build user interfaces. Data within React is stored and handled using state and props.

React components are self-contained. For example you may have a component for a date-picker, address or form elements. Each component has logic and styling. Certain components can also talk to a server on their own. For example, an auto-complete component may fetch the autocompletion list from the server.

To summarise:

  • React is an open-source JavaScript library.
  • Used to build front-end user interfaces (UI).
  • React is the view layer of an MVC (Model View Controller) application.

Here are some of the benefits of using React:

  • React is fast. Apps made with React can handle complex updates and still feel quick and responsive.
  • React is modular. Instead of writing large, dense files of code, you can write much smaller, reusable files.
  • React is scalable. Large programs that display a lot of changing data are where React performs best.
  • React is flexible. You can use React for interesting projects that are not web apps.

React Developer Tools

A browser extension is available called React Developer Tools that will make working with React much easier. Once installed, you will see a tab for React when you open your browser's developer tools (DevTools). The Elements tab will show the actual DOM (Document Object Model) output.

Install React

Node Package Manager (NPM) can be used to install React. Begin by creating a new directory where the React application will be built and setup a package.json file. open your terminal and run the following commands:

bash -- 85x24

    $ mkdir react-app
    $ cd react-app

    $ npm init -y

We now have a directory named react-app and a generated package.json file.

We now want to install the following packages as dependencies:

  • react - the top-level API (Application Program Interface) of the React library.
  • react-dom - DOM (Document Object Model) specific methods used by React.

Both will now be listed in your package.json file.

bash -- 85x24

    $ npm install react react-dom --save

React and ReactDOM were split into two packages from version 0.14. React is the JavaScript library for building User Interfaces and ReactDOM is the JavaScript library that allows React to interact with the DOM. ReactDOM contains many methods all of which deal with the DOM and is where you will find the render() method.

We now need to install Babel and Webpack to run our React application.

Install Babel

Begin by installing the following packages using NPM (Node Package Manager):

  • @babel/core - this is the main dependency, including Babel.
  • @babel/preset-env - the default Babel preset used to transform ES6 into ES5 syntax and to optionally configure browser polyfills.
  • babel-loader - a Webpack loader that hooks Babel into Webpack.

Open your terminal within your project and install each package as a devDependency.

bash -- 85x24

    $ npm install @babel/core @babel/preset-env babel-loader --save-dev

To use Babel with React, the @babel/preset-react package will also need to be installed as a devDependency.

bash -- 85x24

    $ npm install @babel/preset-react --save-dev

The Babel Config File

To allow Babel to work a config file is required. The Babel config file is named .babelrc and should be placed within the root of your project folder. Once created, the config file maybe hidden in the folder. Use CTRL + SHIFT + . (dot) to reveal any hidden files.

Use the terminal to create a Babel config file:

bash -- 85x24

    $ touch .babelrc

Open the config file in your code editor and add the following code:

    

{
    "presets": [ "@babel/preset-env" ]
}

As we are building a React application, the @babel/preset-react package should also be added to the config file. This will allow your React components to be written in the ES6 JavaScript syntax using JSX.

    

{
    "presets": [ 
        "@babel/preset-env",
        "@babel/preset-react" 
    ]
}

Install & Setup Webpack

To allow our React application to be built, we need to setup Webpack. Node Package Manager (NPM) can be used to install Webpack. Both the webpack and webpack-cli packages are required to be installed as devDependencies. Open your terminal and run the following commands to install Webpack

bash -- 85x24

    $ npm install webpack webpack-cli --save-dev

Once installed, open the package.json file and add build scripts to allow Webpack to be run in development or production modes.

    

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "webpack --mode production",
    "build:dev": "webpack --mode development"
},

We want to use Babel within our React application, so we need to create a Webpack config file named webpack.config.js in the root of our project folder. Open your terminal and create an empty config file.

bash -- 85x24

    $ touch webpack.config.js

Open the webpack.config.js file and add the following code using your code editor:

    

module.exports = {
    module: {
        rules: [
            {
                test: /\.js$/,
                exclude: /node_modules/,
                use: {
                    loader: 'babel-loader'
                }
            }
        ]
    }
}

Your project folder should have the following file structure (a .gitignore file has also been included):

    

.babelrc
.gitignore
node_modules/
package.json
webpack.config.js

Hello World

Now that Webpack and Babel are setup, we can begin to use React and build a simple application to render 'Hello World' onto a page. This is the smallest React example.

Begin by creating a new directory in your project folder named src and inside add an empty JavaScript file named index.js. Within the index.js file, add the following code:

    

import React, { Component } from 'react';
import ReactDOM from 'react-dom';

ReactDOM.render(
    <h1>Hello World</h1>,
    document.getElementById('root')
);

Run the production build script in the terminal. This will generate a dist folder in your project directory where a minified main.js file will be generated. This will also generate a package-lock.json file.

bash -- 85x24

    $ npm run build

Should you want to run the build to generate an unminified main.js file, run the build script in development mode. The filename in either build mode will still be main.js.

bash -- 85x24

    $ npm run build:dev

Within the dist folder, create an empty HTML page named index.html and add the following HTML code:

    

<!DOCTYPE html>
<html lang="en-GB">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <title>React App</title>
</head>
<body>

<div id="root"></div>

<script src="main.js"></script>

</body>
</html>

Refresh the HTML page and Hello World will render. The file structure in your project should look like the example below:

    

.babelrc
dist/
    index.html
    main.js
.gitignore
node_modules/
package-lock.json
package.json
src/
    index.js
webpack.config.js

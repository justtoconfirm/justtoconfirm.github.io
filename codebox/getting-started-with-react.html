What is React?

React is a UI (User Interface) component library developed by engineers at Facebook and intially released on May 29, 2013. React uses JavaScript to create components without the need to use a template language. This approach is creating composable UIs and is fundamental to the philosophy of React.

To summarise:

  • React is an open-source JavaScript library for building user interfaces (UI).
  • It allows components to be built, which are pieces of the UI.
  • React is the view layer of an MVC (Model View Controller) application.
  • You can use the .js file extensions for building components, .jsx can also be used and this will provide good code completion.

React allows components to be created that are similar to reusable HTML elements, to quickly build user interfaces. Data within React is stored and handled using state and props.

React components are self-contained. For example you may have a component for a date-picker, address or form elements. Each component has logic and styling. Certain components can also talk to a server on their own. For example, an auto-complete component may fetch the autocompletion list from the server.

Here are some of the benefits of using React:

  • React is fast. Apps made with React can handle complex updates and still feel quick and responsive.
  • React is modular. Instead of writing large, dense files of code, you can write much smaller, reusable files.
  • React is scalable. Large programs that display a lot of changing data are where React performs best.
  • React is flexible. You can use React for interesting projects that are not web apps.

Every React application has at least one component - often called App - that is referred to as the root component. The represents the application and contains other child components.

JSX: JavaScript & XML

In many React components or code examples, you'll see what looks like HTML, but it's not JSX, which stands for JavaScript XML.

JSX is a syntax extension for JavaScript and was written to be used with React. JSX is not valid JavaScript so browsers cannot read it. If a JavaScript file contains JSX then that file needs to be compiled meaning that before the code reaches a web browser, a JSX compiler will translate the JSX into regular JavaScript. This is why we use Babel in our React setup.

React uses props (properties) inside JSX tags to define both HTML attributes and React components. These props are similar to the attribute syntax used in HTML.

React Developer Tools

A browser extension is available called React Developer Tools that will make working with React much easier. Once installed, you will see a tab for React when you open your browser's developer tools (DevTools). The Elements tab will show the actual DOM (Document Object Model) output.

Using Variables

An an example, we can declare a variable and use the variable name inside JSX by wrapping it in curly braces. Here, we are re-writing the example from earlier to display Hello Dan onto the page.

    

import React from 'react';
import { render } from 'react-dom';

const name = 'Dan';
const element = <h1>Hello {name}</h1>;

render(
    element,
    document.getElementById('root')
);

You can also place JavaScript code inside JSX. Any code between the tags of a JSX element will be read as JSX, not as regular JavaScript, but using curly brackets will execute the code as an expression. Here the value of 2 + 3 (5) will be rendered on the page.

    

import React from 'react';
import { render } from 'react-dom';

render(
    <h1>{2 + 3}</h1>,
    document.getElementById('root')
);

Adding Classes

In HTML it's common to add class as an attribute to an element. In JSX, class cannot be used as it's a keyword in JavaScript from when JSX is compiled into JavaScript. In JSX, className is used and this is automatically rendered into a class attribute in the JavaScript code.

    

import React from 'react'
import { render } from 'react-dom'

render(
    <h1 className="title">Hello World</h1>,
    document.getElementById('root')
)

Writing a Simple Component

The simplest way to define a component is to write a JavaScript function. A function can be a valid React component because it accepts a prop (property). Components created using a function are called function components because they are literally JavaScript functions.

    

function Welcome(props) {
    return <h1>Hello {props.name}</h1>
}

Remember, in React, component names should always start with an uppercase letter.

This function can also be written as an ES6 Arrow Function.

    

const Welcome = props => {
    return <h1>Hello {props.name}</h1>
}

We can also use an ES6 class to define a component. Notice that we use this.props.name to reference the value passed into the props within a class.

    

class Welcome extends React.Component {
    render() {
        return <h1>Hello {this.props.name}</h1>;
    }
}

Data can be passed into the Welcome component using props. When rendered, the component will print Hello Dan onto the page.

    

<Welcome name="Dan" />

Functional & Class Components

Let's compare a simple functional component with a class component. Here is a functional component example:

    

const SimpleComponent = () => {
    return <div>Example</div>
}

Here's an example class component:

    

class ClassComponent extends Component {
    render() {
        return <div>Example</div>
    }
}

If the return is contained to a single line, it does not need parenthesis.

Composing Components

Components can refer to other components in their output. This lets us use the same component abstraction for any level of detail. For example, we can create an App component in index.js that renders the Welcome component twice.

Typically most React apps have a single App component.

    

import React from 'react';
import { render } from 'react-dom';

const Welcome = props => {
    return <h1>Hello {props.name}</h1>;
}

const App = () => {
    return (
        <div>
            <Welcome name="Dan" />
            <Welcome name="Pearl" />
        </div>
    );
}

render(
    <App />,
    document.getElementById('root')
);

The empty <div> and </div> elements are needed when multiple JSX elements are being returned. Without this, an error will occur in the terminal. Having empty HTML elements can increase the number of elements rendered in the DOM, so this can be replaced by React Fragments. Fragments let you group a list of children without adding extra nodes to the DOM.

    

const App = () => {
    return (
        <React.Fragment>
            <Welcome name="Dan" />
            <Welcome name="Pearl" />
        </React.Fragment>
    );
}

Or we can use Fragments in this way:

    

const App = () => {
    return (
        <>
            <Welcome name="Dan" />
            <Welcome name="Pearl" />
        </>
    );
}

Adding Props

Now that the Button component has been created, we can begin to add props (properties). To begin, a prop named type is added. This will allow the type attribute for the button to be set.

To provide a prop with a default value this must be defined within the component. This default value will be applied each time the component is rendered or a new value is passed into the prop that will overwrite this default value.

Multiple Props

As the number of props added increases, the less readable the code becomes. Using a const variable is one way to improve the readability of our Button component.

    

import React from 'react'

const Button = props => {

    const {
        type = 'button',
        children
    } = props

    return (
        <button type={type}>{children}</button>
    )
}

A spread operator can also be used to avoid the need to list out all props within the return statement.

    

import React from 'react'

const Button = props => {

    const {
        type = 'button',
        children
    } = props

    return (
        <button {...{type}}>{children}</button>
    )
}

export default Button

Change the Rendering of a Component

A prop can be used to change how a component is rendered. For our Button component, a url prop can be added to take a value and render the Button either as an HTML link or button.

The following line of code is added into the Button component function:

    

const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'))

We then add two render methods and a ternary operator to determine which method to execute based on the value passed into the component url prop.

    

const renderAnchor = () =>
    <a href={url}>{children}</a>

const renderButton = () => 
    <button {...{type}}>{children}</button>

return (
    isAnchor ? renderAnchor() : renderButton()
)

Once the url prop has been added, our Button component code should look like this:

    

import React from 'react'

const Button = props => {

    const {
        type = 'button',
        url = '',
        children
    } = props

    const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'))

    const renderAnchor = () =>
        <a href={url}>{children}</a>

    const renderButton = () => 
        <button {...{type}}>{children}</button>

    return (
        isAnchor ? renderAnchor() : renderButton()
    )
}

export default Button

Now if we render the Button component, a HTML button will be rendered. Should we pass a URL into the Button as a prop value, then the Button will be rendered as a link, as this example below demonstrates.

Setting Classes

To apply a class attribute to a React component the className attribute is used. We can update the Button component to apply the class of btn as a default.

    

import React from 'react'

const Button = props => {

    const {
        type = 'button',
        url = '',
        children
    } = props

    const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'))

    const renderAnchor = () =>
        <a href={url} className="btn">{children}</a>

    const renderButton = () => 
        <button {...{type}} className="btn">{children}</button>

    return (
        isAnchor ? renderAnchor() : renderButton()
    )
}

export default Button

This is fine, but we may also want to toggle classes based on prop values being passed down into our component. To do this, we can install the Classnames package using NPM (Node Package Manager). This package needs to be installed as a dependency within your project.

Begin by opening your terminal and entering the following command:

bash -- 85x24

    $ npm install classnames --save

Once installed, the package will be added as dependency to your package.json file. We now need to update our Button component with a number of props and also import the package itself.

    

import React from 'react'
import classNames from 'classnames'

const Button = props => {

    const {
        type = 'button',
        url = '',
        className = '',
        theme = '',
        isFullWidth = false,
        children
    } = props

    const buttonClass = classNames('btn', className, {
        'btn--primary' : theme === 'primary',
        'btn--secondary' : theme === 'secondary',
        'btn--full-width' : isFullWidth
    })

    const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'))

    const renderAnchor = () =>
        <a href={url} className={buttonClass}>{children}</a>

    const renderButton = () => 
        <button {...{type}} className={buttonClass}>{children}</button>

    return (
        isAnchor ? renderAnchor() : renderButton()
    )
}

export default Button

Now each time the Button component renders, the class of btn will still be the default.

When we render a Button, we can now use the theme prop to specify what additional modifier class is to be applied.

Adding onClick Event Handlers

Our Button component is no use if it doesn't do anything when clicked. When the button is clicked, we want it to call a function to perform a task and do something. Handling events with React elements is very similar to handling events on DOM elements.

  • React events are named using camelCase, rather than lowercase.
  • With JSX you pass a function as the event handler, rather than a string.

As with className, we use onClick in React in-place of onclick as this is a keyword used in JavaScript.

Inside our Button component we need to add a new onClick prop. This will pass in the name of the function we want to call when the button is clicked.

    

import React from 'react'
import PropTypes from 'prop-types'
import classNames from 'classnames'

const Button = props => {

    const {
        type = 'button',
        url = '',
        className = '',
        theme = '',
        isFullWidth = false,
        children, 
        onClick
    } = props

Within the render method, we then need to add the onClick prop.

    

const renderButton = () => 
    <button type={type} onClick={onClick} className={buttonClass}>{children}</button>

Or we can use the spread operator as before:

    

const renderButton = () => 
    <button {...{type, onClick}} className={buttonClass}>{children}</button>

In the App component, we can render the Button component and create a function that's called when the button is clicked to display an alert message.

    

import React from 'react'
import Button from './Button/Button'

const App = () => {

    function sayHello() {
        alert('Hello')
    }

    return (
        <Button onClick={sayHello}>Hello Alert</Button>
    );
}

export default App

We can also turn the sayHello function into an ES6 Arrow function.

    

import React from 'react'
import Button from './Button/Button'

const App = () => {

    const sayHello = () => {
        alert('Hello')
    }

    return (
        <Button onClick={sayHello}>Hello Alert</Button>
    );
}

export default App

Now when the button is clicked, an alert is shown with Hello displayed.

An inline function can also be used which is defined inside of the onClick handler when the React component renders. This would be used to pass in additional arguments to a function, such as a value from a loop, or the target button's value.

    

import React from 'react'
import Button from './Button/Button'

const App = () => {

    return (
        <Button onClick={() => alert('Hello'))}>Hello Alert</Button>
    );
}

export default App

Typechecking with PropTypes

Within your app, typechecking is used to catch bugs and errors. For some applications, you can use JavaScript extensions like Flow or TypeScript to typecheck your whole application. React itself also has some built-in typechecking abilities.

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. TypeScript includes typechecking by default. You may see React components written using TypeScript.

Typechecking ensures that the values within props have the correct data types.

We do need to install the PropTypes package as a dependency as import it into the component.

bash -- 85x24

    $ npm install prop-types --save

To run typechecking on the props for a component, you can assing the propTypes property. Here, we update the Button component with the propTypes property. In this example, we're using PropTypes.string to check that the value for the type prop is only a string.

    

import React from 'react'
import PropTypes from 'prop-types'
import classNames from 'classnames'

const Button = props => {

    const {
        type = 'button',
        url = '',
        className = '',
        theme = '',
        isFullWidth = false,
        children, 
        onClick
    } = props

    const buttonClass = classNames('btn', className, {
        'btn--primary' : theme === 'primary',
        'btn--secondary' : theme === 'secondary',
        'btn--full-width' : isFullWidth
    })

    const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'))

    const renderAnchor = () =>
        <a href={url} className={buttonClass}>{children}</a>

    const renderButton = () => 
        <button {...{type, onClick}} className={buttonClass}>{children}</button>

    return (
        isAnchor ? renderAnchor() : renderButton()
    )
}

Button.propTypes = {
    type: PropTypes.string
}

export default Button

Now when we render the Button in the index.js file, should the type prop have a value that's not a string, such as a number, an error will be displayed in the browser console. For performancee, propTypes is only checked in development mode.

If we want to pass data into our List component and the listItems prop, we need to first import our List component:

    

import List from './List'

Our list component can be extended to allow an unordered or ordered list to be rendered based on the prop value:

    

import React from 'react'

const List = props => (

    const {
        type = 'ul',
        list
    } = props

    const listItems = list.map((item, index) =>
        <li key={index}>{ item }</li>
    )

    const listType = type && (type.includes('ol'))

    const renderUnorderedList = () =>
        <ul>{listItems}</ul>

    const renderOrderedList = () =>
        <ol>{listItems}</ol>

    return (
        listType ? renderOrderedList() : renderUnorderedList()
    )
)

export default List

Our example About component may look something like the example below.

    

import React from 'react'
import List from './List'

// These are the values we are passing into the List component
const mylist = ['a', 'b', 'c']

const About = () => (
    <List listItems={mylist} />
)

export default About

Here we are passing the values from the myList array into the listItems prop and this in-turn is being passed into our List component to render an unordered list onto the page.

This will render the following on the page:

  • a
  • b
  • c

We can also pass in values to render the list as an ordered or unordered list (default). Even if we don't pass in a type prop value, the list will default to be rendered as an unordered list.

    

import React from 'react'
import List from './List'

// These are the values we are passing into each List component
const mylist = ['a', 'b', 'c']
const developers = ['Darren', 'Dave', 'Scott', 'Dan']

const About = () => (
    <>
        <List type='ol' listItems={mylist} />
        <List type='ul' listItems={developers} />
    </>
)

export default About

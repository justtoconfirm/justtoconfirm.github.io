React

React is an open-source, front-end JavaScript library for building user interfaces and components.

Requirements

You will need to have the following installed on your machine to install and run React:

  • Node 14.0.0 or later
  • npm 5.6 or later

You can use nvm (Node Version Manager) or nvm-windows to switch Node versions between projects.

The following packages will be installed as dependencies:

  • react - the top-level API of the React library.
  • react-dom - DOM specific methods used by React.

Frameworks that use React

The following frameworks are built using React:

  • Next.js - lightweight framework for static and server-rendered applications built with React. It includes styling and routing solutions out of the box. Node.js is used as the server environment.
  • Gatsby - framework used to create static websites with React using components. Outputs pre-rendered HTML and CSS to create fast load times.

Install React

We can use Create React App to install React and to setup the development environment. This doesn't handle backend logic or databases, it just creates the frontend build pipeline, so we can use it with any backend we want.

This uses Babel and Webpack.

bash -- 85x24

	npx create-react-app my-app

	cd my-app
	npm start

npx is a package runner tool that comes with npm version 5.2 and later. This will install the current version of React (version 17 as of September 2021).

The start command will run the local development server on port 3000: http://localhost:3000/

Simplify the Application

Update the src/App.js file to simplify the code. For the moment we just need a simple JavaScript function. This will be our App component.

	

const App = () => {
	return (

		<h1>Hello World</h1>

	);
}

export default App;

This will render Hello World on the page.

If we want to add a further element to render, we will need to use React Fragments to wrap around the elements in the return statement. Without this a SyntaxError will occur as adjacent JSX elements must be wrapped in an enclosing tag. Fragments allow you to group elements without adding extra nodes such as <div> to the DOM (Document Object Model).

	

const App = () => {
	return (

		<>
			<h1>Hello World</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

We can also simplify the src/index.js file where our App component is rendered.

	

import React from 'react';
import { render } from 'react-dom';
import App from './App';

render(
	<React.StrictMode>
		<App />
	</React.StrictMode>,
	
	document.getElementById('root')
);

StrictMode is a tool for highlighting potential problems in an application. Like a React Fragment, it does not render any visible UI and checks are run in development mode only, not production.

The React application will render in the root element within the public/index.html file. The root element has the id of "root", but this can be changed.

Variables

In this example, we declare a variable called name and use it inside JSX by wrapping it in curly braces.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

You can put any valid JavaScript expression inside the curly braces in JSX. For example, putting 2 + 2 as an expression will output the value of 4.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Classes

In JSX we cannot use the class attribute as it's a JavaScript keyword. To add a class to an element in a React component we use the className attribute. This will automatically render as a class attribute.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1 className="title">Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Button

Add a button to the index/App.js component and a function to show an alert when the button is clicked. We use the onClick prop to call the function. As with className, we use onClick in React in-place of onclick as this is a JavaScript keyword.

	

const App = () => {

	const name = 'Dan';

	const showAlert = () => {
		alert('Hello World');
	}

	return (

		<>
			<h1 className="title">Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}

			<button type="button" onClick={showAlert}>Alert</button>
		</>

	);
}

export default App;

Link

We can add a link to a React application using a standard HTML anchor tag.

	

const App = () => {

	}

	return (

		<>
			<h1 className="title">Hello {name}</h1>

			{2 + 2}

			<a href="https://reactjs.org/" target="_blank" rel="noopener noreferrer">React</a>

			<button type="button" onClick={showAlert}>Alert</button>
		</>

}

export default App;

Adding CSS

Within the application folder structure we have an App.css stylesheet in the root for the src directory. We can import this CSS stylesheet into our App component to apply styling to our application.

	

import './App.css';

const App = () => {

	const name = 'Dan';

	const showAlert = () => {
		alert('Hello World');
	}

	return (

		<>
			<h1 className="title">Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}

			<a href="https://reactjs.org/" target="_blank" rel="noopener noreferrer">React</a>

			<button type="button" onClick={showAlert}>Alert</button>
		</>

	);
}

export default App;

This will add a inline <style> tag containing the CSS from within the App.css stylesheet into the <head> section of the HTML document that renders the app. This can be seen from within the developer tools inspector.

Image

We have an SVG image named logo.svg in the src directory of our app. We can import this into the App component to render the image inside an <img> tag.

Within the src directory begin by creating a sub-directory named components. This directory will contain reusable components we can use throughout our application.

Inside this directory, create a file named Button.js. This will be our reusable button component. It is recommended to always name your components with an uppercase letter at the start.

Open the Button.js component file and add the following code. This will be a function that will render a button.

	

import React from 'react';

const Button = () => {
	return (

		<button>Button</button>

	);
}

export default Button;

Update the App.js file to import the Button component - components/Button.js - into the app and replace the hard-coded button we were using previously. We will also need to remove the showAlert() function.

We have simplified the App component once again by removing all code except the Button component.

	

import Button from './components/Button';

const App = () => {

	return (

		<>
			<Button />
		</>

	);
}

export default App;

We may want to change the button type. We can use props to do this. Update the Button component to include a type prop. This will have the default value of button if no value is passed into the component.

	

import React from 'react';

const Button = ({ type = 'button' }) => {
	return (

		<button type={type}>Button</button>

	);
}

export default Button;

If we don't want the Button to have the type of button, but a type of submit, we can pass this into the button type prop from the App component.

	

import Button from './components/Button';

const App = () => {

	return (

		<>
			<Button type="submit" />
		</>

	);
}

export default App;

The Children Prop

When we render a component, the children prop will automatically be passed whatever content comes between the opening and closing tags. We can use the children prop to pass a button label into the Button component.

	

import React from 'react';

const Button = ({ type = 'button', children }) => {
	return (

		<button type={type}>{ children }</button>

	);
}

export default Button;

We can then update the App component to pass in the label of the button using the children prop. Our button will be called Register in this example.

	

import Button from './components/Button';

const App = () => {

	return (

		<>
			<Button>Register</Button>
		</>

	);
}

export default App;

Adding the onClick Prop

Once again we can add the onClick event handler as a prop in our Button component.

	

import React from 'react';

const Button = ({ type = 'button', onClick, children }) => {
	return (

		<button type={type} onClick={onClick}>{ children }</button>

	);
}

export default Button;

As before, the App component can be updated to use the onClick event handler prop to call a function when the button is clicked.

	

import Button from './components/Button';

const App = () => {

	const showAlert = () => {
		alert('Hello World');
	}

	return (

		<>
			<Button onClick={showAlert}>Register</Button>
		</>

	);
}

export default App;

Within our Button component we can tidy up the structure by placing our props inside an object.

	

import React from 'react';

const Button = props => {

	const {
		type = 'button',
		onClick,
		children
	} = props;

	return (

		<button type={type} onClick={onClick}>{ children }</button>

	);
}

export default Button;

Let's now add a className attribute so we can set a default class to each button. This will add the class of btn to each button that is rendered.

	

import React from 'react';

const Button = props => {

	const {
		type = 'button',
		onClick,
		children
	} = props;

	return (

		<button type={type} className="btn" onClick={onClick}>{ children }</button>

	);
}

export default Button;

Adding the Spread Operator

The props within the component can make use of the Spread Operator.

	

import React from 'react';

const Button = props => {

	const {
		type = 'button',
		onClick,
		children
	} = props;

	return (

		<button {...{ type, onClick }} className="btn">{ children }</button>

	);
}

export default Button;

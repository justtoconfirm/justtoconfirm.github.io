React

React is an open-source, front-end JavaScript library for building user interfaces and components.

Requirements

You will need to have the following installed on your machine to install and run React:

  • Node 14.0.0 or later
  • npm 5.6 or later

You can use nvm (Node Version Manager) or nvm-windows to switch Node versions between projects.

The following packages will be installed as dependencies:

  • react - the top-level API of the React library.
  • react-dom - DOM specific methods used by React.

Frameworks that use React

The following frameworks are built using React:

  • Next.js - lightweight framework for static and server-rendered applications built with React. It includes styling and routing solutions out of the box. Node.js is used as the server environment.
  • Gatsby - framework used to create static websites with React using components. Outputs pre-rendered HTML and CSS to create fast load times.

Install React

We can use Create React App to install React and to setup the development environment. This doesn't handle backend logic or databases, it just creates the frontend build pipeline, so we can use it with any backend we want.

This uses Babel and Webpack.

bash -- 85x24

	npx create-react-app my-app

	cd my-app
	npm start

npx is a package runner tool that comes with npm version 5.2 and later. This will install the current version of React (version 17 as of September 2021).

The start command will run the local development server on port 3000: http://localhost:3000/

We can update the src/App.js file and the development server will reload the page.

Simplify the Application

Update the src/App.js file to simplify the code. For the moment we just need a simple JavaScript function. This will be our App component.

	

const App = () => {
	return (

		<h1>Hello World</h1>

	);
}

export default App;

This will render Hello World on the page.

If we want to add a further element to render, we will need to use React Fragments to wrap around the elements in the return statement. Without this a SyntaxError will occur as adjacent JSX elements must be wrapped in an enclosing tag. Fragments allow you to group elements without adding extra nodes such as <div> to the DOM (Document Object Model).

	

const App = () => {
	return (

		<>
			<h1>Hello World</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

We can also simplify the src/index.js file where our App component is rendered.

	

import React from 'react';
import { render } from 'react-dom';
import App from './App';

render(
	<React.StrictMode>
		<App />
	</React.StrictMode>,
	
	document.getElementById('root')
);

StrictMode is a tool for highlighting potential problems in an application. Like a React Fragment, it does not render any visible UI and checks are run in development mode only, not production.

The React application will render in the root element within the public/index.html file. The root element has the id of "root", but this can be changed.

Variables

In this example, we declare a variable called name and use it inside JSX by wrapping it in curly braces.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

You can put any valid JavaScript expression inside the curly braces in JSX. For example, putting 2 + 2 as an expression will output the value of 4.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Classes

In JSX we cannot use the class attribute as it's a JavaScript keyword. To add a class to an element in a React component we use the className attribute. This will automatically render as a class attribute.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1 className="title">Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Build the Application

When you're ready to deploy your React application to production, run the build command to create an optimised build of your app in the build folder.

bash -- 85x24

	npm run build

React

React is an open-source, front-end JavaScript library for building user interfaces and components.

Requirements

You will need to have the following installed on your machine to install and run React:

  • Node 14.0.0 or later
  • npm 5.6 or later

You can use nvm (Node Version Manager) or nvm-windows to switch Node versions between projects.

The following packages will be installed as dependencies:

  • react - the top-level API of the React library.
  • react-dom - DOM specific methods used by React.

Frameworks that use React

The following frameworks are built using React:

  • Next.js - lightweight framework for static and server-rendered applications built with React. It includes styling and routing solutions out of the box. Node.js is used as the server environment.
  • Gatsby - framework used to create static websites with React using components. Outputs pre-rendered HTML and CSS to create fast load times. View the Gatsby guide →

Install React

We can use Create React App to install React and to setup the development environment. This doesn't handle backend logic or databases, it just creates the frontend build pipeline, so we can use it with any backend we want.

This uses Babel and Webpack.

bash -- 85x24

	cd my-app
	npm start

npx is a package runner tool that comes with npm version 5.2 and later. This will install the current version of React (version 17 as of September 2021).

The start command will run the local development server on port 3000: http://localhost:3000/

Update the src/App.js file to simplify the code. For the moment we just need a simple JavaScript function. This will be our App component.

This will render Hello World on the page.

If we want to add a further element to render, we will need to use React Fragments to wrap around the elements in the return statement. Without this a SyntaxError will occur as adjacent JSX elements must be wrapped in an enclosing tag. Fragments allow you to group elements without adding extra nodes such as <div> to the DOM (Document Object Model).

	

const App = () => {
	return (

		<>
			<h1>Hello World</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

We can also simplify the src/index.js file where our App component is rendered.

	

import React from 'react';
import { render } from 'react-dom';
import App from './App';

render(
	<React.StrictMode>
		<App />
	</React.StrictMode>,

);

StrictMode is a tool for highlighting potential problems in an application. Like a React Fragment, it does not render any visible UI and checks are run in development mode only, not production.

The React application will render in the root element within the public/index.html file. The root element has the id of "root", but this can be changed.

Variables

In this example, we declare a variable called name and use it inside JSX by wrapping it in curly braces.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>
		</>

	);
}

export default App;

You can put any valid JavaScript expression inside the curly braces in JSX. For example, putting 2 + 2 as an expression will output the value of 4.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1>Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Classes

In JSX we cannot use the class attribute as it's a JavaScript keyword. To add a class to an element in a React component we use the className attribute. This will automatically render as a class attribute.

	

const App = () => {

	const name = 'Dan';

	return (

		<>
			<h1 className="title">Hello {name}</h1>
			<p>Welcome to my React application.</p>

			{2 + 2}
		</>

	);
}

export default App;

Button

Add a button to the index/App.js component and a function to show an alert when the button is clicked. We use the onClick prop to call the function. As with className, we use onClick in React in-place of onclick as this is a JavaScript keyword.

Change how a component renders

We may want to render our Button component as a <button> or a <link> using props. Update the Button component code:

	

import React from 'react';

const Button = props => {

	const {
		type = 'button',
		url = '',
		onClick,
		children
	} = props;

	const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'));

	const renderAsLink = () =>
		<a href={url} role="button" className="btn">{ children }</a>

	const renderAsButton = () =>
		<button {...{ type, onClick }} className="btn">{ children }</button>

	return (

		isAnchor ? renderAsLink() : renderAsButton()

	);
}

export default Button;

If we don't pass a prop of url into the button component, a normal <button> element will be rendered. If we pass in a url prop into the button component, it will render as a link. The example code below shows how we can use the prop to render the button as a link.

	

import Button from './components/Button';

const App = () => {

	return (

		<>
			<Button url="https://reactjs.org/">React</Button>
		</>

	);
}

export default App;

Using the classNames packages

To style our components we can also use the classNames npm package.

As an example we can update the Button component to apply classNames.

	

import React from 'react';
import classNames from 'classnames';

const Button = props => {

	const {
		type = 'button',
		url = '',
		className = '',
		theme = '',
		onClick,
		children
	} = props;

    const buttonClass = classNames('btn', className, {
		'btn--primary' : theme === 'primary',
		'btn--secondary' : theme === 'secondary',
		'btn--full-width' : isFullWidth
    })

	const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'));

		<LinkStyle href={url} role="button" className={buttonClass}>{ children }</LinkStyle>

	);

Each time the button renders the btn class will be applied. We can pass props into the button to change the classes. For example, if we want to the button to secondary and full-width:

	

<Button theme="primary" isFullWidth>Register</Button>

This would apply the classes of: btn btn--secondary btn--full-width

Layout Component

On most websites you may notice that there are a few repeating parts of the UI (User Interface) across all pages, such as the title, navigation, header and footer. It's often better to create one common layout component tht groups shared elements together so they can be reused.

Create an empty file named components/Layout.js with the following code:

	

import React from 'react';

const Layout = ({ children }) => {
	return (

		<>
			<header>
				header
			</header>

			<main role="main">

				{children}

			</main>

			<footer>
				footer
			</footer>
		</>

	);
}

export default Layout;

The header and footer can be moved into their own components to make it easier to structure the code.

	

import React from 'react';
import Header from './Header';
import Footer from './Footer';

	return (

		<>
			<Header />

			<main role="main">

				{children}

			</main>

			<Footer />
		</>

	);
}

export default Layout;

The Header component - components/Header.js - will look like this:

Build a Counter

We will be using the useState hook to create a simple counter that increments when the button is clicked. Hooks were introduced from React 16.8.

Create a new component named Counter.js in the components directory. Add the following code to this component.

	

import React, { useState } from 'react';
import Button from './Button';

const Counter = () => {

	const [count, setCount] = useState(0);

	return (

		<>

			<p>You clicked {count} times.</p>

			<Button onClick={() => setCount(count + 1)}>Increase Count</Button>

		</>

	);
}

export default Counter;

PropTypes

Typechecking is used to catch bugs and errors in your code. For some applications, JavaScript extensions such as Flow and TypeScript can be used to typecheck your application. React itself has built-in typechecking.

We can install the PropTypes package using npm to apply typechecking to our components.

bash -- 85x24

	npm install prop-types

We can then import the prop-types package into our Button component. For performance, typechecking is only checked during development and any errors will show in the terminal.

	

import React from 'react';
import PropTypes from 'prop-types';
import styled from 'styled-components';

const ButtonStyle = styled.button`
	cursor: pointer;
`;

const LinkStyle = styled.a`
	text-decoration: none;
`;

const Button = props => {

	const {
		type = 'button',
		url = '',
		onClick,
		children
	} = props;

	const isAnchor = url && (url.includes('http') || url.startsWith('#') || url.startsWith('mailto') || url.startsWith('/'));

	const renderAsLink = () =>
		<LinkStyle href={url} role="button" className="btn">{ children }</LinkStyle>

	const renderAsButton = () =>
		<ButtonStyle {...{ type, onClick }} className="btn">{ children }</ButtonStyle>

	return (

		isAnchor ? renderAsLink() : renderAsButton()

	);
}

Button.propTypes = {
	type: PropTypes.string
}

export default Button;

This will now check if the value passed into the type prop is a string. If it's not a string, an error will be shown.

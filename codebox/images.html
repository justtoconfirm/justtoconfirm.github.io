Images

In HTML images are defined with the <img> tag. This tag is empty, meaning it has no content or closing tag, and only contains attributes. The src attribute specifies the location of the image.

The alt attribute

The alt attribute, also called the "alt tag", is the written copy that appears in-place of an image if/when it fails to load on a user's screen. This text helps screen-reading tools to describe the image to visually impaired users and to allow search engines to better crawl then rank the page content for SEO (Search Engine Optimisation).

    

<!-- Good -->
<img src="img/escalator.jpg" alt="Man on an escalator.">

<!-- Better -->
<img src="img/escalator.jpg" alt="Man walking on an escalator.">

<!-- Best -->
<img src="img/escalator.jpg" alt="Man wearing a backpack walking on an escalator.">

The value of the alt attribute should describe the image. It is recommended to not include "image of" or "picture of" within the alt attribute text as it's assumed your text already refers to an image so it does not need to be specified.

    

<img src="img/flowers-roses.jpg" alt="Roses growing in a garden.">

If the image is decorative, such as icons or background images, using CSS background images is the preferred method. If HTML must be used, use an empty alt attribute. This will avoid screen readers having to read the image description of the image if it's not part of the content.

    

<img src="icons/facebook-icon.jpg" alt="">

Image dimensions

Dimensions of an image allows the page to be laid out with the appropriate space for the image before the file itself has been downloaded. This avoids a layout shift as the image is downloaded and is something Chrome has recently started measuring in its Cumulative Layout Shift (CLS) metric.

The width and height attribute always define the width and height of an image in pixels. Google recommends always setting the height and width of an image to be the same as the size of the image so the browser doesn't have to do any resizing calculations. If no dimensions are provided, the browser will require a reflow and repaint once the image is downloaded, which can impact performance.

    

<img src="img/flowers-roses.jpg" width="128" height="128" alt="Roses growing in a garden.">

Dimensions such as width and height on an image can cause issues when you try to alter them using CSS. If we set the max-width of an image in our HTML to be 100%, this will override the width of the image and constrain it when necessary. This will lead to a stretched or squashed images as we no longer maintain the aspect ratio of the image as the scale of the image changes.

We can fix this by adding a height property in our CSS with the value of auto so the height attribute from the HTML is overwritten too and the image scales nicely.

    

img {
    max-width: 100%;
    height: auto;
}

SEO Considerations

Search engines also read image filenames and count them towards SEO, therefore give your images descriptive filenames. The filename of flowers-roses.jpg is always better than img8645363.jpg.

It is recommended to host your images on the same server as your website. Pointing your image source to an image hosted elsewhere that you don't have permissions to use or link to, is called "hotlinking" and is not recommended. This also requires the browser to do more work by having to look up an IP address from the DNS server.

Scalable Vector Graphics (SVG)

Scalable Vector Graphics (SVG) is a unique type of image format that uses vector data rather than pixels to make up the images you see. Using SVGs, images can scale up to any resolution. An SVG image is usually much smaller than high-resolution images in other formats, helping to decrease the file size and improve performance.

  • SVG images do not lose their quality when zoomed or resized.
  • They are accessible.
  • SVG images have a small file size and are highly scalable.

An SVG file can be created by using a design program, such as Adobe Illustrator or Photoshop, and saving the image as an SVG. Existing images can be taken and converted to SVG also.

SVG files have the following browser support:

  • Chrome
  • Firefox
  • Safari
  • Opera
  • Edge
  • IE9/10/11 (Partial Support)

SVGs use XML (Extensible Markup Language) format is text-based, making them both searchable and indexable by screen readers and search engines. Screen readers can interpret SVG files as long as the correct accessibility tags are used.

SVG images are ideal for:

  • Logo & Icons - use SVG logos/icons to resolve any scalability issues across websites and social media. Icons and logos must be clear and sharp at any size.
  • Diagrams, Charts & Graphs - ideal for diagrams and illustrations that rely on plain lines.

The simplest method to add an SVG image to a webpage is to use the <img> tag. The SVG image is referenced using the src attribute.

When you add an SVG image using the <img> tag without specifying the size, it assumes the size of the original SVG file. Adding a width and height attribute can be used to set the original size.

    

<img src="img/brand-logo.svg" width="50" height="50" alt="Our brand logo.">

We can use CSS to add an SVG image to our webpage. This is ideal if we want to set the SVG image as a background image.

    

body {
    background-image: url(img/happy.svg);
}

SVG images can be written directly into the HTML document using the <svg> tag. Open the SVG image in a code editor and paste the XML directly into your HTML inside of the <svg> and </svg> tags inside of the <body>

When using SVG as inline in the HTML, it reduces load times because it serves as an HTTP request. Using the inline method also lets you perform more customisation as opposed to using the <img> tag or background-image methods.

You can also use an HTML <object> element to add SVG images to a webpage using the code syntax below:

    

<object data="img/brand-logo.svg" width="50" height="50"> </object>

We use the data attribute to specify the URL of the resource that you'll use by the object.

HTML5 <picture> element

HTML5 introduced the <picture> element to add more flexibility when specifying image resources. This element contains a number of <source> elements, each referring to a different image source. This allows the browser to choose the image file that best fits the current viewport size.

This is a container for the <source> and <img> tags.

The <img> tag is used by the browser to render the image on the page. Browsers that do not support the <picture> tag will automatically render the image in the <img> tag.

    

<picture>
    <source media="(min-width: 64rem)" srcset="img/image-example-350-200.jpg">
    <source srcset="img/image-example.jpg">
    <img src="img/image-example.jpg" height="240" width="300" alt="An example image description here.">
</picture>

WebP

WebP is a modern image format that provides superior lossless compression for images on the web. WebP images are 26% smaller in size compared with PNG files and 25-34% smaller than JPEG files. This format also supports trasparency (alpha channel). The visual quality between files is comparable.

WebP is natively supported by Google Chrome, Firefox, Edge and Opera browsers.

We can use the HTML5 <picture> element to apply WebP images with fallbacks for alternative image formats, such as JPEG and PNG, in browsers that do not support WebP.

    

<picture>
    <source srcset="img/webp-image.webp" type="image/webp>
    <source srcset="img/image.jpg" type="image/jpeg>
    <img src="img/image.jpg" height="240" width="300" alt="An example image description here.">
</picture>

Vue

Vue.js, is an open-source front-end JavaScript framework for building user interfaces and single-page applications (SPA).

Introduction

Vue can be used as much or as little in our applications as we wish. Similar to how jQuery is used, we can build fully-featured single-page applications (SPA) using Vue or parts of an application using this framework.

Requirements

You will need to have the following installed on your machine to install and run React:

  • Node 8.11 or later
  • npm or yarn

You can use nvm (Node Version Manager) or nvm-windows to switch Node versions between projects.

Install Vue

Vue does not support IE8 and below, because it uses ECMAScript 5 features that are un-shimmable in IE8.

The recommended installation method for Vue is npm to install the latest stable version. Vue pairs well with module bundlers such as Webpack.

bash -- 85x24

	npm install vue

To install the CLI (Command-Line Interface), run the following command in your terminal:

bash -- 85x24

	npm install @vue/cli --global

Once installed initialise a new project from within a terminal in the directory where we want to create the project. Replace <project-name> with the name of your project.

The CLI will give you a list of options to configure. These options let you configure things like TypeScript, linting, vue-router, testing, and more.

bash -- 85x24

	vue create <project-name>

Run development server

Once installed, navigate into the directory of the Vue app and start the development server.

bash -- 85x24

	cd <project-name>
	npm run serve

Our Vue app is now running on http://localhost:8080/ locally and on the network.

Once installed, our file structure will look like this:

	

my-app 
	|_node_modules /
	|_public /
		|_favicon.ico
		|_index.html
	|_src /
		|_assets /
			|_logo.png
		|_components /
			|_HelloWorld.vue
		|_App.vue
		|_main.js
	|_.gitignore
	|_babel.config.js
	|_package-lock.json
	|_package.json
	|_README.md

Creating a production build

The Vue app is still in development mode and not optimised for production, so we need to build it once complete to optimise it for production when ready.

bash -- 85x24

	npm run build

Getting Started

The src/App.vue file is the root component of our app where all our components are nested. The src/main.js takes the src/App.vue file, renders it and mounts it to our public/index.html file.

This example is using Vue 3.

Open the src/main.js file and set the following code:

	

import { createApp } from 'vue'
import App from './App'

createApp(App).mount('#app')

In the src/App.vue file, update the code to include the HTML we want to render between an opening and closing <template> tag.

	

<template>
	<h1>Hello World</h1>
</template>

The file stucture of our app has been simplified and will look like this:

	

my-app 
	|_node_modules /
	|_public /
		|_index.html
	|_src /
		|_App.vue
		|_main.js
	|_.gitignore
	|_babel.config.js
	|_package-lock.json
	|_package.json
	|_README.md

Adding Components

Create a components directory in the src folder. Within this newly-created directory, create a component named HelloWorld.vue. The file structure will now look like this:

	

my-app 
	|_node_modules /
	|_public /
		|_index.html
	|_src /
		|_components /
			HelloWorld.vue
		|_App.vue
		|_main.js
	|_.gitignore
	|_babel.config.js
	|_package-lock.json
	|_package.json
	|_README.md

Within the src/components/HelloWorld.vue component, update the code.

	

<template>
	<h1>Hello World Component</h1>
</template>

Next, update the src/App.vue file render the component.

	

<template>
	<HelloWorld />
</template>

<script>
import HelloWorld from './components/HelloWorld'

export default {
    components: {
        HelloWorld
    }
}
</script>

Variables

Variables are containers for storing data.

Declaring Variables

To use a variable, you've got to first create it - more accurately, we call this declaring the variable.

JavaScript is a dynamically typed language, which means that unlike some other languages, you don't need to specify what data type a variable will contain (numbers, strings, arrays, and more).

There are three ways to declare variables in JavaScript:

  • var
  • let
  • const

We type the keyword of the variable - var, const, or let - followed by the name you want to call your variable.

Variables should always be declared with var, let, or const. The let and const keywords were added to JavaScript in 2015.

It's recommended to always declare variables with const. Use const when you can, and use let when you have to.

	

const variableName = 'value';

If you think a value of a variable can change, use let.

	

let variableName;

If an empty variable has no value, they are empty containers. If a variable is is without a value, undefined is returned.

If is recommended to always declare variables are the top of their scope (the top of global code and the top of function code) so it's clear which variable are function scoped (local) and which are resolved on the scope chain.

Initializing a variable

Once a variable has been declared, you can initialize it with a value. You do this by typing the variable name, followed by an equals sign (=), followed by the value you want the variable to have.

	

const myName = 'Dan';
const myAge = 25;

Updating a variable

Once a variable has been initialized with a value, you can change (or update) that value by giving it a different value.

	

let myAge = 25;
myAge = 36;

This is also called reassigning the variable.

Constants in JavaScript

As well as variables, you can declare constants.

  • You must initialize them when you declare them.
  • You cannot assign a constant with a new value after you've initialized them.

This will lead to an error as the constant is not initialized with a value:

	

const count;

Trying to do this using const will also lead to an error:

const count = 1; count = 2;

You can update, add, or remove properties of an object declared using const, because even though content of the object has changed, the constant is still pointing to the same object.

	

const dog = {
	name: 'Spot',
	breed: 'Dalmation',
	gender: 'Male'
};

dog.name = 'Fido';

console.log(dog.name);			// Fido

const

Variables declared with const maintain constant values. This type of variable can only be accessed within the block they were declared - they are block scoped.

The value declared within const remains the same within its scope and cannot be updated or re-declared. We cannot do this:

	

let greeting = "Say Hello";
greeting = "Say Hi instead";		// error: Assignment to constant variable.

Every const declaration must have a value and be initialized when declared.

Scope

Scope essentially means where variables are available for use. Variables can have global scope or block/local scope.

Hoisting

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution.

var

Any variable declared with var outside a function block is available for use in the whole window. The scope is global when a var variable is declared outside a function. Declarations using var are globally scoped or function/locally scoped.

var is function scoped when it's declared within a function. This means that is it available and can be accessed only within that function.

	

var greeting = "Hello World";

const func = () => {
	
	var hello = "Say Hello";

};

console.log(hello);			// error: hello is not defined

Here, greeting is globally scoped because it exists outside of a function while hello is function scoped. We cannot access the variable hello outside of a function.

let

let is now the preferred method of declaring variables. A let variable is block scoped, meaning it is only available for use within that block {}.

	

let greeting = "Hello World";

const func = () => {
	
	let hello = "Say Hello";

};

console.log(hello);			// hello is not defined

We see that using hello outside the function - it's block - returns an error. This is because let variables are block scoped.

let can be updated but not re-declared

Like var, a variable declared with let can be updates within its scope. Unlike var, a let variable cannot be re-declared within its scope.

	

let greeting = "Say Hello";
greeting = "Say Hi instead";

This will return an error, unlike var:

	

let greeting = "Say Hello";
let greeting = "Say Hi instead";		// error: Identifier 'greeting' has already been declared

If the same variable is defined in different scopes, there will be no error:

	

let greeting = "Hello World";

const func = () => {
	
	let greeting = "Say Hello";
	console.log(greeting);			// Say Hello

};

console.log(hello);			// Hello World

Both instances are treated as different variables since they have different scopes - no errors. We can use variables with the same name as a variable only exists within its scope.

Always declare a let variable before using it to avoid a Reference Error.

Objects

In programming, an object is a structure of code that models a real-life object. You could have a simple object that represents a box and contains information about its width, length, and height. Another example could be an object that represents a person, containing data about their name, height, weight, language, and more.

	

const dog = {
	name: 'Spot',
	breed: 'Dalmation',
	gender: 'Male'
};

To retrieve the information stored in the object, we can use the dot.notation syntax:

	

dog.name

CSS Animations & Transitions

CSS allows animation of HTML elements without the need to use JavaScript. An animation lets an element gradually change from one style to another. You can change as many properties as you want and as many times as you want.

CSS animations are well supported in most modern browsers, with the exception of Internet Explorer (IE) 9 and below.

@keyframes rule

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style. The animation must bind to an element.

    

/* The animation */
@keyframes background-color-change {
    from {
        background-color: red;
    }
    to {
        background-color: yellow;
    }
}

/* Apply the animation to the element */
.box {
    width: 100%;
    height: 100px;
    background-color: red;
    animation-name: background-color-change;
    animation-duration: 4s;
}

The animation-name property references the animation to be used.

The animation-duration property defines how long an animation should take to complete. This is required as without the property, no animation will occur as the default value is 0.

In the example, the "from" keyword represents 0% (start) and the "to" keyword represents 100% (completed). It is possible to use percent to allow multiple style changes to be added.

    

/* The animation */
@keyframes background-color-change {
    0% {
        background-color: red;
    }
    25% {
        background-color: yellow;
    }
    50% {
        background-color: blue;
    }
    100% {
        background-color: green;
    }
}

Further properties can be added to each @keyframe if needed also.

    

/* The animation */
@keyframes background-color-change {
    0% {
        background-color: red;
        left: 0;
        top: 0;
    }
    25% {
        background-color: yellow;
        left: 200px;
        top: 0;
    }
    50% {
        background-color: blue;
        left: 200px;
        top: 200px;
    }
    75% {
        background-color: green;
        left: 0;
        top: 200px;
    }
    100% {
        background-color: red;
        left: 0;
        top: 0;
    }
}

The animation-delay property specifies a delay for the start of an animation.

    

/* Apply the animation to the element */
.box {
    animation-delay: 2s;
}

The animation-iteration-count property specifies the number of times an animation should run. The following example will run the animation three times before it stops.

    

/* Apply the animation to the element */
.box {
    animation-iteration-count: 3;
}

If we want to animation to be "infinite" we can also use the animation-iteration-count and update the value.

    

/* Apply the animation to the element */
.box {
    animation-iteration-count: infinite;
}

We can also use the shorthand property of animation to chain multiple values together.

    

/* Apply the animation to the element */
.box {
    animation: background-color-change 5s infinite;
}

So to summarise, the animation property will be the shorthand for:

    

.box {
    animation: animation-name | animation-duration | animation-timing-function | animation-delay | animation-iteration-count | animation-direction
}

Specify the Speed Curve

The animation-timing-function property specifies the speed curve of the animation. This property can have the following values:

  • ease - specifies an animation with a slow start, then fast, then end slowly (this is the default)
  • linear - specifies an animation with the same speed from start to end
  • ease-in - specifies an animation with a slow start
  • ease-out - specifies an animation with a slow end
  • ease-in-out - specifies an animation with a slow start and end

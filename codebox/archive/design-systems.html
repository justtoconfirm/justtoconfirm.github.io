Design Systems

A Design System is an ever-evolving set of design guidelines and reusable elements. Design Systems contain both Style Guides and Pattern Libraries.

Here is Atlassian's Design System.

And here is an blog post on how AirBnB created their visual design language.

The aim of a Design System is to provide a toolbox of "ready-to-go" UI components that can be used for any given application. Instead of seeing a web page as one solid object, you should see it as many independent parts, or components (like Lego bricks).

Style Guide

A Style Guide is a rulebook that explains how a brand presents itself and should be used as a reference document to help maintain consistency to set out your brand guidelines.

It is a document that contains design guidelines and outlines the visual identity and set the guidelines for your brand. The styleguide usually includes things like:

  • Typography (Font/Typeface)
  • Icons
  • Imagery
  • Colours - the colour palette used
  • UI Elements - Logos, Buttons & Headings
  • Tone of Voice (ToV)

The style guide is often developed after the high-fidelity wireframes have been created so that you can see what element are needed.

This guide is a living, breathing document and you must plan for the evolution of your brand.

A Living/Live Style Guide is similar to a normal static Style Guide in the sense that it also sets a group of rules and guidelines for the brand, but it has the advantage of including code elements that can help the development team to focus on building the application without having to code first.

Pattern Libraries

A Pattern Library is a library of User Interface (UI) patterns that designers and design teams use to build digital products. This most often takes the form of a document in a tool such as Sketch, containing a wide range of different patterns organised into specific groups - from smaller components such as Buttons, Labels and Icons, through to larger, more complex components such as Navigation and Headers.

Similar to Lego, designers are able to choose which pieces they need, then snap them together to build a range of different UI elements/components. This approach of building larger components from smaller ones is often referred to as Atomic Design.

A tool like Zeplin can be used to create a Pattern Library/Styleguide within the Design System. Zeplin will also generate code to assist developers. Zeplin is a plugin that takes designs from Sketch, Adobe XD, Figma, or Adobe Photoshop CC and exports them into a format which generates code snippets, design specs and assets. It's defined as a collaboration tool for UI designers and developers.

Figma is a vector graphics editor and prototyping browser-based tool for UI and UX design.

Component Library

A Component Library is a set of re-usable components available to developers that can be used across multiple projects and/or applications. These components serve as the building blocks for layouts, and thanks to their modular nature, we can arrange components in many ways to achieve different effects.

Here is Atlassian's Component Library.

Examples of re-usable components include:

  • Buttons
  • Inputs
  • Modals

Storybook can be used as an open-source tool to compliment a Component Library. This is a tool for developing UI components and pages in isolation and simplifies building, documenting and testing UIs.

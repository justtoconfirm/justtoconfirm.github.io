DOM (Document Object Model)

The Document Object Model, usually referred to as the DOM, is an essential part of making websites interactive. It is the inteface that allows a programming language to manipulate the content, structure and style of a website.

At the most basic level, a website consists of an HTML document. The browser being used to view the website is a program that interprets HTML and CSS to render the style, content, and structure into the page you see. In addition to parsing and rendering the page, the Document Object Model is created which is a representation of the document.

If you inspect a website within the Developer Tools, and select the Elements tab, you'll see the DOM. This may look similar to your HTML source code.

The Document Object

The document object is a built-in object that has many properties and methods that can be used to access and modify web pages.

In Developer Tools, move to the Console tab. Type document into the console and press ENTER. You will see the same output as what you see in the Elements tab.

DevTools: Console

    > document

document is an object, body is a property of that object that we can access using dot notation. Submitting document.body to the console outputs the <body> element and everything inside it.

DevTools: Console

    > document.body

Within the console, we can change some of the live properties of the body object using the style attribute.

DevTools: Console

    > document.body.style.backgroundColor = 'fuchsia';

In Developer Tools, switching the the Elements tab, you'll see that the DOM has now changed.

In order to change the background-color CSS property, we had to type backgroundColor in JavaScript. Any hyphenated CSS property will be written in camelCase in JavaScript.

    

<body style="background-color: fuchsia;">

</body>

We have now updated the DOM, however the source of the website does not contain the new style attribute. If you refresh the page, the new code we added in the console will disappear.

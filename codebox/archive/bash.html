Bash

Bash is a Unix shell which is a command line interface (CLI) for interacting with an operating system (OS). Bash is available by default on Linux and MacOS operating systems. Newer MacOS installations (from Catalina) come installed with Z Shell (zsh) as the new default.

Creating a bash script

Any command that you can run from the command line can be used in a bash script. Scripts are used to run a series of commands.

Create a file called hello-world:

bash -- 85x24

    $ touch hello-world

Edit the file to print a string that says "Hello world" using echo:

    

echo "Hello world"

Now from the command line, run the script using the bash interpreter:

bash -- 85x24

    $ bash hello-world

This script will run from the command line with the bash interpreter, but it's not executable on its own. For the file to run directly, we need to change the permissions to allow the script to be executable. chmod is a command that changes permissions on a file and +x will add execute rights to the script.

bash -- 85x24

    $ chmod +x hello-world

Within the hello-world bash script file, we will need to add a shebang (#!) at the top of the script. In Unix-like systems, a file with a shebang is interpreted as an executable file.

    

#!/bin/bash

echo "Hello world"

Now you can run the hello-world script directly as it's executable. The prefixing of the file with ./ means the file is in the current directory.

bash -- 85x24

    $ ./hello-world

In order to run a bash script without specifying the directory, you would need to add the directory of the script to the PATH running in your .bash_profile:

    

export PATH=$PATH:/path/to/script/directory

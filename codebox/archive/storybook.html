Storybook

What is Storybook?

Storybook is an open-source tool for developing UI (User Interface) components in isolation for React, Vue, Angular, and more. This allows you to work on one component at a time and entire interfaces can be developed without the need to start-up a complex development stack or navigate around your application.

What is a story?

A story captures the rendered state of a UI component. Developers create multiple stories per component that describe each state a component can support.

Each story uses the file extension of .stories.js or .stores.ts and are written in Component Story Format (CSF), an ES6 modules-based standard for writing components.

Install Storybook

To install Storybook for React, use NPM (Node Package Manager) to install Storybook as a devDependency.

bash -- 85x24

    $ npm install @storybook/react --save-dev

Update your package.json file to include a script to run Storybook. When you run the script, Storybook will start a local server on port 9001 and open a browser tab/window.

    

"scripts": {
    "storybook": "start-storybook -p 9001 -c .storybook"
},

In the root of your React project, create a directory named .storybook, this will be where our Storybook config file will be placed. Inside this directory, create a new file named .config.js and add the following code:

    

import { configure } from '@storybook/react'

function loadStories() {
    // You can require as many stories as you need.
    require('./../src/Button/storybook/Button.stories.js')
}

configure(loadStories, module)

In this example, the config file tells Storybook where it can find the Story for the Button component. Any Story you add needs to be added into this config file.

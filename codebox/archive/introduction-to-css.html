Introduction to CSS

CSS is not the largest asset to consider during development but it is the first the browswer discovers. The browser is blocked from rendering the page until the CSS is downloaded and parsed. It's always worthwhile making your CSS stylesheets as lean as possible to improve performance of your web page, website or application.

Adding CSS to HTML

Adding an external CSS stylesheet - a file with the extension .css - to a HTML document requires the following code to be added between the <head> and </head> tags:

    

<link rel="stylesheet" href="stylesheet.css">

You may want to use a minified CSS file to improve performance and page speed. This may have a slightly different file name.

    

<link rel="stylesheet" href="stylesheet.min.css">

Comments

Comments in CSS begin with /* and end with */.

    

/* This is a comment in CSS */

Prefixes

Prefixes are no-longer required as they once were. CSS properties that begin with a dash were considered as prototype properties and were not initially intended for production. Tools such as Autoprefixer can be used to configure your code to remove prefixes that are no longer needed.

Feature Queries

CSS has developed a way for you to test the browser support of new CSS features with the introduction of feature queries. A feature query behaves in much the same way as a media query, except that instead of asking the browser something about the device being used to view the site, a Feature Query asks the browser if it supports a particular CSS feature. This makes it easier to use new features in a safe, progressively enhanced way.

A feature query does not need to be used for every CSS property and you should first write your CSS for browsers which do not support the feature to then test for support of the property you want to use. If the browser confirms support, it will overwrite the fallback code with the code inside the feature query.

This is pretty much the same process that you might adopt when using media queries for responsive design following a mobile-first approach. In that approach, you start with your layout for smaller screens then add or overwrite things for larger ones as you move up through your breakpoints.

Media queries are supported by most modern browswers (IE11 and below is not supported) and means we can remove the need to use tools such as Modernizr.

    

@supports (display: grid) {
    .item {}
}

/* Does not support the CSS grid property */
@supports not (display: grid) {
    .item {}
}

/* Fallback code */
.item {}

Feature queries are really useful for testing new CSS features and properties quickly.

    

/* Support CSS variables */
/* support: http://caniuse.com/#feat=css-variables */
@supports (background-color: var(--bg-color)) {
    body {
        background-color: var(--bg-color);
    }
}

@supports not (background-color: var(--bg-color)) {
    body {
        background-color: #ff6347;
    }
}

Currently feature queries can only test CSS properties and values. We may want to test CSS selectors. For example, if you visit about:config in Firefox and enable the flag layout.css.supports-selector.enabled, you can test to see if various selectors are supported. For example, the test for the :has selector:

    

@support selector(:has) {
    .item {}
}

This specification is currently in development.

CSS Reset

A CSS Reset allows you to start with a clean code base, making it easier to style your website with more predictable outcomes. A CSS Reset resets or overrides the default styles of the browser. You can write your own, or use an existing CSS reset that maybe available.

Every browser renders content a little differently using the built-in user agent stylesheet, so we can use a reset such as Reset or Normalize to combat these differences using either of these stylesheets. Reset completely eradicates all styles, while Normalize attempts to add consistency to the browser styling.

You may also want to consider resetting the border box - not always included within the reset stylesheets - to make padding consistent and prevent frustration.

    

html {
    box-sizing: border-box;
}

*, *:before, *:after {
    box-sizing: inherit;
}

Setting all elements to have the box-sizing property set to border-box will ensure that the padding and border are included in the total width and height of the elements. Adding the following code will work:

    

* {
    box-sizing: border-box;
}

By setting the border-box, this will also ensure that if an element is full-width it won't then overflow outside the area of it's parent element. It's therefore always useful to set this.

The display property

The display property specifies the display behavior of an element and is used to control layout. The property specifies if and how an element is displayed. Every HTML element has a default display value. The default display value for most elements is block or inline.

Every element has a default value, however, we can override this. Changing an element from inline to block, or vice versa, can be useful for making the page look a specific way and still following web standards. A common example is making the inline <li> elements horizontal for menus.

    

li {
    display: inline;
}

Setting the display property of an element only changes how the element is displayed. An inline element is still not allowed to have a block element inside it.

An inline element does not start on a new line and only takes up as much width as necessary. Any height and width properties will have no effect.

  • <span>
  • <a>
  • <img>

inline-block displays an element as an inline-level block container. The element itself is formatted as an inline element, but you can apply height and width values.

block elements start on a new line and takes up the whole width available.

  • <div>
  • <h1-6>
  • <li>
  • <p>
  • <header>
  • <footer>
  • <form>

Elements can also be hidden using the display property. Setting the display property to none, the element will be hidden and the page will be displayed as if the element is not there.

    

.element {
    display: none;
}

If we use the visibility property in-place of display, this too hides the element, however, the element will still take up the same space as before. The element will be hidden, but still affect the layout.

    

.element {
    visibility: hidden;
}

Example HTML

Here is an example HTML code to demonstrate our CSS examples:

    

<div class="container">
    <!-- Child element(s) -->
    <p>Paragraph one.</p>
    <p>Paragraph two.</p>

    <div class="container__summary">
        <!-- Grandchild elements(s) -->
        <p>Paragraph three.</p>
    </div>

    <!-- Child element(s) -->
    <p>Paragraph four.</p>
    <p>Paragraph five.</p>
</div>

CSS Naming Conventions

In CSS, it's better the strive for reusable, recyclable CSS classes rather than writing for specific use cases. This is critical to a successful naming convention.

We can create a class that will set the colour of an element blue when used. In this example, the class name used is too specific and risks being difficult to maintain. If the colour changes, the class name becomes irrelevant and will need to be updated across your HTML.

    

.blue {
    color: blue;
}

The next example depends on the location to render the code correctly using element selectors in-place of a class:

    

header span {
    color: blue;
}

We could create a class that is defines the purpose of the code, but this can be too specific also:

    

.header-color {
    color: blue;
}

Far better to create a reusable, portable and easily maintained class to keep your CSS clean:

    

.highlight-color {
    color: blue;
}

The Single Responsibility Principle is a paradigm that states that all pieces of code should focus on doing one thing only. In CSS, our selectors should be small and provide very specific functionality.

In this example, we have two CSS class selectors that handle layout, structure and styling. This can lead to lots of repetition between classes.

    

.error-message {
    display: block;
    padding: 10px;
    border-top: 1px solid #f00;
    border-bottom: 1px solid #f00;
    background-color: #fee;
    color: #f00;
    font-weight: bold;
}

.success-message {
    display: block;
    padding: 10px;
    border-top: 1px solid #0f0;
    border-bottom: 1px solid #0f0;
    background-color: #efe;
    color: #0f0;
    font-weight: bold;
}

We can apply the Single Responsibility Principle to this code and break these classes into smaller classes with specific tasks, applying abstraction.

    

.box {
    display: block;
    padding: 10px;
}

.message {
    border-style: solid;
    border-width: 1px 0;
    font-weight: bold;
}

.message--error {
    background-color: #fee;
    color: #f00;
}

.message--success {
    background-color: #efe;
    color: #0f0;
}

It is often best to follow the DRY (Don't Repeat Yourself) principle in CSS. This helps us to avoid writing the same bits of code multiple times. CSS is best written when it follows this principle and one way to keep our CSS DRY is to group code together.

    

.menu li {
    color: red;
}

.menu li a {
    color: red;
}

This code example (above) is not DRY as both selectors set the same property and value. We can group these selectors together to turn our CSS DRY:

    

.menu li,
.menu li a {
    color: red;
}

Universal Selector (*)

Universal Selectors such as *, [disabled] and [type="text"], check every element in the DOM (Document Object Model) for a match. This can be very expensive for the browser when considering performance. This selector is often used for box-sizing and other globals.

    

* {
    box-sizing: border-box;
}

Descendent Selector (space)

Adding space between selectors in CSS is known as a descendent selector. In this example, we will target all <p> tags within the element that has the container class and set the font-weight to be bold.

    

.container p {
    font-weight: bold;
}

The Child Combinator (>)

The child combinator will only target direct children of a particular element and only those that are nested directly within the element. This will apply only to child elements and nothing nested further (grand child elements).

    

.container > p {
    font-weight: bold;
}

Adjacent Sibling Combinator (+)

Combines two selectors that have the same parent. The Adjacent Sibling Combinator will only select the first element that is immediately proceeded by the former selector. In this example, the <p> tag immediately after the container__summary element will be selected.

    

.container__summary + p {
    font-weight: bold;
}

Sign (~)

We can select all elements that proceeded the former selector. Here, we can target all <p> tags immediately after the container__summary element.

    

.container__summary ~ p {
    font-weight: bold;
}

Dynamic suffix

You can use CSS to select elements that have dynamic values. For example, if multiple elements have id attribute values of 01234_loginPanel or 43210_loginPanel, we can use the following CSS:

    

div[id$="_loginPanel"] {
    font-weight: bold;
}

We can also target elements with specific values, such as loginPanel:

    

div[id^="loginPanel_"] {
    font-weight: bold;
}

We can also check if an element contains a suffix and prefix:

    

div[id*="loginPanel"] {
    font-weight: bold;
}

We can also target attributes of elements using Attribute Selectors. Here, we will target all links whose href attribute values end with .pdf:

    

a[href$=".pdf"] {
    font-weight: bold;
}

Pseudo Selectors

Pseudo Selectors are CSS selectors with a colon (:) proceeding them. Links often use Pseudo Selectors for state.

    

a:hover {}
a:active {}
a:focus {}
a:visited {}

The ::first-letter selector

Selector to target the first letter of a paragraph or element.

    

p::first-letter {
    font-weight: bold;
}

The ::first-of-type & ::last-of-type selectors

Select the first or last of any type within the current parent element.

    

p::first-of-type {
    font-weight: bold;
}

p::last-of-type {
    font-weight: bold;
}

The ::only-of-type selector

Target elements that do not have any siblings within the parent class. Example maybe an unordered list with a single list item.

    

li::only-of-type {
    font-weight: bold;
}

The :empty selector

Represents an element that has no content within. You may have multiple elements, such as <p> in rendered HTML with no content. The :empty selector can be used to hide the empty elements on the page.

    

p:empty {
    display: none;
}

You could use this selector for features like search results:

    

.search-results:empty {
    display: none;
}

The :is selector

The :is pseudo-class works by taking a passed selector list then translating it into an expanded selector list for us. This allows us to write more compact code. Aside from IE11 and Opera Mini, this has excellent browser support.

When writing CSS, we can usually find ourselves writing bits of repeating code, as this example demonstrates:

    

.post h1,
.post h2,
.post h3 {
    line-height: 1.2;
}

.post img,
.post video {
    width: 100%;
}

Thankfully, the :is pseudo class can be used to simplify our code:

    

.post :is(h1, h2, h3) {
    line-height: 1.2;
}

.post :is(img, video) {
    width: 100%;
}

Arrows

We can create arrows in CSS, for elements such as tooltips, using the CSS pseudo classes of ::before and ::after. The border properties set the colour and direction of the arrow. Here we are setting the arrow to have a solid colour and a different border colour.

    

<div class="arrow-up"></div>

// Colour of the triange
.arrow-up:after {
    content: '';
    position: absolute;
    top: 53px;
    left: 0;
    right: 0;
    margin: 0 auto;
    width: 0;
    height: 0;
    border-left: 9px solid transparent;
    border-right: 9px solid transparent;
    border-bottom: 9px solid white;
}

// Border of the triange
.arrow-up:before {
    content: '';
    position: absolute;
    top: 52px;
    left: 0;
    right: 0;
    margin: 0 auto;
    width: 0;
    height: 0;
    border-left: 9px solid transparent;
    border-right: 9px solid transparent;
    border-bottom: 9px solid red;
}

Scroll Margin

When a browser skips to an element with an id, the targeted element would sit flush to the top of the viewport. Not only did this not look great but it caused issues with fixed headers too.

The property scroll-margin-top can be used to fix this. Browser support is pretty good, only IE11 seems to not support this property.

    

[id] {
    scroll-margin-top: 2px;
}

:any-link

This pseudo class has had cross-browser support for sometime, with the exclusion of Internet Explorer (IE). This can be used to match an anchor hyperlink as long as it has a href attribute and is the equivalent to both :link and :visited at once.

This may reduce your styles by one selector if you're adding basic properties such as color to apply to all links.

    

:any-link {
    color: blue;
}

An important note about specificity, this property will win against a as a selector even if a is placed lower in the CSS cascade.

Lists

Lists are indented into the page by default and the list-style-position property indicates whether the marker should appear on the inside or outside.

    

.list {
    list-style-position: inside | outside;

}

.element {
    background: radial-gradient(circle, black, white);
}

Colour

You can specify colour using CSS in one of three ways:

  • RGB values - these express colours in terms of how much red, green and blue are used.
  • Hex codes - six-digit codes that represent the amount of red, green and blue are used preceded by a pound/hash sign.
  • Colour names - there are 147 predefined colour names that are recognised by the browser.
    

/* RGB */
.element {
    background-color: rgb(100,100,90);
}

/* Hex code */
.element {
    background-color: #ee3e80;
}

/* Colour name */
.element {
    background-color: DarkCyan;
}

CSS3 introduced the opacity property allowing you to specify the opacity of an element and any of its children. The value is a number between 0.0 and 1.0. So, a value of 0.5 is 50% and a value of 0.15% is 15% opacity.

    

.element {
    background-color: rgb(0,0,0);
    opacity: 0.5;
}

The CSS3 rgba property allows you to specify a colour, just like an rgb value, but adds a fourth value to specify opacity. This value is known as an alpha value and is between 0.0 and 1.0. So a value of 0.5 is 50% opacity and 0.15 is 15% opacity. The rgba value will only affect the element on which it's applied.

    

.element {
    background-color: rgb(0,0,0);
    background-color: rgba(0,0,0,0.5);
}

All modern browsers support rgba including IE9-11.

The hsl colour property was introduced in CSS3 as an alternative way to specify colour.

  • Hue - expressed as an angle, between 0 and 360 degrees.
  • Saturation - this is expressed as a percentage.
  • Lightness - this is expressed as a percentage with 0% being white and, 50% normal, 100% black.
  • Alpha - this is for hsla for opacity, with values between 0.0 and 1.0.
    

.element {
    background-color: #c8c8c8;
    background-color: hsl(0,0%,78%);
}

.element {
    background-color: #fff;
    background-color: hsla(0,100%,100%,0.5);
}

Support for hsl is really good in modern browsers.

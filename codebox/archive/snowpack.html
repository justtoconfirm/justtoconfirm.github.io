Snowpack

Snowpack is a frontend build tool and is an alternative to Webpack or Parcel. Snowpack uses JavaScript's native module system, known as ESM (ES Modules), to avoid unnecessary work and improve speed.

ECMAScript modules (ESM) uses import and export keywords and are powered by Web Assembly, making Snowpack very fast. Snowpack also has out-of-the-box support for TypeScript, JSX, CSS Modules, Hot Module Reloading, and more.

Snowpack is a package installed from NPM. Create a package.json file in your project directory to manage dependencies.

bash -- 85x24

    $ npm init -y

You can install Snowpack using NPM (Node Package Manager) or Yarn.

bash -- 85x24

    # npm
    $ npm install snowpack --save-dev

    # yarn
    $ yarn add snowpack --dev

For long-term development, the best way to use Snowpack is with a package.json script. This reduces the need to remember exact Snowpack commands and lets you share common scripts.

    

// Recommended: package.json scripts
// npm run start (or: "yarn run ...")
"scripts": {
    "start": "snowpack dev",
    "build": "snowpack build"
}

The following command will start the local dev server for development. By default, this serves your current working directory to the browser and will look for an index.html file to start.

bash -- 85x24

    $ npm run start

Build a simple HTML document named index.html and run the start command to start the Snowpack development server. If successful, Snowpack automatically opens your site in a new browser window. Any changes made to the HTML file while the server is running will refresh the page and the changes will automatically show.

Building your project will build into a static build/ directory that you can then deploy anywhere.

bash -- 85x24

    $ npm run build

Creating a Config file

A config file may not always be needed, but if you need to create a snowpack.config.js file, you can initialise it using a single command:

bash -- 85x24

    $ npx snowpack init

Setting up React

Snowpack can be used to setup React applications quickly without the need for Babel. Browsers don't understand JSX (or TypeScript), so any compile-to-JavaScript file formats compile to .js in the final browser build. This is why we often use Babel when using Webpack.

Begin by installing React and ReactDOM as dependencies using NPM:

bash -- 85x24

    $ npm install react react-dom --save

Create a simple index.jsx file in your project folder with the following code:

    

import React from 'react';
import ReactDOM from 'react-dom';

ReactDOM.render(<h1>Hello World</h1>, document.getElementById('root'));

Create an HTML file, named index.html, where your application will render.

    

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>Snowpack Test</title>
</head>
<body>

<div id="root"></div>

<script type="module" src="/index.js"></script>

</body>
</html>

Run your build and the localhost server will start to display your React application.

When Snowpack serves our .jsx file it automatically transpiles and serves it as .js

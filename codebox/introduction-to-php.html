Introduction to PHP

PHP (PHP Hypertext Preprocessor) is an open-source, server-side scripting language.

Install & Setup PHP

If we use MAMP PRO to run our PHP code localhost will default to: http://localhost:8888/.

Once MAMP PRO has been installed a Sites directory will be created where we can add our PHP files. For example, the document root for this machine will be: /Users/danieljackson/Sites/ so any PHP files we want to run will need to be added to this directory.

If we create a file named test.php and save it to the root directory we can see our PHP code running on: http://localhost:8888/test.php if our server is running.

PHP Syntax

A PHP script starts with <?php and ends with ?>. The default file extension for PHP files is .php

	

<?php

?>

For PHP files that only contain PHP code, the closing ?> tag is not required. Omitting it prevents the accidental injection of trailing white space.

	

<?php

We can use the echo statement to output a value to the screen. All PHP statements should end with a semi-colon.

	

<?php

	echo "Hello World";

?>

The PHP configuration

The phpinfo() method outputs information about the current PHP configuration.

	

<?php

	phpinfo();

?>

Adding Comments to PHP

A comment in the PHP code is a line that is not executed within the program and is used to explain to the developer what the code is doing.

	

<?php

	// This is a single-line comment in PHP

	# This is also a single-line comment in PHP

	/*
		This is a multi-line comment block in PHP
		that can span over multiple lines
		of code
	*/

?>

PHP Variables

In PHP, a variable is declared (created) with the dollar sign ($), followed by the name of the variable.

	

<?php

	$name = "John Smith";
	$age = 25;

?>

We can use the echo statement to output the value of a variable.

	

<?php

	$name = "John Smith";
	$age = 25;
	$currentYear = 2021;

	echo "<h2>" . $name . "</h2>";
	echo "<p>" . $age . "</p>";

	echo $currentYear - $age;

?>

The var_dump() function

The var_dump() function dumps information about PHP variables onto the page.

	

<?php

	$name = "John Smith";

	var_dump($name);

?>

This will output the type and value of the variable:

	

string(10) "John Smith"

Functions

We can declare functions in PHP using the keyword of function. In the code example below, we can also demonstrate how to call the function.

	

<?php

	function functionName() {
		
		// Code to be executed 

	}

	functionName();		// Call the function

?>

Function Arguments

Information can be passed to function through the use of arguments. An argument is placed within the parenthesis of a function and are like variables. We can add as many arguments as we wish, just separate them with a comma.

	

<?php

	function name($firstName) {
		
		echo "Hello $firstName";

	}

	name("Dan");

?>

Since PHP 7 the type declarations were introduced. This provides an option to specify the expected data type when declaring a function.

To apply this feature, we add the strict declaration to the top of the PHP script. This will throw a Fatal Error if the data type is not correct.

	

<?php declare(strict_types=1);		// The strict declaration

	function addNumbers(int $a, int $b) {
		
		return $a + $b;

	}

	echo addNumbers(5, 10);

?>

We are using the return statement here to let the function return a value.

We can also set default parameters within a function. Here we set the default parameter value of 20 for $b. If no value is passed into this argument then the default value of 20 will be applied.

	

<?php declare(strict_types=1);		// The strict declaration

	function addNumbers(int $a, int $b = 20) {
		
		return $a + $b;

	}

	echo addNumbers(5);

?>

Arrays

An array can be created using the array() language construct. It takes a number of comma-separated key/value pairs as arguments.

	

<?php

	$array = array(
		"foo" => "bar",
		"bar" => "foo",
	);

?>

Include & Require

It is possible to share code from one PHP file into another PHP file using the require or include statements.

  • require will produce a fatal error (E_COMPILE_ERROR) and the script will stop executing.
  • include will produce a warning (E_WARNING) but the script will continue to execute.

Building complex applications using PHP, such as frameworks or a CMS (Content Management System), it's recommended to use the require statement for security and integrity of your application.

Using either statement allows our code to be shared and reduces repetition.

	

<?php

	include 'header.php';

	require 'footer.php';

?>

As an example, we can create a navigation menu within a file named navigation.php.

	

<nav>
	<ul class="nav">
		<li class="nav__item"><a href="index.php">Home</a></li>
		<li class="nav__item"><a href="about.php">About</a></li>
		<li class="nav__item"><a href="contact.php">Contact</a></li>
	</ul>
</nav>

The require statement can then be used to include our navigation.php file in each of our pages.

	

<?php

	require 'navigation.php';

?>

PHP Sessions

Session variables, also known as sessions, allow us to store information that can be used across multiple pages, such as the username once a user has logged into the site. By default, session variables last until the user closes the browser.

Most sessions set a user-key on the user's computer. When a session is opened on another page, it scans the computer for a user-key. If there's a match, the session is accessed.

For permanent storage, use a database.

A session is started with the session_start() function. We can set session variables using the PHP global variable of $_SESSION.

	

<?php

	// Start the session
	session_start();

	// Set the session variable(s)
	$_SESSION["username"] = "codebox";

?>

The value stored in a session variable, such as on a separate page, can be displayed like so:

	

<?php

	echo "Username: " . $_SESSION["username"] . ".";

?>

We can display all the session variables for a user session using the following code:

	

<?php

	print_r($_SESSION);

?>

Remove/Delete PHP Sessions

To remove all global session variables, use the session_unset() and session_destroy() functions.

	

<?php

	// Start the session
	session_start();

	// Set the session variable(s)
	$_SESSION["username"] = "codebox";

	// Remove all session variables
	session_unset();

	// Destroy the session
	session_destroy();

?>

Forms & PHP

We can use the PHP superglobals of $_GET and $_POST to get data from a form. We can begin by writing a simple HTML form.

This form will submit to the same page as we have omitted the action attribute from the HTML <form> tag.

	

<form method="GET">
	<label for="name">Name:</label>
	<input type="text" name="name" id="name">

	<button type="submit">Submit</button>
</form>

Should we want to open a new page on the form submission, add the action attribute to the HTML <form> tag.

	

<form action="welcome.php" method="GET">
	<label for="name">Name:</label>
	<input type="text" name="name" id="name">

	<button type="submit">Submit</button>
</form>

When the form is submitted we can get the value of the input by using the $_GET PHP superglobal.

	

<?php

	echo $_GET["name"];

?>

Getting Started with Vue.js

What is Vue.js?

Vue.js is an open-source JavaScript framework for building user interfaces and single-page applications (SPA). It was created by Evan You and was released in February 2014.

Vue.js does not support IE8 or below because it uses ECMAScript 5 (ES5) features that are un-shimmable in IE8.

Install Vue.js using CDN

For prototyping or learning purposes, you can use the latest CDN version link.

    

<script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>

For production, linking to a specific version number and build will avoid unexpected breakage from newer versions.

    

<script src="https://cdn.jsdelivr.net/npm/vue@2.6.11"></script>

A simple HTML page can be created to demonstrate the Vue.js application working.

    

<!DOCTYPE html>
<html lang="en-GB">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <title>Vue App</title>

    <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
</head>
<body>

<div id="app">
    {{ message }}
</div>

<script>
var app = new Vue({ 
    el: '#app',
    data: {
        message: 'Hello World'
    }
});
</script>

</body>
</html>

Install Vue.js using the Vue CLI tool

Vue CLI requires Node.js version 8.9 or above (8.11.0+ recommended). Multiple Node versions can be managed using NVM (Node Version Manager).

The package name changed from vue-cli to @vue/cli.

To install, NPM or Yarn can be used:

bash -- 85x24

    $ npm install @vue/cli -g
    # OR
    $ yarn global add @vue/cli

After installation, you will have access to the vue binary in your command line. You can verify the installation by simply checking the version installed:

bash -- 85x24

    $ npm vue --version

To upgrade the version installed:

bash -- 85x24

    $ npm update @vue/cli -g
    # OR
    $ yarn global upgrade @vue/cli --latest

To create a new project, use the following terminal command:

bash -- 85x24

    $ vue create project-name-here

Once created, open your terminal and cd into the project folder. NPM can then be used to run a server and view the Vue.js app.

bash -- 85x24

    $ cd project-name-here
    $ npm run serve

Main App Component

When using the @vue/cli tool, we start with the main top level component. Vue CLI already generates a main component that can be found in the src directory and named App.vue.

Here is the full code for the main App.vue component:

    

<template>
    <div id="app">
        <img alt="Vue logo" src="./assets/logo.png">
        <HelloWorld msg="Welcome to Your Vue.js App"/>
    </div>
</template>

<script>
import HelloWorld from './components/HelloWorld.vue'

export default {
    name: 'App',
    components: {
        HelloWorld
    }
}
</script>

<style>
#app {
    font-family: Avenir, Helvetica, Arial, sans-serif;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
    text-align: center;
    color: #2c3e50;
    margin-top: 60px;
}
</style>

Here you can see we're passing in a value of "Welcome to Your Vue.js App" into the msg property within the HelloWorld.vue component.

The file structure when creating a Vue application using Vue CLI will look something like this:

A component file consists of three parts; template, component class/scripts and styles section. The template area is the visual part of the component. Behaviour, events and data storage for the template are handled by the class. The style section serves to further improve the appearance of the template.

Importing Components

To utilise components that have been created, we need to import them into our main component. Inside of src/App.vue you will see an import statement used to currently import the HelloWorld.vue component.

The script section within the App.vue main component will look like this:

    

<script>
import HelloWorld from './components/HelloWorld.vue'

export default {
    name: 'App',
    components: {
        HelloWorld
    }
}
</script>

Install Vue.js with NPM

Node Package Manager (NPM) can be used to install Vue.js. Begin by creating a new directory where the Vue.js application will be built and setup a package.json file. Open your terminal and run the following commands:

bash -- 85x24

    $ mkdir vue-app
    $ cd vue-app

    $ npm init -y

We now have a directory named vue-app and a generated package.json file.

NPM is the recommended installation method when building large scale applications with Vue.js and pairs nicely with module bundlers such as Webpack.

We now want to install the latest stable build of Vue.js as a dependency:

bash -- 85x24

    $ npm install vue --save

The following packages may need to be installed without vue-cli:

  • vue
  • vue-loader - for compiling and bundling .vue single file components
  • vue-template-compiler - used to pre-compile Vue.js 2.0 templates into render functions
  • webpack
  • webpack-cli
  • babel-loader - for transpiling JavaScript
  • @babel/core
  • @babel/preset-env
  • file-loader - for image assets
  • style-loader - for injecting styles into the DOM at runtime
  • css-loader - for loading modules referenced in CSS files like images and fonts

You can of course use vue-cli to generate the scaffolding for your application as this uses Webpack under-the-hood and adstracts away the configuration needed with a sensible default. It is recommended that beginners do not use vue-cli as explained within the Vue.js documentation.

Let's install the Vue.js packages we need:

bash -- 85x24

    $ npm install vue-loader vue-template-compiler --save

Webpack will need to be installed as our build tool as a devDependency.

bash -- 85x24

    $ npm install webpack webpack-cli --save-dev

A build script can be added into the package.json file to run the Webpack build tool in production mode.

    

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "build": "webpack --mode production"
},

A src directory will need to be created and within the directory an index.js file. Without the index.js file, the build will fail. If the build is successful, a dist directory containing a main.js file will be generated.

Let's now install the Babel packages as devDependencies using NPM:

bash -- 85x24

    $ npm install babel-loader @babel/core @babel/preset-env --save-dev

A .babelrc configuration file needs to be created in the root of your project folder.

This maybe a hidden file once created, so use the keyboard shortcut of: COMMAND + SHIFT + . (DOT) to make this file visible.

    

module.exports = {
    presets: ['@babel/preset-env']
}

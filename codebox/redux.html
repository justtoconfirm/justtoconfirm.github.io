Redux

Redux is a state container for JavaScript applications. Normally, with React, you manage state at a component level and pass state around via props. With Redux, the entire state of your application is managed in one immutable object.

Redux was originally created by Dan Abramov and Andrew Clark. The documentation can be found here.

  • Access or update any part of the state from a Redux-connected component.
  • Keep track of changes with Redux DevTools. Any action or state change is tracked and easy to follow.

Redux does require a lot of initial boilerplate to setup. A smaller application may also not need React and may instead benefit from using the Context API for global state needs. You may want to start building your app with the Context API and latet convert to using Redux as your app grows in size for better maintanence and organisation.

Redux has a number of key terms to be familiar with:

  • Actions
  • Reducers
  • Store
  • Dispatch
  • Connect

There is an introductory guide to React, should you need it.

You may want to create the directories for Redux using your terminal:

bash -- 85x24

    $ mkdir actions components pages reducers

Both the pages and components directories are created for React, actions and reducers for Redux.

The file structure for a React application using Redux may look like this:

    

src /
    actions /
    components /
    pages /
    reducers /
    App.js
    index.js

Getting Started

Redux requires a few dependencies:

  • Redux - Core library
  • React Redux - React bindings for Redux
  • Redux DevTools Extension - Connects Redux app to Redux DevTools

The Redux & React Redux packages can be installed using NPM (Node Package Manager):

bash -- 85x24

    $ npm install redux --save
    $ npm install react-redux --save

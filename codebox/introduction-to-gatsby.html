Introduction to Gatsby

What is Gatsby?

Gastby is an open-source React-based framework for creating websites and web applications. Components are written in React and rendered at build time to the DOM (Document Object Model) as static HTML, CSS & JavaScript files. By default, Gatsby builds a PWA (Progressive Web App).

The website for Gatsby features useful documentation.

Gastby renders static content from sources including Markdown (md), CSV (Comma Separated Values) and from CMS (Content Management System) platforms such as WordPress and Drupal.

You will require plugins to extend the core functionality of Gatsby.

Building with Components

Broadly defined, a component is a building block for your application. It is a self-contained piece of code that describes a section of the UI (User Interface). Gatsby is built on React. When we talk about defining components, we're referring to React components, usually written with JSX.

Install Gatsby

Install the Gastby CLI (Command Line Interface) globally in the terminal. You will need NPM (Node Package Manager) installed. Once installed this will allow you to use the gatsby command in your terminal. You may need to set permissions using sudo to install Gatsby globally on your system.

bash -- 85x24

    $ npm install gatsby-cli -g

    # Permissions
    $ sudo npm install gatsby-cli -g

Gatsby will require Node version 12.13.0 or greater.

To create a new project, we use the following command. Here the name of our app is gatsby-app.

bash -- 85x24

    $ gatsby new gatsby-app

To start the development server:

bash -- 85x24

    $ cd gatsby-app

    $ gatsby develop

This will run the local server and the application can be viewed at http://localhost:8000/ in the browser.

Setup Gastby without creating a new project

We can setup Gatsby without creating a new project. Follow these instructions:

  • Install Node version 12.13.0 or higher.
  • If not installed, install gatsby-cli using your terminal.

Once these steps have been completed, create a directory to hold the Gatsby project and setup a package.json file using NPM (Node Package Manager). You may also want to add a .gitignore file.

bash -- 85x24

    $ npm init -y

Next, install the Gatsby and React packages:

bash -- 85x24

    $ npm install gatsby react react-dom

Now, add a src directory and a pages directory inside your project.

bash -- 85x24

    $ mkdir src
    $ cd src
    $ mkdir pages

Inside the pages directory, create an index.js file and add some React code as a starting point for the project.

    

import React from 'react'

function Home() {
    return <h1>Hello Gatsby</h1>
}

export default Home

From within the root of your project, run the gatsby develop command to start a development server.

bash -- 85x24

    $ gatsby develop

Your initial page should now be running on a localhost development server.

  • http://localhost:8000

A GraphiQL IDE will also be running:

  • http://localhost:8000/___graphql

A public directory will be created when the develop command is run from within the terminal.

Adding Plugins

To add a plugin to Gatsby, navigate to and edit the gatsby-config.js file in the root directory.

The gatsby-plugin-react-helmet plugin comes pre-installed as default.

Gastby runs the gatsby-config.js during build and implements all installed plugins.

Hot Reloading

Gatsby comes with a hot reload feature so changes made on the source files are immediately visible on the rendered website. Essentially when you're running a Gatsby development server, the files are being "watched" in the background.

Adding Pages

Any React component defined in the src/pages directory will automatically become a new page added to our Gatsby app. Create a new file named about.js in the src/pages directory. Each page is created as a JavaScript file. The new about page can be viewed at http://localhost:8000/about and the code for this page is below:

    

import React from 'react'

const About = () => (
    <>
        <h1>About</h1>
        <p>This is the about page.</p>
    <>
)

export default About

Adding Links

Gatsby has a built-in Link component that can be used to navigate between different pages within your application.

    

import React from 'react'
import { Link } from 'gatsby'

const Home = () => (
    <>
        <h1>About</h1>
        <p>This is the about page.</p>

        <Link to='/'>Home</Link>
        <Link to='/about/'>About</Link>
    <>
)

export default About

Adding a 404 page

To add a 404 error page, create a new page in the pages directory named 404.js.

Adding Styling using SCSS

Begin by installing the following packages using NPM as dependencies:

bash -- 85x24

    $ npm install sass gatsby-plugin-sass

In the root of your Gatbsy project folder, create a config file named gatsby-config.js and add the following code to add the gatsby-plugin-sass plugin to your application:

    

module.exports = {
    plugins: [
        `gatsby-plugin-sass`
    ]
}

Create your SCSS partial file(s) containing your styling within the src directory. You will the need to import your SCSS partial into the React component/page you want to apply the styling.

    

import React from 'react'
import { Link } from 'gatsby'

import './main.scss'

const Home = () => (
    <>
        <h1>About</h1>
        <p>This is the about page.</p>

        <Link to='/'>Home</Link>
        <Link to='/about/'>About</Link>
    <>
)

export default About

If the development server is running, your Gatsby application will reload and your styling will be applied.

Building the application

Once the development of the application is completed, the static code needs to be generated as HTML, CSS and JavaScript for deployment to an environment. Run the following terminal command to output the HTML, CSS & JavaScript in the public directory for hosting and create a production build.

bash -- 85x24

    $ gatsby build

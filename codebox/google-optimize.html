Google Optimize

Google Optimize allows you to test variants of a web page and see how it performs.

The Google Optimize documentation

On the free version of the Google Optimize, only five tests can be created at any one time.

A/B tests

An A/B test, sometimes also called an A/B/n test, is an experiment using two or more variants of the same web page (A and B). Variant A is the original, Variant B (through n) each contain at least one element that is modified from the original.

An example of an A/B test maybe to set a different colour of a CTA (call-to-action) button to see if this drives conversions.

  • An A/B test example of a product page, Variant A has a blue photo, Variant B has a red photo.

Redirect tests

A redirect test, also called a split URL test, is a type of A/B test that allows you to test separate web pages against each other. In a redirect test, variants are identified by a URL or path instead of by element(s) on the page.

This type of test is useful when you want to test two very different landing pages, or a redesign of a page.

An example maybe to test two different landing pages, with different URLs:

  • Original - www.example.com/landing1
  • Variant - www.example.com/landing2

Multivariate tests (MVT)

A multivariate test (MVT) tests variants of two or more elements simultaneously to see which combination creates the best outcome.

Getting Started

You will need a Google account, such as an Gmail account, to login to Google Optimize

  • Log into Optimize
  • Click on the Let's go button under the Create your first experience section
  • Add a name for your experience
  • Add a URL of the page you'd like to use for the test.
  • Set the type of experience, for example, A/B test.
  • Click the Create button.

You will need to setup the Google Optimize code snippet and add it to the <head> section of your page or use Google Tag Manager (GTM) to inject it into the page within a tag.

    

<script src="https://www.googleoptimize.com/optimize.js?id=OPT_CONTAINER_ID"></script>

The container ID is part of your Optimize snippet and can be found within the container setup. As of the start of 2020, all IDs begin with OPT-.

Within the Google Optimize dashboard, you will have an Original variant (this is the control and is the current website), and you can then create a new variant (which we will call Variant 1 for simplicity), that will be our test. Once you've completed the setup of your test you can then edit the weight of each variant as this is set to 50% of traffic for each and preview/edit the variant as required.

You will need to setup Google Analytics and link to your GA account to connect this to the Google Optimize test being created.

Edit the test variant code

Once you've setup the test variant, you can then edit the code on the page to set how you want the test to appear.

  • Click on a specific element and the Edit element button to edit the text, HTML, or run JavaScript.
  • Or, click the <> code button in the top-right corner of the dashboard to edit the CSS or the page and/or edit the global JavaScript of the page.
  • Remember to click Save after your code changes.

In the dashboard, click the Edit button to do this.

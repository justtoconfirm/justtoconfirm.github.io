Styleguide Driven Development (SGDD)

Styleguide Driven Development (SGDD) is a practice that encourages the separation of UX, Design and Frontend from Backend concerns. The aim of a style guide is to provide a toolbox of "ready-to-go" UI (User Interface) components that can be used for any given application.

A frequently used analogy to describe SGDD is "building with Lego". Instead of seeing a web page as one solid chunk, you should aim to see it as many independent parts (i.e. Lego bricks).

The word "Styleguide" can be confusing at first. Typically, it is easy to think of "brand guidelines" as styles. Typically a styleguide may become a "living styleguide" where we not only document the different elements of our products, but we generate their assets with code as well. We call this "a source of truth".

To summarise, there are two main types of style guide:

  • A static design style guide
  • A dynamic front end style guide

A living style guide is a system that allows the components within the style guide to remain in sync with the components in the site, without having to update the code in both places.

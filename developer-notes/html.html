HTML

Simple HTML Template

Here is a simple HTML template. Save the file as index.html when creating a new project.

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />

    <title>My Page</title>
</head>
<body>

<!-- Page content here -->

</body>
</html>

Tags & Elements

HTML tag is just the opening or closing entity in HTML. For example, <p> is an opening tag and </p> is the closing tag. These are examples of HTML tags. Tags are used to mark up the start and end of an HTML element.

HTML elements contain opening, closing tags and often content. In HTML an element represents some kind of structure or semantics.

This example simply uses a paragraph tag to simply demonstrate this.

        

<!-- Opening HTML tag -->
<p>

<!-- Closing HTML tag -->
</p>

<!-- HTML element, it's all together with content -->
<p>This is content</p>

You can of course get HTML tags which are self-closing and are elements too, such as the <img> tag.

        

<img src="#" />

When we talk about attributes, they are used to define a property of an element and consist of attribute properties and value pairs. They are added to the element's start (opening) tag and this tag can contain any number of attributes.

The most popular misuse of the term "tag" is referring to the alt attribute on images as "alt tags". Alt is an attribute, not a tag.

The example below shows an <img> element with a "src" and "alt" attribute.

        

<img src="#" alt="Empty image example" />

We can use attributes to add classes and id's to HTML elements for CSS and JavaScript.

        

<p class="bold">This is bold text styled using CSS</p>

And this is the CSS:

        

.bold {
    font-weight: bold;
}

DOM (Document Object Model)

The DOM has a tree-like structure. Each element, attribute and piece of text in the markup language becomes a DOM node in the tree structure. The nodes are defined by their relationship to other DOM nodes. Some elements are parents of child nodes and child nodes have siblings.

Let's assume the following HTML code:

        

<p>
    Let's use:
    <span>Cascading</span>
    <span>Style</span>
    <span>Sheets</span>
</p>

In the DOM, the node corresponding to our <p> element is a parent. Its children are a text code and the nodes corresponding to our <span> elements. The SPAN nodes are also parents with text nodes as their children.

        

P
├─ "Let's use:"
├─ SPAN
|  └─ "Cascading"
├─ SPAN
|  └─ "Style"
└─ SPAN
   └─ "Sheets"

The browser renders the DOM tree then outputs it in the browser like so:

        

Let's use: Cascading Style Sheets

Detailed notes relating to the DOM (Document Object Model) can be found in the JavaScript notes.

Comments

The comment tag is used to insert comments into the HTML source code. Comments are not displayed in the browser. Comments can be used to explain your code or used for debugging.

        

<!-- This is a comment in HTML -->

Define a canonical page

If you have a single page accessible by multiple URLs, or a different page with similar content (for example, a page with both a mobile and desktop version), Google can see these as duplicate versions of the same page. Google will choose one URL as the canonical version and crawl that. All other URLs will be considered as duplicate URLs and crawled less often.

        

<link rel="canonical" href="https://www.crunch.co.uk" />

<!DOCTYPE> Declaration

The <!DOCTYPE> declaration must be the very first thing in your HTML document, before the <html> tag. This is not an HTML tag, but an instruction to the web browser about the version of HTML being used.

        

<!DOCTYPE html>

Icons

You can create a favicon.ico file which contains a 16x16 and 32x32 image then declare it within your HTML code. This will display an icon next to your page title within the browser tab. Other file formats such as .gif or .jpeg can be used, but .ico is recommended.

        

<link rel="shortcut icon" href="/crunch-core/images/favicon.ico" />

On an Apple iOS device, you may want users to be able to add your web application or webpage link to the Home screen. These links, represented by an icon, are called Web Clips. Often these icons are in PNG (.png) format and are named apple-touch-icon.png

        

<link rel="apple-touch-icon" href="/crunch-core/images/apple-touch-icon.png" />
<link rel="apple-touch-icon" sizes="72x72" href="/crunch-core/images/apple-touch-icon-72x72.png" />
<link rel="apple-touch-icon" sizes="144x144" href="/crunch-core/images/apple-touch-icon-144x144.png" />

So you may want to use different icons for different devices:

        

<link rel="apple-touch-icon" href="touch-icon-iphone.png" />
<link rel="apple-touch-icon" sizes="152x152" href="touch-icon-ipad.png" />
<link rel="apple-touch-icon" sizes="180x180" href="touch-icon-iphone-retina.png" />
<link rel="apple-touch-icon" sizes="167x167" href="touch-icon-ipad-retina.png" />

Title

The <title> element defines the title of the document and is a requirement in all HTML documents.

        

<title>Page Title</title>

The HTML <p> element defines a paragraph.

        

<p>This is a paragraph.</p>

The "rel" attribute

The "rel" attribute stands for relationship.

Abbreviations

The <abbr> tag defines an abbreviation or an acronym. They are both shortened versions of something else.

        

The <abbr title="World Health Organisation">WHO</abbr> was founded in 1948.

HTML Entities

An HTML entity is a piece of text ("string") that begins with an ampersand (&) and ends with a semicolon (;). Entities are frequently used to display reserved characters which would otherwise be interpreted as HTML code, and invisible characters, such as non-breaking spaces. You can use them in place of other characters that are difficult to type with a standard keyboard.

Some special characters are reserved for use in HTML, meaning that your browser will parse them as HTML code. For example, if you use the less-then (<) sign, the browser interprets any text that follows as a tag.

        

& - &amp; - Interpreted as the beginning of an entity or character reference.
< - &lt; - Interpreted as the beginning of a tag.
> - &gt; - Interpreted as the ending of a tag.
" - &quot; - Interpreted as the beginning and end of an attribute's value.

I often use this reference for HTML special characters.

Links

Links are found in nearly all web pages. Links allow users to click their way from page to page. HTML links are hyperlinks. You can click on a link and jump to another document.

In HTML, links are defined with the <a> tag.

        

<a href="url">link text</a>

The href attribute specifies the destination address/URL of the link. The link text is the visible part/label on the page.

        

<a href="https://www.google.co.uk">Go to Google</a>

Opening links in new windows & tabs

When opening links in new browser windows/tabs, we should apply the 'rel' attribute as a precaution against reverse tabnabbing.

When using the target attribute in this way, the page we are linking to gains partial access to the linking page via the window.opener object. The newly opened tab can be used to, say, change the window.opener.location to an unsecured page or execute JavaScript on the opener-page. This is referred to as tabnabbing.

        

<a href="#" target="_blank" rel="noopener noreferrer">Link</a>

So, we add this to our outgoing links:

        

rel="noopener"

For Firefox (FF), "noopener" is not supported, so we need to modify this slightly:

        

rel="noopener noreferrer"

<detail>

The <details> element can be used to create an interactive widget that the user can open and close. Any type of content can be put inside the <details> element. The content of the element should not be visible unless the open attribute is set.

The element is supported in most modern browsers with the exception being Internet Explorer (IE) and Edge.

        

<detail>
    <p>This is an excellent hiding place.</p>
    <p>I can place content here.</p>
</detail>

<button>

<button> elements are much easier to style than <input> elements and you can add inner HTML content or use ::after and ::before pseudo-elements to achieve complext rendering, while <input> only accepts a text value attribute.

If your buttons are not to submit form data to a server, be sure to set their type attribute to button, otherwise they will try to submit form data and load a nonexistent response. If you need to submit data, set the type to be 'submit'.

        

<button type="button">Button</button>
<button type="submit">Button</button>

It is not recommended, but if you need to use an element in-place of a button, set the role attribute to be "button" for accessibility standards to be met.

        

<button type="button">Button</button>

<div role="button">>Fake Button</div>

You may want to add a name attribute to a <button> element also. This is submitted with the form data.

        

<button name="button" type="button">Button</button>

CSS Variables

CSS Variables (also known as Custom Properties) brings the power of variables to CSS resulting in less repetition, better readability and more flexibility. Unline variables from CSS preprocessors, CSS variables are part of the DOM.

Let's look at how we would write CSS normally:

        

.heading {
    color: #ff6f69;
}

.quote {
    color: #ff6f69;
}

In the example above it's much better to create a variable for the colour than repeating it.

        

:root {
    --red: #ff6f69;
}

.heading {
    color: var(--red);
}

.quote {
    color: var(--red);
}

Not only will this make your code easier to read but provides more flexibility as well, should you want to change this colour.

CSS variables have the advantage that they don't require any transpiling to work as their native to the browser, so no need to setup SASS or LESS. This is an advantage because CSS variables have access to the DOM whereas LESS or SASS variables are compiled down to regular CSS. Using preprocessors such as SASS and LESS can extra complexity to our processes.

Another advantage of living in the DOM is that you can access the variables using JavaScript and even update them. This is ideal if you want to provide your users with the ability to change your website, such as adjusting the font size.

This method of mixin CSS variables with JavaScript is ideal should you want to create themes that can be switched.

Browser Support

At the time of writing, 77 percent of global website traffic supports CSS variables. Internet Explorer (IE) is the main browser that does not provide support as this page from caniuse details.

You may need to provide a fallback for IE should you want to use CSS variables.

Declare a Variable

To declare a variable you must first decide what scope the variable should live in. If you want it to be available globally, define it on the :root psuedo class. It matches the root element in your document tree, usually the <html> element. As variables are inherited, this will make your variable available throughout your application as all your DOM elements are descendants of the <html> element.

The standard cascade rules also apply to the CSS variables. So, if a custom property is declared multiple times, the lowermost definition in the CSS file overwrites the ones above it.

You declare a variable in the same way as you'd set any CSS property. The variable, however, must start with two dashes.

        

:root {
    --main-color: #ff6f69;
}

To access a variable you need to use the var() function and pass in the name of the variable you want to use as the parameter.

        

.element {
    color: var(--main-color);
}

Local Variables

You can also create local variables which are accessible only to the element it's declared at and to its children. This makes sense if you know that a variable will only be used in a specific part(s) of your application.

For example, you may have an alert box that uses a colour that isn't being used elsewhere within your application. In that case, it may make sense to avoid placing it in the global scope.

        

.alert {
    --alert-color: #ff6f69;
}

This variable can now be used by its children.

        

.alert p {
    color: var(--alert-color);
    border: 1px solid var(--alert-color);
}

If you try to use the alert-color variable elsewhere in your application, it simply won't work. The browser will just ignore that line of CSS.

Media Queries with CSS Variables

CSS variables can be used within media queries where the variable values can change based upon the width of the screen, as this example demonstrates:

        

:root {
    --main-font-size: 16px;
}

@media all and (max-width: 600px) {
    :root {
        --main-font-size: 12px;
    }
}

With these simple few lines of code you can update the main font size across your entire application when viewed on small screens.

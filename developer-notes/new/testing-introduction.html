Testing: Introduction

Test-driven development (TDD)

Test-driven development (TDD) is an approach to development that consists of writing tests, followed by writing production code, and refactoring as needed. The tests are written to fail initially before the developer then writes the code to fulfill the requirements of the test so that they pass.

TDD is a design process and not a testing process. TDD is a robust way of designing software components ("units") interactively so that their behaviour is specified through unit tests.

Unit Testing

Unit testing is a level of software testing where individual units, or components, are tested. JavaScript unit tests are usually run in the browser or on the frontend.

Unit tests cover blocks of code to ensure that they run without problems. A tested unit can be a function, a module or a class. Unit tests should be isolated and independent of each other. Unit tests can make sure that individual parts of your application work as expected by finding problems early and avoiding regression.

Jest is a JavaScript testing framework that can be used to write unit tests.

End to End tests (E2E)

E2E tests are always run inside a browser, or browser-like environment. This can be an actual browser that opens and the tests are run inside or a headless browser environment (a browser running without the user interface). E2E tests are used to emulate an actual user within our application. The tests will simulate behavior like scrolling, clicking and typing to check the application works well from the point of view of an actual user.

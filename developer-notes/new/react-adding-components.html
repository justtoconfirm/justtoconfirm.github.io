React: Adding Components

So far, we've created a single component - the Button component. Almost everything in React consists of components. Most React applications have many small components and everything loads into the main App component.

We now want to add a further React component to our application. Within the src directory, begin by creating a new empty JavaScript file named App.js. We are going to import this new component into our index.js file.

Open the index.js file and update the code so it appears like the example below:

        

import React from 'react'
import ReactDOM from 'react-dom'

import App from './App'

ReactDOM.render(
    <App />,
    document.getElementById('app')
)

Next, open the App.js component and add the following code. This will import the Button component we created earlier and render it within our app once again:

        

import React from 'react'
import Button from './Button/Button'

const App = () => {
    return (
        <Button>Sign in</Button>
    )
}

export default App

Remember to rebuild your application once the changes have been made to update the app.

Let's attempt to render two buttons within our App component.

        

import React from 'react'
import Button from './Button/Button'

const App = () => {
    return (
        <Button>Sign in</Button>
        <Button>Register</Button>
    )
}

export default App

When you rebuild the application, you may find that the terminal throws and error. This is because our component can only return single elements/components. The error may appear like so:

        

Adjacent JSX elements must be wrapped in an enclosing tag. Did you want a JSX fragment <>...</>?

To fix this issue, we simply need to wrap our two Button components with an element.

We can begin by wrapping our two Button components using a <div> or perhaps <span> element. This would fix our error, however a further element would be added to the DOM (Document Object Model) when the application renders.

        

import React from 'react'
import Button from './Button/Button'

const App = () => {
    return (
        <div>
            <Button>Sign in</Button>
            <Button>Register</Button>
        </div>
    )
}

export default App

An alternative would be to use React Fragments. We can either wrap our two Button components with <React.Fragment> or we can use the shorthand alternative of <>. This avoids the need to render the wrapper element in the DOM as neither one of these options will render in our application but will fix the error in the terminal we encountered earlier.

        

import React from 'react'
import Button from './Button/Button'

const App = () => {
    return (
        <React.Fragment>
            <Button>Sign in</Button>
            <Button>Register</Button>
        </React.Fragment>
    )
}

export default App

We are now reusing the same Button component to render two buttons within our application.

The file structure should look something like this:

        

.babelrc
node_modules/
webpack.config.js
package.json
package-lock.json
dist/
    index.html
    main.js
src/
    App.js
    index.js
    Button/
        Button.js

Building a Table Component

We can now build a table component to demonstrate how to display data on a page within our React application. Open a new file in the src directory named Table.js and create a simple table with hard-coded values.

        

import React from 'react'

const Table = () => {
    return (
        <table>
            <thead>
                <tr>
                    <th>Name</th>
                    <th>Job</th>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td>Darren</td>
                    <td>Web Developer</td>
                </tr>
                <tr>
                    <td>Dave</td>
                    <td>Web Developer</td>
                </tr>
                <tr>
                    <td>Dan</td>
                    <td>Web Developer</td>
                </tr>
            </tbody>
        </table>
    )
}

export default Table

Next, we open the App.js component to import the Table component so it can be rendered on the page. You will need to build the application once again once these changes have been made.

        

import React from 'react'
import Button from './Button/Button'
import Table from './Table/Table'

const App = () => {
    return (
        <React.Fragment>
            <Button>Sign in</Button>
            <Button>Register</Button>
            <Table />
        </React.Fragment>
    )
}

export default App

The file structure within the project should look something like this:

        

.babelrc
node_modules/
webpack.config.js
package.json
package-lock.json
dist/
    index.html
    main.js
src/
    App.js
    index.js
    Button/
        Button.js
    Table/
        Table.js

Right now, we have a Table component but the data in the table is being hard-coded. One of the big features of React is how it handles data. It handles data with properties, referred to as props, and with state.

JavaScript: Comments

Comments are annotations in the source code of a program that are ignored by the interpreter and therefore have no effect on the actual output of the code. Comments are helpful in explaining the intent of what your code is, or should, be doing. Commenting your code early will reinforce good programming habits throughout your code to avoid issues later.

Comment Syntax

Let's take a look at the two different types of JavaScript comment syntax.

Single-line comments are written with two forward slashes (//)

        

// This is a comment

All characters immediately following the // syntax until the end of the line will be ignored by JavaScript.

Block comments, sometimes referred to as multi-line comments, are written with opening tags and closing tags. If you know CSS, then you're already familiar with block-level comments.

        

/* This is
a block or multi-line
comment */

Everything between the opening and closing tags in the code block will be ignored.

You may also sometimes see a slightly modified version of the block comment syntax, which starts with /** and includes asterisks through the left side of the comment block.

        

/**
 * Initialize constant with an array of strings.
 * Loop through each item in the arry and print
 * it to the console.
 */

Both single-line and multi-line comments are written above the code thay are designated to explain, as demonstrated in this example:

        

// Print "Hello, World!" to the console
console.log('Hello, World!')

When writing comments, indent them at the same level as the code immediately below them.

        

// Initialize a function
function alphabetizeOceans() {
    // Define oceans variables as a list of strings
    const oceans = ['Pacific', 'Atlantic', 'Indian', 'Antarctic', 'Arctic']

    // Print alphabetized array to the console
    console.log(oceans.sort())
}

Note that comments are just as much a part of the code as the program itself. Remember to maintain and update comments regularly along with everything else.

Inline Comments

Single-line comments are referred to as inline comments when they appear at the end of a line of code. Inline comments are used for quick annotation on small, specific snippets of content. Since the comment should only relate to the exact line it's written on, it is the most obvious type of comment.

        

let x = 99          // assign numerical value to x
let y = x + 2       // assign the sum of x + 2 to y

JSHint

What is a linter?

A static program analysis tool, lint, or a linter, is a tool that analyzes source code to flag programming errors, bugs, syntax errors and issues. The term originates from a Unix utility that examined C language source code.

What is JSHint?

JSHint is a static code analysis tool used to check JavaScript source code for errors and potential issues. Errors can include typos and syntax errors within your code. By analysing our code, this helps to reduce issues or bugs reaching production. JSHint should ideally be combined with unit/functional testing and code reviews.

Here is the documentation for JSHint. the CLI documentation is also available.

JSHint was created in 2011 by Anton Kovalyov as a fork of the JSLint project.

JSHint is used by the following organisation:

  • Mozilla
  • Wikipedia
  • Facebook
  • Medium
  • Spotify

Other tools that can be used include:

  • ESLint
  • StandardJS

Install JSHint

Open the project folder, and your terminal, and use NPM (Node Package Manager) to install JSHint as a devDependency.

        

$ npm install jshint --save-dev

Once installed, as a testing script to your package.json file to run JSHint. This will execute over the entire project.

        

"test": "jshint",

Should we want to run JSHint over a specific directory or file, append this to the end of the test script like so:

        

"test": "jshint WebContent/src/js",

You may want to exclude certain files or directories from being analyzed. Use the --exclude-path flag to add the file or directory you wish to exclude. In this example, the .gitignore file is being referenced as this contains the node_modules directory that we want to always exclude.

        

"test": "jshint WebContent/src/js --exclude-path .gitignore",

A .jshintignore file can be created in the project root should you want to ignore files or directories also.

Running the test script

To run the test script, simply open the terminal and enter the following command:

        

$ npm run test

Any issues within your JavaScript code will be output in the terminal.

Configuration

JSHint works without any config, but should you want to configure the tool, a .jshintrc config file can be created in the root of your project folder.

The config file may be needed to allow ES6 syntax to be ignored when analyzing your JavaScript code. If this is not set, JSHint will suggest issues with any ES6 syntax.

To allow ES6 syntax to be supported, open the .jshintrc config file and add the following property:

        

{
    "esversion": 6
}

JavaScript: Functions

A function is a block of code that performs an action, returns a value or specific task. Functions are a series of statements that have been grouped together. This block of code can then be reused without the need to repeat the same set of statements. Being reusable, functions can therefore make your programs more modular and more efficient.

Declare a Function

Functions are declared, or defined, with the function keyword. Below is the syntax for a function in JavaScript.

        

function functionName() {
    // Code to be executed here
}

The declaration begins with the function keyword followed by the name - often called the identifier - of the function. Function names follow the same rules as variables - they can contain letters, numbers, underscores and dollar signs. The name of a function is often written in camelCase. The name of the function iis followed by a set of parenthesis which can be used for optional parameters. The code to be executed is contained within the curly brackets.

When naming a function the name should describe the task or action of the function.

In this simple example, we'll make a function declaration to print a greeting statement to the console.

        

function sayGreeting() {
    console.log("Hello, World");
}

Invoke a Function

Here we have the code to print Hello, World to the console contained inside the sayGreeting() function. Nothing will happen and no code will execute until we invoke, or call, the function. You can invoke a function by writing the name of the function followed by the parenthesis.

        

// Invoke a function
sayGreeting();

Now we have our greeting code within our sayGreeting() function, you can reuse the function as many times as you want to. Using parameters, we can also make the code more dynamic.

Function Parameters

We have created a very simple function that prints Hello, World to the console. Using parameters, we can add additional functionality that will make the code more flexible. Parameters behave as local variables inside a function and allow values to be passed into a function.

When a user logs into an application we may want to greet them by name instead of just saying "Hello, World". We'll add a parameter into our function, called name, to represent the name of the person being greeted.

        

// Initialize a custom greeting function
function sayGreeting(name) {
    console.log(`Hello, ${name}`);
}

Instead of a static string consisting of Hello, World, we now have a template literal string containing our parameter, which is now behaving as a local variable.

This function can also be re-written without the template literal string:

        

// Initialize a custom greeting function
function sayGreeting(name) {
    console.log("Hello, " + name);
}

Assuming our user name is "Dan", we'll call function and place the name as the argument. The argument is the value that gets passed into the function, in this case the argument is "Dan".

        

// Invoke the function with "Dan" as the argument
sayGreeting("Dan");

When the function is invoked the greeting should be: "Hello, Dan". In a real world example, the function name would pull the username from a database instead of directly supplying the name as an argument value.

Variables can be declared inside of functions. These variables are known as local variables and will only exist inside the scope of their own function block.

Default parameters

Functions can be initialized with default parameters which will only be used if an argument is not invoked through the function. In ES5, the function would look like this:

        

var func = function(a, b) {
    b = b === undefined ? 2 : b
    return a + b
}

In ES6:

        

let func = (a, b = 2) => {
    return a + b
}

We can call the function:

        

func(10)            // returns 12
func(10, 5)         // returns 15

The return keyword

When the return keyword is used, the function ceases to execute and the value of the expression is returned. In this example, the value will be displayed in the console, it is however not the same as console.log() used to print to the console. Invoking the function will output the value exactly where the function was invoked. This value can be used immediately or placed into a variable.

        

// Initialize the function
function add(x, y) {
    return x + y;
}

// Invoke the function to find the sum
add(9, 7);          // 16

Function Expressions

Previously a function declaration was used to get the sum of two numbers and return that value. We can also create a function expression by assigning a function to a variable. Using our same add function as an example, we can directly apply the returned value to a variable, in this example, sum.

        

// Assign the add function to the sum constant (variable)
const sum = function add(x, y) {
    return x + y;
}

// Invoke function to find the sum using the variable
sum(20, 5);         // 25

Now that the sum constant is a function, we can make this expression more concise by turning it into an anonymous function (an unnamed function). Currently, our function has the name add, but with function expressions, it is not necessary and is usually omitted.

        

// Assign function to the sum constant (variable)
const sum = function(x, y) {
    return x + y;
}

// Invoke function to find the sum
sum(20, 5);         // 25

A named function expression might be used to aid in debugging, but is usually omitted.

Arrow Functions

So far we've learnt how to define functions using the function keyword, however, there is a newer, more concise method of defining a function known as an arrow function expression (or, fat arrow). Arrow functions, as they are commonly known, are represented by an equals sign ( = ) followed by a 'greater than' ( > ) sign.

Arrow functions are a feature of the ECMAScript 6 (ES6) syntax.

Arrow functions are always anonymous functions and a type of function expression. We can create a simple example to find the total of two numbers.

        

// Define the multiply function
const multiply = (x, y) => {
    return x * y
}

// Invoke the function to find the total
multiply(30, 4)         // 120

Instead of writing out the function keyword, we use the => arrow to indicate a function. Otherwise, it works similar to a regular function expression with so differences.

In the case of only one parameter, the parenthesis can be excluded.

        

// Define the square function
const square = x => {
    return x * x
}

// Invoke the function to find the total
square(8)         // 64

In the case of no parameters, an empty set of parenthesis () is required in arrow functions.

With examples that only consist of a return statement, arrow functions allow the syntax to be reduced even further. If the function is only a single line return, as with the example above, both the curly brackets and the return statement can be omitted, as seen in the example below.

        

// Define the square function
const square = x => x * x

// Invoke the function to find the total
square(8)         // 64

All three syntax used to create a function results in the same output. It is generally a matter of preference or coding standards that determines the decision on how you will structure your own functions.

JavaScript: This, Bind, Call & Apply

Global

In the global context, the keyword of this refers to the global object. When you're working in a browser, the global context would be window. When working with Node.js, the global context is global.

If you log the value of this without any other code, you'll see what object this refers to.

        

console.log(this);      // Window {postMessage: ƒ, blur: ƒ, focus: ƒ, close: ƒ, parent: Window, …}

You can see that the this refers to window. This is the global object of the a browser.

A top-level function will still retain the this reference of the global object. When using a simple function that is not associated with any object, like this:

        

function printThis() {
    console.log(this);
}

printThis();

The function will still refer to the window object. However, when using strict mode, the context of this within a function on the global context will be undefined.

        

'use strict';

function printThis() {
    console.log(this);
}

printThis();        // Undefined

Generally it is safer to use strict mode to reduce the probability of this having an unexpected scope. Rarely will you want to refer to the window object using this.

The Object Method

A method is a function on an object, or a task that an object can perform. A method uses this to refer to the properties of the object.

        

const america = {
    name: 'The United States of America',
    yearFounded: 1776,

    describe() {
        console.log(`${this.name} was founded in ${this.yearFounded}.`)
    },
}

america.describe();

The function will log the following to the console:

        

"The United States of America was founded in 1776."

In this example, this is the same as america. In a nested object, this refers to the current object.

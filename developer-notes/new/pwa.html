PWA - Progressive Web Applications

Progressive Web Applications (PWA) are a type of application delivered through the web and built using common web technologies including HTML, CSS and JavaScript. They are intended to work on any platform that uses a standards-compliant browser. PWAs often use the newest JavaScript features making them feel and function like native Android or iOS apps, therefore building a PWA is similar to building a web app, but from the user's perspective, it's similar to a native app.

As a specific term, websites can be tested against the following baseline criteria to qualify as a PWA:

  • It must run under HTTPS
  • It must include a Web App Manifest
  • It must implement a service worker

An advantage of a PWA over a native mobile app is that they consume less data. While native apps reside in the user's phone, PWAs are only accessed when users go to the website - when data is being used.

Progressive Web Apps are installed and live on the user's home screen without the need for an app store. They offer an immersive full screen experience with help from a web app manifest file and can re-engage users with push notifications.

Here are a few of the other benefits of PWAs:

  • SEOs promote PWAs making it easier for users to find them through Google or other search engines.
  • PWAs load directly onto a smartphone user screen without needing an app store to download.
  • PWAs are HTML5 based, they instantly load on any operating system and any device.
  • Easy to update, new versions of the app can be pushed directly to users.

Twitter Lite is an example of a successful PWA. When introduced:

  • Pages per session increased 65%
  • Tweets sent increased 75%
  • Bounce rate decreased 20%
  • Average load time decreased 30%
  • As of 2017, Twitter Lite is not the default mobile web experience for all Twitter users.

The Web App Manifest allows you to control how your app appears and how it's launched. You can specify home screen icons, the page to load when the app is launched, screen orientation and if to show the browser in Chrome.

Starting in Chrome 73, Progressive Web Apps are now supported on all desktop platforms, including Chrome OS, Linux, Mac, and Windows.

Desktop PWAs run in their own window. Native feel is important to users of PWAs. Desktop Progressive Web Apps are launched from the same place as other desktop apps and they run in an app window without tabs or an address bar. They look and feel like other apps on the desktop.

When launched from a user's home screen, service workers enable a Progressive Web App to load instantly regardless of the network state. A service worker, written in JavaScript, is like a client-side proxy and puts you in control of the cache and how to respond to resources requested. By pre-caching key resources you can eliminate the dependence on the network, ensuring an instant and reliable experience for your users.

By using a service worker to pre-cache content, the PWA will start consistently fast because it elimantes any network bottlenecks.

For apps that require a network connection, a service worker is still valuable. They give you complete control over the online and offline experience. For example, if the user is offline, you could show cached data or provide your own custom offline page.

Manifest

The Web App Manifest is a simple JSON file that tells the browser about your web application and how it should behave when "installed" on your user's mobile device or desktop. Having a manifest is required by Chrome to show the Add to Home Screen prompt.

A typical manifest file includes information about the app name, icons it should use, the start_url it should start at when launched, and more.

An example manifest.json file:

        

{
    "short_name": "Maps",
    "name": "Google Maps",
    "icons": [
        {
            "src": "/images/icons-192.png",
            "type": "image/png",
            "sizes": "192x192"
        },
        {
            "src": "/images/icons-512.png",
            "type": "image/png",
            "sizes": "512x512"
        }
    ],
    "start_url": "/maps/?source=pwa",
    "background_color": "#3367D6",
    "display": "standalone",
    "scope": "/maps/",
    "theme_color": "#3367D6"
}

Once your manifest has been created, add a link tag to all the pages that encompass your web app.

        

<link rel="manifest" href="/manifest.json">

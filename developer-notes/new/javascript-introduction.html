JavaScript: Introduction

JavaScript has two main characteristics to understand how our code will work. First, JavaScript is synchronous meaning the code will run line after line (most of the time), almost as you read it. Secondly, JavaScript is single-threaded meaning only one command will be executed at any time.

V8 is a JavaScript engine built by Google. It is open source and written in C++. The JavaScript engine compiles JavaScript code into machine code at execution by implementing a JIT (Just-In-Time) compiler. This is a similar approach to how a number of modern JavaScript engines, such as SpiderMonkey or Rhino (Mozilla), work.

A compiler is a program that processes statements written in a particular programming language and turns them into machine language or "code" that a computer's processor uses. Typically, a developer writes code in a language such as JavaScript, Java or C++ using a code editor.

The JIT (Just-In-Time) compiler, converts our JavaScript code into machine language. When commands are entered into the console within the browser developer tools (also known as DevTools), they are passed to the JIT compiler, compiled, executed/run, and the output is returned back to the console.

JavaScript files typically have the file extension of .js

What is ECMAScript?

For its initial release in 1995, Netscape labeled their new language LiveScript before renaming it to JavaScript a year later, hoping to capitalize on the popularity of Java at the time. In 1996, Netscape submitted JavaScript to ECMA International for standardization. This eventually resulted in a new language standard, labeled ECMAScript. All major JavaScript implementations since have been implementations of the ECMAScript standard, but the term JavaScript has stuck around for historical and marketing reasons.

In most instances, ECMAScript is usually used to refer to the standard while JavaScript is used when talking about the language in practice.

JavaScript has no actual relationship to Java, the two are separate.

ECMAScript didn't change much for the first 15 years of existence. After the initial version of ECMAScript, work on the language continued and two more versions were quickly published. After ECMAScript 3 (ES3) was launched in 1999, no major changes were made to the official standard for ten years. Instead, various browser vendors made their own custom extensions to the language leaving developers to support multiple, varied, APIs.

ECMAScript 5 (ES5) was published in 2009 and around 2012, things started to change with a more standard approach to writing JavaScript being adopted. In June 2015, ECMAScript 6 (ES6 or ES2015) was released as a new version of the standard. This is a standard which has been implemented fairly completely in most modern browsers

ES6 and beyond

Since ECMAScript 6 (also known as ES6 or ECMAScript 2015) was released, it introduced a huge set of new features. They include arrow functions, sets, maps, classes, destructuring and much more. The Ecma Technical Committee that governs the ECMA specification, have since decided to release a new version of ECMAScript each year from 2015. A yearly update means no large releases like with ES6.

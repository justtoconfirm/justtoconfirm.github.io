TypeScript

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. This is aimed at making the language more scalable and reliable. It has been maintained by Microsoft since they created it in 2012 and is open-source. The Angular framework (Angular 2) is now written in TypeScript.

TypeScript is a typed version of JavaScript. This means we can specify types to different variables at the time of declaration. Typing is very useful as it ensures reliability and scalability. Type checking helps to ensure our code works as expected and this helps when hunting down bugs or errors and the documenting of our code.

TypeScript allows you to use the latest JavaScript features in your code without having to worry about browser support. Once you've written the code, you can compile it to plain old JavaScript supported by all browsers with ease.

TypeScript can be installed globally using NPM (Node Package Manager).

        

$ npm install typescript -g

If you don't want to install TypeScript globally, just add it to the dependency of your project and create an NPM script for it.

        

"scripts": {
    "tsc": "tsc"
},

This will work as NPM scripts will look for the binary in the ./node_modules/.bin folder and add it to the PATH when running scripts. You can then access tsc using:

        

$ npm run tsc

Options can be passed to tsc using this syntax:

        

$ npm run tsc -- --all

This will list all the available options for TypeScript.

Once installed, we can verify it by running the command which will display the version of TypeScript installed:

        

$ tsc -v

Browsers are unable to run TypeScript code, so this is why it needs to be compiled into JavaScript.

TypeScript files use a .ts file extension. You can write JavaScript code within a TypeScript file and run the TypeScript compiler using the following terminal command:

        

$ tsc filename.ts

The command above will compile our TypeScript into JavaScript. This will create a JavaScript file with the same filename as our TypeScript file but with a different extension. In this example, we will compile our code to create a file named filename.js

If you want to compile all the TypeScript files inside any folder, use the command:

        

$ tsc *.ts

For now, we're just going to write a few lines of plain old JavaScript, as all JavaScript code is valid TypeScript code.

        

let a = 5;
let b = 5;
let c = a + b;

console.log(c);

We can create a more detailed code example in TypeScript and show the compiled JavaScript code.

        

let a: number = 12;
let b: number = 17;

function showProduct(first: number, second: number): void {
    console.log("The product is: " + first * second);
}

showProduct(a, b);

And now the compiled JavaScript:

        

let a = 12;
let b = 17;

function showProduct(first, second) {
    console.log("The product is: " + first * second);
}

showProduct(a, b);

In the TypeScript code we have specified the type of variables as numbers. This means that if we later try to set the value of one of these variables to a string like "Apple", TypeScript will show an error that "Apple" is not a number. Your code will still compile to JavaScript, but this error will notify you that you made a mistake and helps to avoid problems at runtime.

Type Checking

The syntax to assign a type to any variable is to write the name of the variable followed by a : sign, then the name of the type followed by a = sign and the value of the variable.

There are three different types in TypeScript. The any type, the Built-in types, and the User-defined types.

Data type: any

The any data type is the superset of all the data types in TypeScript. Giving any variable this type is equivalent to opting out of type checking for a variable.

        

let myVariable: any = 'This is a string';

Data type: Built-in

These are the types which are built into TypeScript. They include: number, string, boolean, void, null, and undefined.

        

let num: number = 5;
let name: string = 'Dan';
let isPresent: boolean = true;

Data type: User-defined

The User-defined types include: enum, class, interface, array, and tuple.

Class

TypeScript supportes features of object-oriented programming such as classes and interfaces. In object-oriented programming, a class is the template of objects. A class defines how an object would look like in terms of that object's features and functionality. A class also encapsulates data for the object.

TypeScript has a built-in support for classes which were unsupported in ES5 and earlier versions of JavaScript. This means we can use the class keyword to easily declare one.

        

class Car {
    // Fields
    model: String;
    doors: Number;
    isElectric: Boolean;

    constructor(model: String, doors: Number, isElectric: Boolean) {
        this.model = model;
        this.doors = doors;
        this.isElectric = isElectric;
    }

    displayModel(): void {
        console.log(`This car is a ${this.model}`);
    }
}

We have declared a Car class along with some of it's properties which we're initalizing in the constructor. We have also included a method which would display some message using its property.

We can now create a new instance of the class.

        

const Prius = new Car('Prius', 4, true);
Prius.displayModel();    // This car is a Prius

To create an object of a class, we use the keyword of new and call the constructor of the class and pass it the properties. Now this object, Prius, has its own properties of model, doors, and isElectric. The object can also call the method displayMake, which would have access to the properties of Prius.

Configuration

You can pass options to the TypeScript compiler by either using the CLI (Command Line Interface) or creating a special file called tsconfig.json. As we may want to use the same settings for different tasks, the tsconfig.json file maybe the preferred option.

A tsconfig.json file can be created where you can specify all the configuration options for compiling your TypeScript file to JavaScript.

By using this configuration file, we are telling TypeScript things like the build target (ES5, ES6, ES7, etc.), what module system to expect, where to put the build JavaScript file, or whether it should create source-maps as well.

Here is an example tsconfig.json file:

        

{
    "compilerOptions": {
        "target": "es6",
        "module": "commonjs",
        "outDir": "dist",
        "sourceMap": true
    },
    "files": [
        "./node_modules/@types/mocha/index.d.ts",
        "./node_modules/@types/node/index.d.ts"
    ],
    "include": [
        "src/**/*.ts"
    ],
    "exclude": [
        "node_modules"
    ]
}

Once you've added this TypeScript configuration file, you can build your application using the tsc command.

Adding ESLint

As with most projects, you want to have linters to check for style issues in your code. To use ESLint with TypeScript, you will need to add an extra package. This is a parser, so ESLint can understand TypeScript. This is called: typescript-eslint-parser.

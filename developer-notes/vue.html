Vue

Vue is a JavaScript framework for building user interfaces. Vue was created by Even You and originally released in February 2014. Vue itself is lightweight, weighing around 20KB in size.

A JavaScript framework helps us to create modern applications. Vue is called a progressive framework. This means that it adapts to the needs of the developer.

Compatibility

Vue does not support IE8 and below, because it uses ECMAScript 5 features that are un-shimmable in IE8.

Install

NPM (Node Package Manager) is the recommended installation method when building large scale applications with Vue. It pairs nicely with module bundles such as Webpack or Browserify.

        

# Install latest stable version
$ npm install vue

Installing Vue CLI will install Vue by default so the above step is not required.

You can also install Vue using the Vue CLI (Command Line Interface), which is available as an NPM package. Make sure Node and NPM are installed, and use NPM to install Vue CLI globally on your local system:

        

$ npm install vue-cli -g

Update: The package name for Vue CLI has changed from vue-cli (1.x or 2.x) to @vue/cli (3.x). You will need to uninstall the previous versions with NPM or Yarn.

        

$ npm uninstall vue-cli -g

# OR
$ yarn global remove vue-cli

Vue CLI requires Node version 8.9 or above (8.11.0+ recommended). The new way to install Vue CLI is as follows:

        

$ npm install @vue/cli -g

# OR
$ yarn global add @vue/cli

You can check that you have the right version of Vue (3.x) installed with this command:

        

$ vue --version

Having installed the client successfully, the vue command becomes available. The following command will tell vue to initiate a project and use the webpack template. We are also giving the project the name vueapp01. The project is created in the folder named 'vueapp01'.

We will also change into the directory once the folder has been created.

        

$ vue init webpack vueapp01

# Change into the directory
$ cd vueapp01

Start installing the dependencies using NPM.

        

$ npm install

After having completed the installation of the packages, you can start the web server in development mode using NPM.

        

$ npm run dev

This will start the server on port 8080 and the application output is displayed within the browser at http://localhost:8080

Later, if you want to build for production you can use the following command. In this case, a dist folder is created containing the files needed for productive development.

        

$ npm run build

Creating a Project - Vue CLI

Once Vue CLI (3.x) has been installed, you can create a new project.

        

$ vue create hello-world

You can also create and manage projects using a GUI (Graphical User Interface). This will guide you through the project creation process from within the browser. Vue has efficient state management (Vuex) and routing (Vue Router).

Summary of Installation

Here is a summary of installing Vue (version 3.x) using Vue CLI. You will need Node version 8.9 or later to install the later versions of Vue and you can use NVM (Node Version Manager) to switch versions if needed.

        

# Install Vue CLI globally
$ npm install @vue/cli -g

# Check version of Vue installed
$ vue --version

# Create Vue project directory
$ vue create my-app
$ cd my-app

# Start server
$ npm run serve

Once the server starts, the application can be viewed in the browser at: http://localhost:8081/

        

$ vue ui

Declarative Rendering

At the core of Vue is a system that enables us to declaratively render data to the DOM using straightforward template syntax:

        

<div id="app">
    {{ message }}
</div>
        
    
        

var app = new Vue({
    el: '#app',
    data: {
        message: 'Hello Vue!'
    }
})

In this example, the data and the DOM are now linked and everything is reactive. Open your browser's JavaScript console and set app.message to a different value. You should see the rendered example above update accordingly.

Main App Component

Vue CLI already generates a main component that can be found in src/App.vue. Every Vue application needs to have a top level component that serves as the framework for the entire application.

Vue CLI will also create a component during the setup that can be found in src/components/HelloWorld.vue

Here is the structure of the 'src' directory:

        

vue-project/
    src/
        App.vue
        assets /
            logo.png
        components /
            HelloWorld.vue
        main.js

You may also notice that in the 'public' directory, an index.html file can be found. This will serve as the root directory for your project. This is what the code looks like:

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width,initial-scale=1.0">
    <link rel="icon" href="<%= BASE_URL %>favicon.ico">
    <title>vue-project</title>
</head>
<body>
    <noscript>
        <strong>We're sorry but vue-project doesn't work properly without JavaScript enabled. Please enable it to continue.</strong>
    </noscript>
    <div id="app"></div>
    <!-- built files will be auto injected -->
</body>
</html>

Routing

Routing is what handles navigation through the app as you would with a normal website. For example, going from the homepage to the about page, you can consider each page to be a different route.

Vue needs to know what you would like displayed at these routes. It needs a router to do this. Vue does not natively support routing on its own, you can however use Vue Router.

HTML5

HTML5 is the latest version of Hypertext Markup Language (HTML), the code that describes web pages.

<nav>

The HTML <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.

It is not necessary for all links to be contained within a <nav> element. <nav> is intended only for a major block of navigation links. Typically links within the <footer> element often has a list of links. These links do not need to be within a <nav> element.

        

<nav class="menu">
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
    </ul>
</nav>

A document may have more than one <nav> element. For example, one for site navigation and another for inter-page navigation. The aria-labelledby attribute can be used in such cases to promote accessibility and to identify each navigation.

        

<header>
    <nav aria-labelledby="primary-navigation">
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>
</header>

<!-- Page content -->

<footer>
    <nav aria-labelledby="footer-navigation">
        <ul>
            <li><a href="#">Privacy Policy</a></li>
            <li><a href="#">Terms of Service</a></li>
        </ul>
    </nav>
</footer>

<main>

The HTML <main> element represents the dominant content of the <body> of a HTML document. The document must not have more than one <main> element. The content of a <main> element should be unique to the document. Content that is repeated, such as sidebars, navigation links, copyright information, site logos, and search forms should not be included unless the search form is the main function of the page.

The <main> element is widely supported by browsers. For Internet Explorer 11 (IE11) and below, it is suggested that an ARIA role attribute of "main" is added to the <main> element to ensure it is accessible. Screen readers will be able to then understand the semantic meaning of the <main> element once the role attribute is added.

        

<main role="main">
    <! -- Content here -->
</main>

You may need to set the <main> element to be a block element using CSS. This ensures that any browsers that do not support the element will still display it as a block level element.

    

main {
    display: block;
}

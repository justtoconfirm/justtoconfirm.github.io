BEM (Block Element Modifier)

BEM, which stands for Block Element Modifier, is a naming convention standard for CSS class names. It is immensely useful in writing CSS that is easier to read, understand, and scale. BEM is really useful in communicating purpose or function and component structure.

A BEM class name includes up to three parts:

  • Block: The outermost parent element of the component is defined as the block.
  • Element: Inside of the component may be one or more children, called elements.
  • Modifier: Either a block or element may have a modifier.

Block & Element

What is a Block?

Encapsulates a standalone entity that has meaning on its own. Any DOM node can be a block if it accepts a class name.

        

<div class="block"> ... </div>

What is an Element?

Parts of a block that have no standalone meaning are known as elements. Any element is semantically tied to its block. CSS classes for elements are formed as block names plus two underscores followed by the name of the element.

As out block has the class 'block', the element will have the class name of 'block__elem'.

        

<!-- Block -->
<div class="block">

    <!-- Element -->
    <div class="block__elem"> ... </div>

</div>

Adding a Modifier

We can change the appearance or our blocks or elements using modifiers. Our blocks and elements will have styling that can be shared but modifiers allow the default behavior, appearance, or state to be changed (modified).

CSS class names for modifiers are formed from the block's or element's name, followed by two dashes, such as:

        

/* Block */
.block--mod {}
.block--theme-xmas {}

/* Element */
.block__elem-mod {}
.block__elem-shadow {}

Modifier is an extra class name which you can add to a block/element DOM node. Add modifier classes only to blocks or elements they modify, and keep the original class.

        

<!-- Block -->
<div class="block block--mod">

    <!-- Element -->
    <div class="block__elem block__elem--mod"> ... </div>

</div>

We have documented further details about modifiers below.

An Example of BEM

To see the full power of BEM naming, consider this form example. According to the BEM methodology, the form is implemented using the form block. In HTML, the block name is included in the class attribute.

All the parts the make the form (the form block) that don't make sense on their own, are considered the elements of the form block. For instance, the search box and the submit button, are elements of the form block and the classes indicate that an element belongs to the block.

        

<form class="form" action="/">
    <input class="form__search" type="text" name="search" />
    <input class="form__submit" type="submit" />
</form>

Note that the block's name is separated from the element's name with a special separator - two underscores.

Selector names make it clear that in order to move the form to another project, you will need to copy all of it's components:

        

.form__search {}
.form__submit {}

Using blocks and elements for class names solves an important problem, it helps us to rid the use of nested selectors. All selectors in a BEM project have the same weight. To use the same form in another project, you can just copy its layout and styles.

Modifier

A modifier defines the look, state, and behavior of a block or an element. Adding modifiers is optional.

A modifier contains only additional styles that change the original block implementation in some way. This allows you to set the appearance of a universal block only once and add only those features that differ from the original block code into the modifier styles.

        

.form {
    /* Universal block styling */
}

.form--theme-xmas {
    /* Added styling */
}

This is why a modifier should always be on the same DOM node with the block and the element it is associated with.

Lets add a few modifier classes to our previous form example.

        

<form class="form form--theme-xmas" action="/">
    <input class="form__search" type="text" name="search" />
    <input class="form__submit form__submit--disabled" type="submit" />
</form>

Our CSS selectors would look something like this:

        

.form {}
.form--theme-xmas {}
.form__search {}
.form__submit {}
.form__submit--disabled {}

Examples of BEM in action

Here are a few examples of BEM in action.

Component with No Elements or Modifiers

Simple components may only employ a single element and thus a single class would be the block.

        

<button class="btn">Button</button>

.btn {
    display: inline-block;
    color: blue;
}

Component with a Modifier

        

<button class="btn btn--secondary">Button</button>

.btn {
    display: inline-block;
    color: blue;
}

.btn--secondary {
    color: green;
}

Don't use the modifier class by itself. The modifier class is intended to augment, not replace, the base class.

        

<!-- DON'T DO THIS -->
<button class="btn--secondary">Button</button>

.btn--secondary {
    display: inline-block;
    color: green;
}

Components with Elements

More complex components will have child elements. One of the purposes behind BEM is to keep specificity low and consistent. Don't omit class names from the child elements in your HTML. One goal of BEM is for most selectors to use just a single class name. To avoid you to use a selector with increased specificity.

        

<!-- DO THIS -->
<figure class="photo">
    <img class="photo__img" src="me.jpg" alt="Image of me" />
    <figcaption class="photo__caption">Look at me!</figcaption>
</figure>

.photo { }
.photo__img { }
.photo__caption { }


<!-- DON'T THIS -->
<figure class="photo">
    <img src="me.jpg" alt="Image of me" />
    <figcaption>Look at me!</figcaption>
</figure>

.photo { }
.photo img { }
.photo figcaption { }

BEM is not intended to communicate structural depth. A BEM class name representing a child element in the component should only include the base/block name and the one element name. Note that in the example below, photo__caption__quote is an incorrect usage of BEM, while photo__quote is more appropriate.

        

<!-- DO THIS -->
<figure class="photo">
    <img class="photo__img" src="me.jpg" alt="Image of me" />
    <figcaption class="photo__caption">
        <blockquote class="photo__quote">
            Look at me!
        </blockquote>
    </figcaption>
</figure>

.photo { }
.photo__img { }
.photo__caption { }
.photo__quote { }


<!-- DON'T THIS -->
<figure class="photo">
    <img class="photo__img" src="me.jpg" alt="Image of me" />
    <figcaption class="photo__caption>
        <blockquote class="photo__caption__quote">
            Look at me!
        </blockquote>
    </figcaption>
</figure>

.photo { }
.photo__img { }
.photo__caption { }
.photo__caption__quote { }

Element with a Modifier

In some cases, you may want to cary a single element in a component. In those cases, add a modifier to the element instead of the component.

        

<figure class="photo">
    <img class="photo__img photo__img--framed" src="me.jpg" alt="Image of me" />
    <figcaption class="photo__caption photo__caption--large">
        Look at me!
    </figcaption>
</figure>

.photo__img--framed { }
.photo__caption--large { }

Component Modifier

If you find yourself consistently modifying elements of the same component together in the same way, then consider adding the modifier to the base of the component and adjusting the styles for each child element based on that one modifier. This will increase specificity, but it makes modifiying components that much simpler.

        

<figure class="photo photo--highlighted">
    <img class="photo__img" src="me.jpg" alt="Image of me" />
    <figcaption class="photo__caption">
        Look at me!
    </figcaption>
</figure>

.photo--highlighted .photo__img { }
.photo--highlighted .photo__caption { }

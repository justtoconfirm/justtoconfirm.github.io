React: Container Components

Components let you split the UI (User Interface) into independent, reusable pieces, and think about each piece in isolation. The first time you install React (npm install react), you get a component and its APIs. Similar to JavaScript functions, a component accepts input called "props" (properties) and returns React elements which describe what the user interface (UI) should look like. This is why React is referred to as a declarative API because you tell it what you want the UI to look like and React takes care of the rest.

Here are the five APIs you get when you install React:

  • render
  • state
  • props
  • context
  • lifecycle events

Components can be categorised between stateful and stateless. Stateful component typically utilise stateful APIs: render, state, and lifecycle events. Stateless components use: render, props, and context.

This is where we introduce component patterns. These are best use practices that were introduced to split data or logic layers and the UI or presentational layer. By doing this, you can create more reusable components that can be used to compose complext UI's. Common component patterns include:

  • Container
  • Presentational
  • Higher ordrr components (HOC)
  • Render callback

What are Containers?

Containers are your data or logic layer and utilize stateful APIs. Using lifecycle events, you can connect to a state management store such as Redux and pass down data as props to child components.

Here is an example of a Container and Presentational components:

        

// Presentational
const GreetingCard = (props) => {
    return (
        <div>
            <h1>{props.name}</h1>
        </div>
    )
}

// Container
class Greeting extends React.Component {
    constructor() {
        super();
        this.state = {
            name: "",
        };
    }

    componentDidMount() {
        // AJAX
        this.setState(() => {
            return {
                name: "William",
            };
        });
    }

    render() {
        return (
            <div>
                <GreetingCard name={this.state.name} />
            </div>
        );
    }
}

As you can see, we've removed the presentational part from the Greeting class component into it's own functional stateless component.

CSS Grid

CSS Grid Layout introduces a two-dimensional grid system to CSS. Grids can be used to layout major page areas or small UI (User Interface) elements. It is the most powerful layout system available in CSS. It is a two-dimensional system, meaning it can handle both columns and rows, unlike Flexbox, which is largely a one-dimensional system.

CSS Grid was added to most modern browsers (Except IE11) in 2017. It allows us to create grid structures in CSS and not HTML. The latest versions of Chrome, Firefox, Edge, Opera, and Safari have support for CSS Grid as they are evergreen browsers meaning thet automatically update themselves without prompting the user.

To ensure your layout works in every browser, you can start with a default float-based fallback then use progressive enhancement techniques to apply a modern Grid layout. Those with older browsers will not receive an identical expericne, but it will be good enough.

You can check the browser support for CSS Grid using caniuse.com.

A grid is an intersecting set of horizontal and vertical lines, one set defining columns, the other defining rows.

Progressive Enhancement

Progressive Enhancement (PE) is the principle of starting with a rock-solid foundation and then adding enhancements. PE differs from Graceful Degredation (GD) in that GD is the journey from complexity to simplicity, whereas PE is the journey from simplicity to complexity. With PE, you always have a strong foundation to fall back on as complexity is introduced to the project.

  • The first layer is clean, semantic HTML.
  • The second layer is CSS.
  • The third layer is JavaScript.

When adding the CSS Grid layout on top of your fallback layout, you don't need to override any tags or use completely seperate CSS styles. In a browser that does not support CSS Grid, the grid properties you add will simply be ignored. You can also check feature support in CSS using the @supports rule. This allows us to override fallback styles where necessary while older browsers ignore the @supports block.

If you're using floats for laying out elements, keep in mind that grid items take precedence over float. If you add the float CSS property to an element which is also a grid element, the float will be ignored in favor of grid.

Grid Container

As soon as we create a grid container, all direct children of that element will become grid items. We create a grid container by setting the display property.

The two core ingredients of a CSS Grid are the wrapper (parent) and the items (children). The wrapper is the actual grid and the items are the content inside the grid.

To turn our wrapper div into a grid, we simply give it a display of grid:

        

.wrapper {
    display: grid;
}

The HTML structure can the be created:

        

<div class="wrapper">
    <div>One</div>
    <div>Two</div>
    <div>Three</div>
    <div>Four</div>
    <div>Five</div>
    <div>Six</div>
</div>

Grid items are placed in rows by default and span the full width of the grid container. As we haven't defined how we want our grid to look like, it'll simply stack the three <div> elements on top of each other.

Columns & Rows

To make the Grid two-dimensional, we will need to define the colums and rows. We can use the grid-template-row and grid-template-column properties to do this.

        

.wrapper {
    display: grid;
    grid-template-columns: 100px 100px 100px;
    grid-template-rows: 50px 50px;
}

As we've written three values for the grid-template-columns, we'll get three columns. As we've got two values for the grid-template-rows property, we will get two rows.

The values dictate how wide we want our columns to be (100px) and how tall we want our rows to be (50px).

Placing Items

We can create a 3x3 grid using the same markup as before. When using the markup you will only see a 3x2 (3 columns, 2 rows) grid on the page, because we only have six items to fill the grid, so we don't need another row. If we had seven, eight, or nine items, then the third row would be filled as well.

        

.wrapper {
    display: grid;
    grid-template-columns: 100px 100px 100px;
    grid-template-rows: 100px 100px 100px;
}

As an example, this would show a third row, with each item being 100px (comments have been added to help visualise this):

To position and resize the items, we'll target them and use the grid-column and grid-row properties:

        

.item1 {
    grid-column-start: 1;
    grid-column-end: 4;
}

What we're saying here is that we want the element with the class of item1, to start on the first grid line and end on the fourth line. In other words, it'll take up the entire row. It maybe confusing to think we have 4 column lines but only 3 columns, but the first column line is on the left side of the first column, so there are four in total, like a border.

You may see this syntax in a simplified way:

        

.item1 {
    grid-column: 1 / 4;
}

The fr unit indicates the fraction of the remaining space that the item occupies. You can combine it with other units such as % and px. This is great news, as you no longer have to make calculations with percentages while considering the number of columns, amount of margin between items, and further considerations. It simply adjusts to the overall space you have left and to the spacing you set between the items.

        

.wrapper {
    display: grid;
    grid-template-columns: 100px 1fr 100px;
}

Or as a simplified syntax:

        

.wrapper {
    display: grid;
    grid-template-columns: repeat(4, 1fr);
}

Example

As you can see in the example, the central column using the fr unit will stretch to fit any unused space that is not being used by the other two columns.

.wrapper {
    display: grid;
    grid-gap: 25px;

Responsive Grid

Instead of specifying a finite number of columns, we can define sizing properties for the tracks and let the grid calculate how many fit the available space.

As an aside, it's important to know that CSS Grid does not have to be used on an entire page layout. Grids can be used in certain areas of the page if needed. For example, a card layout can utilize CSS Grid even if the rest of the page does not. As long as the grid area is defined and the display:grid property is declared on the wrapper, this approach can be used.

Let's build a simple card-based structure.

In some cases it's easy to plan for a set number of cards. Let's begin by creating a wrapper to contain our cards. We will wrap this in a feature query to check that the browser supports CSS Grid.

        

@supports (display: grid) {
    .cards {
        display: grid;
    }
}

The HTML should look something like this:

        

<div class="cards">
    <!-- This is a wrapper for our cards -->
</div>

The actual card styling will be declared within the class of .card

Let's add some cards. We will begin by adding three four even cards (columns):

        

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(4, 1fr);
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

The HTML should now look like this:

        

<div class="cards">
    <div class="card">
        <p>Card</p>
    </div>

    <div class="card">
        <p>Card</p>
    </div>

    <div class="card">
        <p>Card</p>
    </div>

    <div class="card">
        <p>Card</p>
    </div>
</div>

This will create four columns (cards) that are horizontal but only reduce in size, they are not fluid and they don't wrap underneath as the screen/viewport reduces.

The repeat() function has been used. This function takes two arguments, the number of times to repeat and the value to repeat. For example:

        

grid-template-columns: repeat(4, 200px);

# Computes to:
grid-template-columns: 200px 200px 200px 200px;

We are also using the Fractional unit (fr). This unit of measurement is popular with CSS Grid. 1fr simply takes up 1 part of the available space and helps to simplify space calculations. The example below that we are therefore using simple specifies four columns that will take up the available space within the parent.

        

grid-template-columns: repeat(4, 1fr);

When grid columns are 1fr 2fr 1fr, two fractional units take up twice the space as one fractional unit.

In most cases, you may not know the number of cards that are needed, so planning for a set number won't work. We can use the same approach we had before using repeat() but we can use auto-fill. Width declarations are needed in addition to auto-fill. When using px (pixel) values, there maybe space off to the right when there isn't enough space for the card to fit.

        

grid-template-columns: repeat(auto-fill, 250px);

Let's update our CSS code:

        

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, 250px);
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

Our CSS Grid is now fluid and the four columns will wrap/stack as the viewport is reduced in width, however, a huge gap/space remains on the right of the last column.

To resolve this issue, minmax is the perfect solution since a fractional (fr) unit by itself would make the cards full width. Adding minmax to the repeat() method will remove empty space and the cards will appear more fluid.

        

grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));

Columns will always fit the grid and the cards will be at least 300px. When the minimum space does not perfectly fit the space, this is where the max-width comes into play. When another 300px card won't fit, the other cards will be larger than 300px so they stretch to fill the row with more space. This is what we can use for a three-column layout.

The minimum width of the card is 300px. When the device is very small, there is a good chance that there isn't the space for two cards in a row. That's when the 1fr comes into play. This is ideal for responsive designs. When 1fr has been declared as the max width, a single card will take up all the available space and they will stack on top of each other. This is ideal for mobile responsive designs without the use of media queries.

        

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

The HTML has been set to use only three columns.

        

<div class="cards">
    <div class="card">
        <p>Card</p>
    </div>

    <div class="card">
        <p>Card</p>
    </div>

    <div class="card">
        <p>Card</p>
    </div>
</div>

We can now add spacing between our columns and rows. When you see grid-gap, it is shorthand for grid-row-gap and grid-column-gap. The grid-row-gap is the space on the top and bottom of the grid (card), grid-column-gap is the space to the left and right of the grid (card).

We will set a gutter (gaps/space) to 1rem:

        

grid-gap: 1rem;

Our CSS code should now look like this:

        

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
        grid-gap: 1rem;
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

We can also use the grid-auto-rows CSS property to specify the size of an implicitly-created grid row track. We will set this to auto, but you can specify a size if you wish.

        

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
        grid-auto-rows: auto;
        grid-gap: 1rem;
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

Your CSS Grid is now fluid and responsive. Here you can create card components within a three column grid to display your content. If you have six columns, this will give you an even number of columns as you reduce and increase the viewport size.

        

html, body {
    margin: 0;
    padding: 0;
}

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
        grid-auto-rows: auto;
        grid-gap: 1rem;
        padding: 10px;
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

Footer

It is often tricky to keep the page footer at the bottom of the page, but this can be achieved using CSS Grid. First, we need to set the height on the html and body elements to be 100%. We will also set the min-height on our container to be 100%. In our example, this will be the HTML <main> element we will add.

        

<header role="banner">
    <p>Page Header</p>
</header>

<main role="main">

    <div class="cards">
        <div class="card">
            <p>Card</p>
        </div>

        <div class="card">
            <p>Card</p>
        </div>

        <div class="card">
            <p>Card</p>
        </div>
    </div>

</main>

<footer>
    <p>Page Footer</p>
</footer>

We can update the CSS like so:

        

html, body {
    margin: 0;
    padding: 0;
    height: 100%;
    width: 100%;
}

main {
    display: block;
    min-height: 100%;
}

@supports (display: grid) {
    .cards {
        display: grid;
        grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
        grid-auto-rows: auto;
        grid-gap: 1rem;
        padding: 10px;
    }

    .card {
        border: 1px solid #eaeaea;
    }
}

In examples I have seen, it is also recommended to update the grid-auto-rows property, but this doesn't seem to be needed.

        

grid-auto-rows: auto 1fr auto;

The page footer will now stay at the bottom of the page.

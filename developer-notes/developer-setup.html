Terminal Colours: Git Branch

Often it can be nice to set the colour scheme for your terminal. When working with Git, it can also be useful to show what Git branch your on. If not already created, begin by creating a .bash_profile dotfile in your home directory.

        

$ touch .bash_profile

The bash_profile is a configuration file for bash shell. When bash is invoked as an interactive login shell it first reads and executes cmmands from the .bash_profile dotfile.

When you open terminal, in Mac OS, it automatically prompts you with a bash shell. Bash is a unix based shell and is an acronym for Bourne-again shell.

Open the newly created .bash_profile dotfile and add the following:

        

export CLICOLOR=1
export LSCOLORS=ExFxBxDxCxegedabagacad
parse_git_branch() {
  git branch 2> /dev/null | sed -e '/^[^*]/d' -e 's/* \(.*\)/ (\1)/'
}
export PS1="\[\033[36m\]\u\[\033[m\]@\[\033[32m\]\h:\[\033[33;1m\]\w\$(parse_git_branch)\[\033[m\]\$ "

You may need to close and reopen the terminal for this to take effect.

When entering commands into the terminal, many examples will include a leading $ at the start. This is a common convention used to denote what follows as a command to be run. The command does not require $ to be added to the start of a command.

Terminal commands

Here is a list and descriptions for key terminal commands:

        

$ pwd - print working directory, the current location (working directory)

$ cd - change directory (without directory, takes you to home directory (~))
$ cd Documents - change directory to the 'Documents' directory
$ cd .. - from my Documents directory, take me up to my home directory

$ mkdir - make a new directory
$ mkdir myApp - make a new directory named 'MyApp'

$ touch - make a new empty file
$ touch MyApp - make a new empty file named 'MyApp'

$ cp - copy a file
$ mv - move a file
$ rm - remove a file or directory

$ clear - clear terminal

Setup GitHub

The first thing you should do with Git is to set your global configuration. We can do this by running a lot of small commands which update the Git configuration file.

        

$ git config --global user.name "First Last"

Or, we can just create the Git configuration file within the home directory and input it all ourselves.

        

$ touch .gitconfig

Here I'll input my name, email, GitHub username and some aliases to be able to type less and do more. This will connect Git to the Mac OS Keychain so I don't have to type my username and password every time I want to push to GitHub.

        

[user]
    name = First Last
    email = email@email.com
[github]
    user = username
[alias]
    a = add
    ca = commit -a
    cam = commit -am
    s = status
    pom = push origin master
    pog = push origin gh-pages
    puom = pull origin master
    puog = pull origin gh-pages
    cob = checkout -b
[credential]
    helper = osxkeychain

With the above aliases, I can run 'git s' instead of 'git status', for example.

Keychain

The Keychain in Mac OS is Apple's password management system. A keychain can store all your passwords for applications, servers, and websites. It was introduced in Mac OS 8.6 and has been included in all subsequent versions.

NextJS

NextJS (Next.js) is a minimalistic framework for server-rendered React applications. NextJS was open-sourced on 25th October 2016.

Client side rendering - Normally, when you use React, your browser will download a minimal HTML page and the content will be filled in by JavaScript.

Server side rendering - The initial content is generated on the server, so your browser can download a page with HTML content already in place. Updates to the content are still handled in the browser.

Think about how webapps are created with PHP. You create some files, write PHP code, then simply deploy it. We don't have to worry about routing much and the app is rendered on the server by default. That's exactly what we do with Next.js. Instead of PHP, we build the app with JavaScript and React.

To begin using NextJS in your project, you just need to install it as an NPM (Node Package Manager) module.

        

$ npm install next --save

You may need to also have React and React DOM installed and to create a directory for this:

        

$ mkdir next-react
$ cd next-react
$ npm init -y
$ npm install react react-dom next --save
$ mkdir pages

You may need to add a build script within your package.json:

        

"scripts": {
    "dev": "next"
}

You may eventually have a package.json file like this:

        

{
  "scripts": {
    "dev": "next",
    "build": "next build",
    "start": "next start",
    "export": "next export"
  },
  "dependencies": {
    "next": "^7.0.0",
    "react": "^16.5.1",
    "react-dom": "^16.5.1"
  }
}

If everything is setup correctly, we should have React, Webpack, hot module loading, routing, server-side rendering, pre-fetching and much more, right out of the box.

NextJS will show us a 404 page without any pages, so this is why we can create a pages directory in our project setup. Let's create a component inside our pages directory named index.js.

        

$ touch pages/index.js

The page can be a very simple React component, but with Next we don't need to import React within the component. Each time we make a change to the component, you can see hot reloading in action.

        

// pages/index.js

const Index = () => (
    <div>
        <h1>Hello Next</h1>
        <p>I'm already in love with you!</p>
    </div>
);

export default Index;

Our React app is now being server side rendered.

In this example, we have two files that are components that represent pages:

        

// pages/about.js

import Link from 'next/link'

export default () => (
    <div>
        <p>This is the about page</p>
        <Link href='/'><a>Go home</a></Link>
    </div>
)

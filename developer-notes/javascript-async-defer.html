JavaScript Async & Defer

When loading a script on an HTML page, you need to be careful not to harm the loading performance of the page. Whenever the HTML parser finds the <script> element, a request will be made to fetch the script and the script is then executed. Once this process is done, the parsing can resume and the rest of the HTML can be analyzed. This can impact the loading time of the page.

If the script takes a little longer to load than expected, for example if the network is a bit slow or the page is being viewed on a mobile device with a slow connection, the visitor will likely see an empty page until the script is loaded and then executed.

It is best to place any <script> tags at the bottom of your HTML page, just before the closing </body> element. Doing so, the script is loaded and executed after all the page is already parsed and loaded. If the <script> tag were to be placed inside the <head> section of the HTML document, this would cause the page to remain blank until the script has been parsed and executed.

        

<script src="myscript.js"></script>

</body>
</html>

Both async and defer are boolean attributes. Their usage is similar.

        

// Async
<script src="myscript.js" async></script>

// Defer
<script src="myscript.js" defer></script>

If you specify both, async takes precedence on modern browsers. Older browsers that support defer but not async, will fallback to defer.

If the <script> elements are needed in the <head> section of your HTML, these attributes are ideal to use.

The best thing to do to speed up your page loading, is to place your <script> elements in the <head> and add a defer attribute to your <script> tag. This is the scenario that triggers the faster domInteractive event. Unless you are fine with delaying the first render of the page, make sure that when the page is parsed, the JavaScript you want is already executed.

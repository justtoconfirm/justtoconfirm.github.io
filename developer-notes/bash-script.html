Bash Profile & Bash Script

Bash, which stands for Bourne Again SHell (Bash), is a command-line interpreter that runs on Unix, Linux, and machines running Mac OS. It is the standard shell under OS X - meaning it is the standard program to interface with the operating system's (OS) underpinnings.

Bash profile

Bash Profile or bash_profile is a configuration file for the bash shell. When bash is invoked as an interactive login shell, it first reads and executes commands from the .bash_profile. This file can be used to export variables in shell. These shell variables include: PATH, USER, and HOSTNAME.

Begin by setting up a .bash_profile file. First, open your terminal and add the following commands:

        

$ cd ~/                 # This will take you to the home directory
$ touch .bash_profile   # Create a .bash_profile file

Within the Finder you should be able to locate the .bash_profile file within the home directory. If hidden, use the following keyboard shortcut to reveal hidden files:

        

Command + Shift + . (dot)

Creating a Bash script

A bash script is a script used to tell the Bash shell what it should do. It is a plain text file that contains a series of commands. It is convention to give files that are Bash scripts an extension of .sh

Let's create a bash script that will allow us to make git commits using a single command within the terminal. Open your terminal, and follow these commands:

        

$ cd ~
$ mkdir bin             # Create a directory named 'bin'
$ cd bin
$ touch git-deploy      # Create a script file, this should have no file extension

Open the .bash_profile and add the following line:

        

export PATH=$PATH:/Users/jacksod/bin

Replace the 'jacksod' part of the path with your own username.

Within the git-deploy script file, add the following code:

        

#!/bin/bash

# gets the current git branch
branch=$(git branch | sed -n -e 's/^\* \(.*\)/\1/p')

# prompt the use for a commit message
read -r -p 'Commit message: ' desc
# track all files
git add *
# commit with message
git commit -m "$desc"
# push to origin
git push origin $branch

We now need to set rights access to the file. Within the terminal, navigate to the git-deploy script file and run the following command:

        

$ chmod u+x git-deploy

Close the terminal and reopen it. Now run the bash script file using the following command:

        

$ git-deploy

JavaScript Closures

A closure is a feature in JavaScript where an inner function has access to the outer (enclosing) function's variables - a scope chain. Ths is basically what is created when you declare a function within another function.

Imagine a Russian doll (Matryoshka), an outer doll wouldn't be able to see the inside of an inner doll. The inner doll can see everything. This is a metaphor for scope.

        

// Global scope
function outer() {
    // Closure's outer scope
    function closure() {
        // Closure's inner scope
    }
}

We can conclude that the inner function has three scope chains. The first chain allows the inner function to access its own variables. The second, allows it to access the variables and parameters of the outer function. The third, allows it access to the global scope and global variables.

Each time we add a function, we're adding a new link to the chain.

By defining a function within another function, we create a closure.

Closures have access to its outer function scope even after the outer function has returned. This means a closure can remember and access variables and arguments of its outer function even after the function has finished.

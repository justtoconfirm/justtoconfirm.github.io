Debugging

Debugging is the process of identifying and removing errors from hardware or software. In Chrome we can use Developer Tools (DevTools) to debug issues within our browser.

In Chrome, the Sources panel in DevTools is where you debug JavaScript.

A common method for debugging a problem in JavaScript is to insert a lot of console.log() statements into the code in order to inspect values as the script executes.

Outputting something to the console is often called "Logging".

You may also want to use console.table() to display data as a table which can greatly improve readability when logging output. This is ideal for arrays.

        

var rainbowArray = ["red", "orange", "yellow", "green", "blue"];
console.table(rainbowArray);

Breakpoints can also be used as they are quicker that using console.log() and they let you pause your code in the middle of its execution. Using breakpoints allows you to examine all values at that moment in time. They also avoid the need to manually open the source code to insert console.log() statements and reload the page.

The "Sources" pane

Open your page and turn on the Chrome DevTools using the F12 keyboard shortcut or right click on the page and click the 'Inspect' menu option. When DevTools is showing, select the Sources pane. This will open with nothing showing. The left panel will contain a list of files. Select an HTML (.html) or JavaScript (.js) file in the tree view. The code will be displayed in the central panel (Source Zone).

  • The Resources zone - Lists the HTML, JavaScript, CSS, and other files that are attached to the page. Chrome extensions may also appear here too. This is the left most panel.
  • The Source zone - Shows the source code once a file is selected.
  • Information & control zone - The panel on the right is for debugging. Should include options such as: 'Event Listener Breakpoints'.

The Source zone may feature "{ }" on the bottom left hand side. By clicking this, your code will be beutified for HTML, CSS or JavaScript.

The Console below these panels allow commands to be typed there. Press Enter to execute a command. After a statement is executed, its result is shown below.

Within the Source zone panel that shows the source code, should you click on one of the line numbers down the left side, a breakpoint will be set. A breakpoint is a point of code where the debugger will automatically pause the JavaScript execution. While the code is paused, we can examine current variables, execute commands in the console etc and anything else we may use to debug your code.

When we add a breakpoint, it will be listed in the 'Breakpoints' section of the Information and control zone panel (right panel) within the "Sources" pane. This will also allow you to toggle the breakpoints on and off for debugging.

We can also pause the code using the debugger command.

        

function hello(name) {
    let phrase = `Hello, ${name}!`;

    debugger;   // the debugger stops here

    say(phrase);
}

This is very convenient when we are in the code editor and don't want to switch to the browser or set the breakpoints.

Pseudo State

If you would like to see how a pseudo state looks like on a certain element, right click the element and select 'Force state'. Select the pseudo state (:hover, :active, etc). This is really useful in checking to see what an element's hover or visited state is.

Reference the currently selected element

Should you want to reference the currently selected element from within the console, type:

        

$0

To get the value of the last operation from the console:

        

$_

To write commands that span multiple lines in the console: shift + enter

To capture a screenshot of an element you would like to capture:

        

# Then type: screen - this is the Command Menu
command + shift + p

If you are interested in playing around with experimental Chrome development features, check out: chrome://flags/ It allows you to turn on certain Chrome features.

Unused CSS and JavaScript

Since Chrome 59, a feature has been enabled in DevTools allowing you to check how much CSS or JavaScript is used in your webpage. Find unused CSS and JavaScript code with the new Coverage tab. When you load or run a page, this tab tells you how much code was used versus how much was loaded. You can reduce the size of your pages by only shipping the code that you need.

This feature is in the Sources/Coverage tab.

Clicking on a URL reveals the file in the Sources panel with a breakdown of which lines of code executed. Solid green means the code is executed, red means it did not execute. If the colours are mixed, it means only some of the code executed.

  • Open the Command Menu (CMD + SHIFT + P)
  • Start typing Coverage and select Show Coverage

As you interact with the page, the results with change as code is executed.

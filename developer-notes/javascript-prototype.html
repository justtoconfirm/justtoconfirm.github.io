JavaScript Prototype

JavaScript is a prototype-based language, meaning object properties and methods can be shared through objects that have the ability to be cloned and extended. This is known as prototypical inheritance and differs from class inheritance. Among popular object-oriented programming languages, JavaScript is relatively unique, as other languages such as PHP, Python, and Java, are class-based languages, which instead define classes as blueprints for objects.

We will learn how prototypes can be used to extend objects.

Every object in JavaScript has an internal property called prototype. Prototype is an internal property that cannot be accessed directly in code.

We can demonstate this by creating a new, empty object.

        

var x = {};

// We can also use an object constructor
var x = new Object();

To find the prototype of this newly created object, we will use the getPrototypeOf() method.

        

Object.getPrototypeOf(x);

The output will consist of several built-in properties and methods.

        

// Output
{constructor: ƒ, __defineGetter__: ƒ, __defineSetter__: ƒ, hasOwnProperty: ƒ, __lookupGetter__: ƒ, …}

It is important that every object in JavaScript has a prototype as it creates a way for any two or more objects to be linked. Objects that you create have a prototype as do built-in objects such as Date and Array. A reference to this internal property from one object to another via the prototype property.

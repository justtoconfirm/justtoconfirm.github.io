CSS Modules

A CSS Modules is a CSS file in which all class names are scoped locally by default. With CSS Modules, your CSS class names become similar to local variables in JavaScript. A CSS Module is just a CSS file but is called a 'CSS Module' if you plan on using it with a CSS Modules compiler. A CSS Module goes into the compiler and CSS comes out the other side.

With CSS Modules, we can modularize things in React and add some structure to the flow.

CSS is a language of globals. Global CSS can easily rewrite styling in other classes. A change in one class can ultimately change the UI on multiple components. Generally, nobody wants that to happen.

Often when developing an app, we have styling that becomes redundant or no longer in use. This can, over time, result in large file sizes. With CSS Modules, you only import the styles you need, therefore, reducing redundancy and increase our ability to spot CSS that is not being used.

When working with Modular CSS in React, we can work in a few ways:

  • Using CSS as JavaScript Objects.
  • Using CSS Modules.

CSS as JavaScript Objects

Writing CSS as JavaScript objects are easier in the context of React as it ensures you do not have to stray too far from JavaScript to apply CSS styling. CSS written as JavaScript objects can be leveraged directly in React components by using the style attribute on components.

        

// lets have some CSS in js: style.css.js
const button = {
    backgroundColor: '#202020',
    padding: '12px',
    borderRadius: '2px',
    width: '100%'
}
export default { button }

//importing and using in a JSX
import styles from './style.css.js'

const button = () => {
    return <button style={styles.button}>Vanity Button</button>
}

This allows us to have styling that is unique to that element.

CSS Modules

CSS Modules are not very different from the method above, however, with CSS Modules, we can write our usual CSS code and import that into our component.

        

// lets have some CSS in JS: style.css
.button {
    background-color: #202020;
    padding: 12px;
    border-radius: 2px;
    width: 100px;
}

// Importing and using in a jsx
import styles from './style.css'
import React from 'react';

export default class Button extends React.Component {
    render(){
        return (
            <button className={styles.button}>Vanity Button</button>
        )
    }
}

This example will not work on it's own, you will need to setup a CSS loader for the CSS to be applied when using a CSS Module. This is likely to be a CSS loader applied to the Webpack config file (webpack.config.js).

Both examples allow us to style our components, but the CSS styling will be output with our JavaScript code within our application.

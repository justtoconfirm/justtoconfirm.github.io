Tutorial: Build a Website

The aim of this tutorial is to build a website. You will require one of the following code editors:

  • Sublime
  • Atom
  • Visual Studio Code (VSC)

Here is a list of the notes that will help support this tutorial:

  • HTML
  • HTML5
  • CSS

Here is the demo website we are building: Demo

You may want to create a new folder/directory for all your files. In this tutorial, this folder/directory will be named 'my-website'.

HTML Structure

We shall begin by coding the HTML (Hypertext Markup Language) of our page. Open a new file in your code editor and add the following HTML code:

        

<html>
<head>

    <!-- Empty for now -->

</head>
<body>

<!-- Content goes here -->

</body>
</html>

You will need to save this file as index.html

You may want to save this within your folder/directory created named 'my-website'. Here is the structure of your folder/directory:

        

my-website /
    index.html

If you open this HTML file, it should open in a web browser as an empty webpage without any content. Each time you make changes to your HTML code you will need to save the file and refresh the page in the browser to see your changes.

Adding a page title

Every HTML document should have a page title. The HTML <title> tag is used for declaring the title of the HTML document that is displayed in the browser's title bar and also in bookmarks and search results. The <title> tag is placed between the opening <head> and closing </head> tags.

The HTML code has been updated to now include the <title> element for the page title. The page title of 'My Webpage' has been used in this example:

        

<html>
<head>

    <title>My Webpage</title>

</head>
<body>

<!-- Content goes here -->

</body>
</html>

<!DOCTYPE> declaration

The <!DOCTYPE> declaration is not an HTML tag; it's an instruction to the web browser confirming the version of HTML the page is written in. You should always add the <!DOCTYPE> declaration to your HTML documents so the browser knows what type of document to expect. This should be on the very first line of your HTML code.

As we are using HTML5, there is only one <!DOCTYPE> we need to add:

        

<!DOCTYPE html>

The HTML code should now look like this:

        

<!DOCTYPE html>
<html>
<head>

    <title>My Webpage</title>

</head>
<body>

<!-- Content goes here -->

</body>
</html>

Attributes

All HTML elements can have attributes. Attributes provide additional information about an element and are always specified in the start tag of an element. They usually come in name/value pairs.

Setting the Language

The "lang" attribute specifies the language of the element's content. Common examples are "en" for English, "es" for Spanish and "fr" for French. This attribute can be used on any HTML element within your document.

Within the <html> tag, we are going to add the "lang" attribute to set the language for our HTML document to be English.

        

<html lang="en">

Character Encoding

Next, we need to add a <meta> tag to our HTML document to set the character encoding of our document. Often we use UTF-8 (Universal Character Set + Transformation Format - 8 bit) encoding as it is capable of encoding all possible characters (called code points) in Unicode. Basically, it lets you write text in all kind of languages so characters such as accents will appear perfectly fine and the browser will understand them correctly.

Place the following code just after the opening <head> tag within your HTML:

        

<meta charset="utf-8" />

Here is our HTML code:

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />

    <title>My Webpage</title>

</head>
<body>

<!-- Content goes here -->

</body>
</html>

Adding Content

Within the HTML code, remove the following HTML code comment:

        

<!-- Content goes here -->

Once removed, we will now add simple text that reads 'Hello World' in it's place. This will be displayed on the page so when you open the webpage in your browser 'Hello World' will be visible. We will wrap the text within opening <p> and closing </p> HTML paragraph tags.

Any content you want to add to your webpage should be added between the opening <body> and closing </body> HTML tags.

        

<p>Hello World</p>

Viewport Meta Tag

A typical mobile-optimized, responsive site contains something like the following:

        

<meta name="viewport" content="width=device-width, initial-scale=1" />

Place this line of code underneath the earlier <meta> tag that was added earlier.

Here is our initial HTML code structure following the recent changes:

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />

    <title>My Webpage</title>

</head>
<body>

<p>Hello World</p>

</body>
</html>

HTML5 <main> element

The HTML <main> element represents the main content of the <body> of the HTML document. The content inside the <main> element should be unique to the document and should not contain any content that is repeated across the documents/pages, such as page headers, page footers and navigation. You should only ever have one <main> element within your HTML document.

Add the <main> element to your HTML document after the opening <body> tag and closing </body> tag.

        

<body>
<main>

<p>Hello World</p>

</main>
</body>
</html>

The <main> element is widely supported in modern browsers. For Internet Explorer 11 (IE11) and below, it's suggested that an ARIA (Accessible Rich Internet Applications) role attribute of "main" is added to the HTML <main> element to ensure accessibility. Screen readers will then be able to understand the semantic meaning of the <main> element once the role attribute is added.

        

<body>
<main role="main">

<p>Hello World</p>

</main>
</body>
</html>

Styling with CSS

Now our initial HTML page structure has been created, we can begin to apply styling. This is often done through the use of CSS (Cascading Style Sheets). When creating a website, the CSS is often placed in a separate file so we can use the same stylesheet across multiple pages without having to replicate the same code for each page.

Open an new empty file in your code editor and save it within your project folder/directory as 'main.css'. The file will need to have the file extension of '.css' to be an external stylesheet. Often, this is placed in a sub-directory named 'styles' within your project. Your folder/directory structure may now look like this:

        

my-website /
    index.html
    styles/
        main.css

Often the first page of your website (the homepage), is named index.html as it will likely be the first filename a webserver will look for. It has therefore become a tradition to name your homepage index.html when creating a website.

Within your HTML code we can now link our HTML to our CSS stylesheet using the <link> element.

        

<link rel="stylesheet" href="styles/main.css" />

The <link> element has a "rel" relationship attribute set to "stylesheet" and a "href" attribute which is a reference to the location of the external CSS stylesheet you want to use.

This element must be placed between the opening <head> and closing </head> tags within your HTML document.

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />

    <title>My Webpage</title>

    <link rel="stylesheet" href="styles/main.css" />
</head>
<body>
<main role="main">

<p>Hello World</p>

</main>
</body>
</html>

As we have not added any styling into our CSS stylesheet as yet, when you reload your webpage, nothing will change.

Adding CSS

Within the 'main.css' CSS stylesheet, we can begin to add our CSS code. The stylesheet itself should be empty and currently contain no code.

The CSS syntax often looks like this. This is known as a CSS declaration block:

        

selector {
    property: value;
}

CSS Box Sizing

The CSS box-sizing property allows us to include the padding and border in an element's total width and height. By default, the width and height of an element is calculated like this:

        

width + padding + border = width of an element
height + padding + border = height of an element

This means that when you set the width or height of an element, the element often appears bigger than you have set because the element's border and padding are also added. The box-sizing property allows us to include the padding and border in an element's total width or height.

We simply need to set the box-sizing property to have the value 'border-box'. This can be applied to the <html> element within our HTML document. We can then use the select all selector (*) to inherit the border-box value from the parent <html> element and apply it to every child element within our HTML code, including any elements that use :before or :after pseudo classes.

Open the 'main.css' stylesheet and add the following CSS code:

        

html {
    box-sizing: border-box;
}

*,
*:before,
*:after {
    box-sizing: inherit;
}

Using the select all CSS selector (*) can be useful should you want to apply the same styling to every element, however, this is not always recommended as it can harm performance.

<main> element

We have a <main> HTML element within our HTML document code. We will need to set this to be a block element. This will ensure that any browser that does not support the element will still display it as a block element.

        

html {
    box-sizing: border-box;
}

*,
*:before,
*:after {
    box-sizing: inherit;
}

main {
    display: block;
}

Once again, you shouldn't see any changes to your webpage styling.

Setting Text/Font Sizes

Begin by adding a new CSS property of font-size to the html selector within your CSS stylesheet:

        

html {
    box-sizing: border-box;
    font-size: 62.5%;
}

We will be setting our font sizes to use rem (root element) units rather than pixels. Rem units are equal to the computed value of the font-size on the root element. This means that 1rem equals the font size of the <html> element, which for most browsers has a default value of 16px.

By setting the font-size of the <html> element to be 62.5%, it allows us to easily calculate the rem value compared with using pixels.

        

2rem    = 20px
1.8rem  = 18px
1.6rem  = 16px
1.4rem  = 14px
1.2rem  = 12px

Add a body selector to your CSS stylesheet which sets the font-size to be 1.6rem (16px) by default on your page. Your CSS stylesheet should look something like this:

        

html {
    box-sizing: border-box;
    font-size: 62.5%;
}

*,
*:before,
*:after {
    box-sizing: inherit;
}

body {
    font-size: 1.6rem;
}

main {
    display: block;
}

If you add a CSS property that the browser does not understand, it will simply ignore it. We can add a fallback for browsers that do not understand rem values but understand px (pixel) values to set our text size.

        

body {
    font-size: 16px;
    font-size: 1.6rem;
}

In this example, rem will be used on browsers which support that unit and px will be used on those which do not. According to caniuse.com, most modern browsers, including Internet Explorer 11 (IE11) support rem units, however older browsers such as IE6-8 have no support and IE9-10 have partial support.

So we are adding the fallback to pixel values to support all recent versions of Internet Explorer (IE). If you feel this is not necessary, you can avoid the use of the pixel fallback and use rem units.

        

html {
    box-sizing: border-box;
    font-size: 62.5%;
}

*,
*:before,
*:after {
    box-sizing: inherit;
}

body {
    font-size: 16px;
    font-size: 1.6rem;
}

main {
    display: block;
}

Adding Links

Links - also referred to as 'hyperlinks' - allow users to navigate around your website, to different pages, or away from the page to an external website. Links within HTML are really easy to create. You will need to reopen your index.html page once again.

Between the opening <body> and closing </body> tag, we are going to add a link. This will not yet navigate anywhere as yet, but will act as a demonstration of how links work.

Add the following code to the <body> section of your HTML page:

        

<a>This is a link</a>

You may want to remove the line of code containing the 'Hello World' text from your HTML code.

As you can see you a link element is made up of an opening <a> tag, called an anchor tag, a closing </a> anchor tag and the text of the link between the two. The text is what appears on the page and is displayed to your users.

As an example, should you want to navigate to Google when you click a link, you simply need to add the 'href' attribute to your link containing the address of the webpage or site you want to link to. It maybe sensible to also update the text within the link to identify where you will navigate your users to if they should click the link on the page.

        

<a href="https://www.google.co.uk">Google</a>

But you may just want to navigate through your website from page to page. For this we simply change the value of the 'href' attribute to the name of a page within your website instead of a URL. As an example, we will update our link to open an 'About us' page. This page will have the filename of 'about-us.html' and this is what we shall use.

        

<a href="about-us.html">About us</a>

We currently do not have an 'About us' page created, so clicking this link will not work. Here is how your HTML code should look now:

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />

    <title>My Webpage</title>

    <link rel="stylesheet" href="styles/main.css" />
</head>
<body>
<main role="main">

<a href="about-us.html">About us</a>

</main>
</body>
</html>

To save time, I have created the 'About us' page by copying our HTML code from index.html and creating a new file named 'about-us.html'. This new file will contain the following code:

        

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />

    <title>My Webpage - About us</title>

    <link rel="stylesheet" href="styles/main.css" />
</head>
<body>
<main role="main">

<a href="index.html">Home</a>

</main>
</body>
</html>

As you can see, the 'index.html' and 'about-us.html' are almost identical. The title is slightly different and so is the link, which allows a user to navigate back to the homepage of your website ('index.html'), but both pages have a similar HTML structure and use the same CSS stylesheet meaning any CSS changes will affect both pages.

Here is how the file structure should look in your folder/directory:

        

my-website /
    index.html
    about-us.html
    styles/
        main.css

Styling the links

Links can be styled with a variety of CSS selectors and can be styled dependening on what state they are in. The four states of links are:

  • a:link - a normal, unvisited link.
  • a:visited - a link the user has visited.
  • a:hover - a link when the user moves the mouse cursor over it.
  • a:active - a link the moment it is clicked.

When setting the style for link states there are some rules to the order within your CSS:

  • a:hover state MUST come after a:link and a:visited
  • a:active state MUST come after a:hover

As we are not yet applying CSS classes to our code, therefore we will update our CSS to apply styling to all links within our website. This is not ideal, but it does allow us to demonstrate how we can style the four states of our links.

Add the following CSS code to your 'main.css' CSS stylesheet:

        

a:link {
    color: #ffa500;
}

a:visited {
    color: #ffa500;
}

a:hover {
    color: #ff8c00;
    text-decoration: none;
}

a:active {
    color: #ffa500;
}

You should now see the styling of your link change within your HTML page(s).

Adding <header> and <footer> elements

Within our HTML code, we now want to update the code structure on both the index.html and about-us.html pages to include a page header and footer. We will be using the HTML5 <header> and <footer> elements to do this.

Open both HTML documents in your code editor and add begin by adding the <header> element. This needs to be added after the opening <body> tag but before the opening <main> tag.

        

<body>
<header>
    Page Header
</header>

<main role="main">

We need to also add the <footer> element to your HTML code. This can be placed just after the closing </main> tag but before the closing </body> tag.

        

</main>

<footer>
    Page Footer
</footer>
</body>
</html>

Your website won't look too impressive at the moment, but we're still building the HTML structure and adding small bits of CSS styling so it's a start.

You may want to also add a role attribute of "banner" to your <header> element.

        

<body>
<header role="banner">
    Page Header
</header>

<main role="main">

    Page Footer
</footer>

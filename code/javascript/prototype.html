Prototype

The prototype is essentially an object to which we can attach methods and properties, allowing other objects to inherit these methods and properties.

Introduction to Prototype

JavaScript is a prototype based language, meaning object properties and methods can be shared through objects. Whenever we create an object, such as a function or array, the JavaScript engine will add an internal prototype property inside our object where methods and properties can be attached. This is known as the Prototype Object and is identified as [[Prototype]].

Every object, those that we create and built-in objects such as Date or Array, in JavaScript, has an internal property called [[Prototype]]. The double square brackets that enclose [[Prototype]] signify that this is an internal property and cannot be accessed directly in the code.

The prototype is essentially an object to which we can attach methods and properties, allowing other objects to inherit these methods and properties. Every object in JavaScript has a [[Prototype]] as it creates a way for two or more objects to be linked.

If we add a property or method to an object, the additional properties and methods can only be accessed by that object as it's only been defined in the one object. We can therefore use prototype to add new properties and methods to an object so that they can be inherited by other objects.

Demonstration of JavaScript Prototype

An Array has many built-in methods, such as pop() and push(). We can access these methods when we create an array because any array we create has access to the properties and methods on the Array.prototype.

Here we are creating an empty array:

	

let y = [];

Every object, those that we create and built-in objects such as Date or Array, in JavaScript has an internal property called [[Prototype]].

Accessing the Prototype Object

To find the [[Prototype]] of an object, we will use the getPrototypeOf() method. The Object.getPrototypeOf() method returns the prototype object (the value of the internal [[Prototype]] property).

	

Object.getPrototypeOf(y);		// Return the [[Prototype]] object of the array

The browser support for the Object.getPrototypeOf() method:

  • Chrome - version 5 +
  • Edge - version 12 +
  • Firefox - version 3.5 +
  • Internet Explorer 9 (IE9) +
  • Opera - version 12.1 +
  • Safari - version 5 +

We can also find the [[Prototype]] through the __proto__ property, however, this property is a legacy feature and should not be used in production code.

	

y.__proto__ 		// Legacy, avoid for production, use Object.getPrototypeOf()

If we take a look at the [[Prototype]] of our array object, we can see that it has more properties and methods that a simple object.

Prototype Inheritance

When we attempt to access a property or method of an object, JavaScript will first search on the object itself. If the property or method is not found, it will search the object's [[Prototype]]. If the property or method is not still not found in the object or [[Prototype]], JavaScript will check the prototype of the linked object and will continue searching until the end of the prototype chain is reached.

At the end of the prototype chain is Object.prototype. All objects inherit the properties and methods of Object. Any attempt to search beyond the end of the chain results in null being returned.

We can create an empty object:

	

let x = {}

We can log the object into the console and this will also show the prototype object.

	

console.log(x);

our empty object inherits from Object. Our object can use any property of method of Object, such as toString(), for example.

	

x.toString();

Constructor Functions

A Constructor Function is just a regular function but it is called on by an instance with the new keyword. In JavaScript, we capitalize the first letter of a Constructor Function by convention.

One way to create an object in JavaScript is with a function. Properties and values can be added to an object using dot notation.

When a function is declared, the [[Prototype]] object will be created. In JavaScript, a function is also an object.

Regex

Regular Expressions (Regex) is a sequence of characters that specifies a search pattern within a string.

What is Regex?

Regex stands for Regular Expression and is a sequence of characters used to specify a search pattern. In JavaScript, Regex is used in many applications, including form validation. Essentially, Regex are text patterns that we can use to match, or replace, elements within strings.

To check our Regular Expressions, we can use online Regex checkers:

  • https://regex101.com/
  • https://regexr.com/

Creating a Regular Expression

You can construct a regular expression in one of two ways:

Using a regular expression literal, which consists of a pattern enclosed between slashes, which compiles when the script is loaded:

	

const regex = /ab+c/;

Or, we can call the constructor function of the RegExp object:

	

const regex = new RegExp('ab+c');

We use the literal method if the expression remains constant and the constructor function when we know the Regex will change, such as from user input.

Syntax

Regex is made up of simple characters and patterns.

Direct match

For example, the following pattern will match with a string only when the exact sequence of "abc" occurs in the same order:

	

/abc/
  • Hi, do you know your abc's?
  • The latest place designs evolved from slabcraft.

There would be no match in the string "Grab crab" because while it contains "ab c", it does not contain the exact substring of "abc".

Further examples:

	

"the" => The fat cat sat on the mat.

"The" => The fat cat sat on the mat

The Full Stop

The full stop (.) is the simplest example of a meta character. This matches any single character. For example, the Regex of .ar means, any character, followed by the letter a, followed by the letter r.

	

".ar" => The car parked in the garage.

The Caret

The caret symbol (^) is used to check if a matching character is the first character of a string.

If we apply ^a, meaning "a" is the starting character to the string "abc", it will match "a". If we apply the Regex of ^b to the same string, it won't match because "b" is not the starting character in the "abc" string.

In this example, the first character in the string must either be an uppercase "T" or a lowercase "t".

	

"^(T|t)" => The car is parked in the garage.

We can extend this further to match the first character of "T" or "t" followed by a lowercase "h" and "e":

	

"^(T|t)he" => The car is parked in the garage.

Then to find all instances of "The" or "the" in the string, we can remove the caret.

	

"(T|t)he" => The car is parked in the garage.

The Question Mark

The meta character (?) makes the preceding character optional. For example, the Regex below will search for an uppercase "T" followed by a lowercase "h" followed by a lowercase "e".

	

"[T]he" => The car is parked in the garage.

"[T]?he" => The car is parked in the garage.

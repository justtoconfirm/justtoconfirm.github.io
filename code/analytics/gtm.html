Google Tag Manager (GTM)

GTM is a Tag Management System (TMS) that allows us to instantly deploy tags and tagging to a website or mobile app.

Introduction to GTM

Here is a simple introduction to GTM:

  • Tags - Tags contain the tracking code that is added to your website. Google provides some templated tags for Google Analytics and other third parties. Custom tags can also be created to suit your needs.
  • Triggers - Formally called rules, they control when a tag should be fired, on certain pages or upon certain user interactions. A tag can have more than one trigger.
  • Variables - Formally called macros, they store information about the page of your website and the element on it. We can reuse variables across any number of rules and tags.

Adding Google Tag Manager (GTM) to a website

The GTM code snippet to add to your website can be found within your Google Tag Manager account dashboard.

It is highly recommended that you place the GTM <script> code into the <head> section of your website code. For better tracking, the higher up the code snippet is, the faster it will be loaded.

Older versions of GTM recommended that the code snippet be placed in the <body> tag, but this would potentially miss uses that had left the page before the <body> tag loaded due to slow page speeds. The lower the tracking codes are placed in the website's code, the later the tracking will load.

Google Tag Manager (GTM) can be used to add Google Analytics code to your website.

If our website is already using Google Tag Manager (GTM) then the gtag.js script is not necessary to be added. Google Analytics (GA) can be added using an analytics tag within GTM.

What is the data layer?

The data layer is a JSON (JavaScript Object Notation) object used by Google Tag Manager (GTM) that temporarily stores the information you need.

The data layer object will be within the GTM script code and will look like this:

	

window.dataLayer = window.dataLayer || [];

The data layer is like a virtual layer of your website which contains various data points that GTM can use within tags, triggers and/or variables.

This data can then be transferred to other tools such as Google Analytics (GA), Google Ads, Facebook Pixel, or any further tools/applications.

When the GTM script/code is placed on a web page it automatically creates the data layer.

  • The website will push data to the data layer.
  • The data layer is then used by GTM to temporarily store this data.
  • We can then push this data to tools such as GA (Google Analytics).

Pushing data to the data layer

The recommended way to push data to the data layer is to use the push() method:

	

dataLayer.push()

It's often useful to prefix the window object to the dataLayer.push() method to push the data to the data layer.

The push to the data layer can be placed above or below the GTM container.

	

window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
	'event': 'new_subscriber'
});

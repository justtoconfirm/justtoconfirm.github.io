Git

Git is a distributed version control system. It is open-source and was originally developed by Linus Torvalds in 2005.

Introduction to Git

One of the most common things to do when working with Git is adding and committing files. The standard way to do this is usually running an add command followed by a commit.

bash -- 85x24

	git add .
	git commit -m "Commit message here"

This works fine, but we can combine these into a single command.

By using the -a flag we are telling Git to add all the files that have been modified and then to commit them. This only adds modified files and not new or deleted files.

bash -- 85x24

	git commit -a -m "Commit message here"

Using the dot (.) will only add all files in the current folder your terminal is open to. Using -A will add all files within the repository.

bash -- 85x24

	git add -A
	git commit -m "Commit message here"

Check the Git version

We can check the version of Git installed.

bash -- 85x24

	git --version

Display the branch

Display the current branch within the terminal.

bash -- 85x24

	git branch

Pull

Pull down the latest changes from a branch.

bash -- 85x24

	git pull

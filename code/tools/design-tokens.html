Design Tokens

Design Tokens are a set of styles - colours, typography, and more, used across the project/product that can be converted into a format for different platforms.

Introduction

Design systems are a great tool to speed up the design and development process. However, there is still a delay between a change in the design and when it applies to the development environment. One way to speed up this process is by creating Design Tokens.

Design Tokens help to keep things consistent between design and development. After the initial setup, these tokens will help push changes from our design tool, such as Figma, to the development environment - completely automating the process.

Design Tokens are sometimes called style properties.

Simply put - Design Tokens are a set of styles - colours, typography, and more, used across the project/product that can be converted into a format for different platforms such as: Android/iOS and web applications.

Design Tokens are entities in the shape of key/value pairs, often in a specific format usually JSON or YAML.

Examples of Design Tokens include:

  • Colour
  • Typography/Font
  • Spacers (Padding/Margin)
  • Shadows
  • Animations

An example:

  • Colour specified in the design system tool e.g. #0473DF
  • Token for this colour e.g. --color-primary
  • This is then applied to the UI component(s).

Figma API

We can hook up directly to the Figma API, if we want to use Design Tokens.

The API documentation can be found on the Figma developer platform pages.

Once an account has been created on Figma, to get data from the Figma API, we need an Auth Token and the ID of our Figma.

Auth Token (API Key)

The Auth Token can be generated under Settings in Figma and then Personal access tokens. Give the token a name, hit Enter and the token will be generated. The token itself will only appear once, so make a note of this somewhere safe/secure.

Figma ID

The ID of our Figma is in the URL. This will normally be the string ?fuid query parameter in the URL. In the example URL below, the ID would be 1234

	

https://www.figma.com/files/recent?fuid=1234

The File Key

To access data from a specific file in Figma using the API, we need to get the File Key. This is the identifier that identifies the file within Figma.

We can retrieve the File Key from the URL of the file. In this case, FILE_KEY_HERE is the File Key:

	

https://www.figma.com/file/FILE_KEY_HERE/sampleFileName

API Endpoint

We can use cURL, a tool for transferring data using various network protocols, to get data from Figma using the API.

This will pass the Personal token to the header to authorise the request.

	

curl -H 'X-FIGMA-TOKEN: PERSONAL_AUTH_TOKEN_HERE' 'https://api.figma.com/v1/files/FILE_KEY_HERE'

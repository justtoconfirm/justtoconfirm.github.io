Vite

JavaScript build tool that has performance benefits over Webpack.

Introduction to Vite

Vite is a new generation JavaScript build tool that aims to improve the developer experience over Webpack when it comes to building JavaScript applications.

Vite leverages two improvements made recently in the JavaScript ecosystem to offer a better developer experience. The availability of ES Modules (ESM) in the browser and compile-to-native bundler tools like esbuild.

The availability of ES Modules (ESM) in the browser allow you to run a JavaScript application within the browser without having to bundle them together. Vite serves a piece of our application code using ES Modules when the browser requests it.

When we start the development build, Vite will start by dividing the JavaScript modules into two categories:

  • dependency modules
  • application modules

The dependency modules are JavaScript modules that we have imported from the node_modules folder. These modules will be processed and bundled using esbuild, a JavaScript bundler written in Go that performs 10-100x faster than Webpack.

The application modules are modules that we write for our application which often involves library-specific extensions like .jsx, .vue or .scss files.

Vite supports the following languages and requires Node version 12.2.0 or greater:

  • JavaScript
  • TypeScript

Vite uses esbuild which is written in Go and pre-bundles dependencies 10-100x faster than JavaScript-based bundlers.

While a bundler-based workflow like Webpack will have to process the entire JavaScript modules before a single browser request, Vite only processes the dependency modules before a single browser request.

The application modules will be transformed and served by Vite when they are required by the application.

Vite ships with a pre-configured build command that bundles the application using Rollup. By using Rollup, Vite ensures performance optimisation like tree-shaking, lazy-loading, and chunk splitting are implemented into the production build.

Vite & React

Create React App (CRA) uses Webpack to bundle to code. Webpack bundles the entire code, so if our codebase grows, the development server may take longer to start. While Webpack is still the default JavaScript build tool for many popular applications, like Create React App (CRA), Vite is becoming more popular.

Instead of using CRA, we could migrate to Vite. Vite is a newer frontend tool for building web applications.

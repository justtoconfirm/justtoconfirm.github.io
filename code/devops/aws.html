Amazon Web Services (AWS)

Amazon Web Services (AWS) is a set of web services provided by Amazon. AWS is a cloud computing platform. The first product, S3 (Amazon Simple Storage Service), was released in 2006.

  • AWS is the largest of the cloud providers.
  • AWS competence and experience is popular in the job market.

Cloud computing is pay-as-you-go model for delivering IT resources. We only pay for what we use.

AWS offers a free tier for one-year, allowing us to explore and try services AWS offers.

AWS Lambda

AWS Lambda is a service which computes the code without any server and is said to be serverless.

We can run code without managing infrastructure. Simply write and upload the code. To work with AWS Lambda, we need to login to AWS. Lambda will only work with AWS services.

Supported languages:

  • Node
  • Java
  • Python
  • C#
  • Go

For writing code, we can use the AWS online editor, we can use the following:

  • Visual Studio IDE
  • Eclipse IDE

We also have AWS-cli to create and deploy code.

It is not suitable for small projects and since AWS Lambda relies completely on AWS for the infrastructure, we cannot install any additional software if our code demands it.

To get working with AWS Lambda, we just have to push the code in AWS Lambda service. All other tasks, infrastructure and resources, such as operating system, maintenance of the server, code monitoring, logs and security, is taken care of by AWS.

The code is executed based on the response of events in AWS services such as adding/removing files in an S3 (Amazon Simple Storage Service) bucket, updating Amazon DynamoDB tables, HTTP requests from Amazon API Gateway, and much more.

Examples of when code could be executed:

  • User uploads files to an S3 (Amazon Simple Storage Service) bucket.
  • HTTP GET/POST endpoint URL is hit.
  • Data is added/updated/deleted in Dynamo DB tables.
  • Push notifications.
  • GET/POST calls to an API Gateway.
  • Hosting of website.
  • Sending of Email.
  • Log history in CloudTrail.

AWS Lambda executes your code only when needed and scales automatically, from a few requests per day to thousands per second. We only pay for the compute time we consume/use. There is no charge when our code is not running.

Use cases of AWS Lambda

  • S3 Object : Amazon S3 passes the event details to AWS Lambda when there is any file upload in S3. The details of the file upload, movement, or deletion of a file is passed to AWS Lambda. The code in AWS Lambda can take the necessary step for when it receives the event details. For example, creating a thumbnail of the image inserted into S3.
  • DynamoDB and AWS Lambda : DynamoDB can trigger AWS Lambda when there is data added, updated or deleted from the table. AWS Lambda event has all the details of the AWS DynamoDB table.
  • API Gateway : Can trigger AWS Lambda on GET/POST methods. We can create a from and share details with the API Gateway endpoint and use it with AWS Lambda for further processing.

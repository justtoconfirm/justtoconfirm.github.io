Definitions

Technology Terms, Abbreviations and Terminology explained.

A

  • Analytics - analytics is the collection, reporting, and analysis of website data.
  • API - stands for Application Program Interface, which can be defined as a set of methods of communication between various software components. In other words, an API allows software to communicate.
  • Art direction - change the image displayed to suit different image display sizes.

B

  • Blue/Green deployment - Ensures we have two production environments, as identical as possible. At any time one of them is live, the other is prepared with a new release of our code. Once the changes have been tested, the router is switched so that all incoming requests go to the other environment and the first environment is now idle. This also allows rapid rollback. If anything goes wrong, we can switch the router back to the initial environment.

C

  • Code Coverage - testing to determine how much code is being tested. For RSA, our code coverage should be greater than 85.0%.
    Code Coverage = Number of lines of code exercised/Total number of lines of code * 100% (as a Percentage).
  • Code Smell - also known as bad smell, in code, refers to any symptom in the source code of a program that possibly indicates a deeper problem. It is a surface indication that usually corresponds to a deeper problem in the system.
  • Compiler - A compiler converts codes into machine code that the processor can execute. An example could be a C++ compiler.
  • CRUD - Create, Read, Update and Delete.
  • CSS - Cascading Style Sheet

D

  • Development Environment - developers are free to change and push changes locally any time in order to quickly test their code and integrate with other changes without fear of breaking anything. This is usually connected to a test database and/or services.

E

F

  • Firefox (FF) - Web Browser created by Mozilla, formally Netscape.
  • FTP - File Transfer Protocol

G

H

  • HTML - Hypertext Markup Language
  • HTTP - Hypertext Transfer Protocol
  • HTTPS - Hypertext Transfer Protocol Secure

I

  • IDE - Integrated Development Environment is a software application used for development and normally consists of at least a source code editor.
  • Internet Explorer (IE) - Web Browser created by Microsoft and replaced by Edge.
  • Interpreter - An interpreter goes through the program line by line and executes each command.

J

  • JS - JavaScript
  • JSF - JavaServer Faces
  • JSX - JavaScript XML

K

  • KPI - Key Performance Indicators

L

M

  • Minification - the process of reducing code size to reduce the size of our files. JavaScript and CSS files are often minfied to improve page speed/performance.
  • Mobile first - Mobile first is the consideration to think about mobile first and work up to the large desktop screen size.

N

  • NVM - Node Version Manager, allows developers to switch Node versions within projects.

O

  • OOP - Object-Oriented Progamming

P

  • PHP - PHP Hypertext Preprocessor
  • PR - stands for Pull Request, a review of code changes made by the developers that is reviewed by other developers to check for errors and to add comments on improvements that could be made before the code change is merged.
  • Pre-production Environment - usually built before applications go into production. This simulates the actual production environment as closely as possible so we can test the application before it goes into production.
  • Production Environment - the live environment.
  • Programming Language - a programming language is just a set of predefined rules. To make them useful, we need something to understand those rules. Those things are compilers & interpreters.

Q

  • QA - stands for Quality Assurance and is a way of preventing mistakes and defects during software testing.

R

  • Responsive Web Design (RWD) - originally defined by Ethan Marcotte, is the approach in design and development to respond to a user's behaviour and environment based on screen size, platform, and orientation. Responsive design is an approach to creating websites in which the layout adapts to fit the device's screen.

S

  • Sass - Syntactically Awesome Style Sheets
  • SCSS - Sassy CSS
  • Staging Environment - similar to UAT but setup with a copy of production data, sometimes anonymised. The primary focus of the Staging environment is to ensure that the application will work in production the same way it worked in UAT. Staging environments is a testing environment dedicated to QA or software testers to review changes. This environment is used to typically test the whole app.
  • SQL - SQL (pronounced "ess-que-el" or "see-quel") stands for Structured Query Language and is used to communicate with databases. SQL is a database management language for Relational Database Management Systems (RDMS) like MySQL, MS Access, Oracle, Postgres and SQL Server.

T

  • TS - TypeScript

U

  • UAT - stands for User Acceptance Testing and is the environment in which user acceptance testing is performed. It should contain a "as good as possible" copy of the production environment. The UAT environment is connected to UAT databases with a copy of production data and is used by the QA team and users to validate changes before they reach production.
  • UI - User Interface
  • UX - User Experience

V

W

  • www - World Wide Web

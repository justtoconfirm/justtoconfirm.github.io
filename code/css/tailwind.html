Tailwind CSS

Utility-first CSS framework that uses classes applied directly to the markup.

Introduction to Tailwind

Tailwind CSS is performance focused with the aim to reduce the size of the CSS file produced by only generating the CSS you are actually using in your project.

Combined with minification and network compression, this usually leads to CSS files that are less than 10kb in size. Netflix uses Tailwind and the website delivers only 6.5kb of CSS over the network. Small CSS stylesheets can be downloaded once and cached until you redeploy your site without the need to use complex solutions such as code-splitting.

Tool such as cssnano and Brotli can be used to minify your CSS.

Browser support

  • Chrome - latest stable version
  • Firefox - latest stable version
  • Safari - latest stable version

No version of IE, including IE 11, is supported. If using Tailwind CLI, vendor prefixes will be added automatically.

Syntax support

Tailwind CSS uses a lot of custom CSS at-rules like @tailwind, @apply, and @screen that can trigger warnings or errors in some IDE/Code editors. In VSCode, the PostCSS Language Support extension can be used that works well with Tailwind CSS.

Install

The Tailwind documentation can be found here: https://tailwindcss.com/docs/installation

Add the CDN script tag to the <head> section of your HTML file before the closing </head> tag. This is best used for development and not production builds.

	

<script src="https://cdn.tailwindcss.com"></script>

Much better to install Tailwind via npm. This will install the Tailwind CLI tool.

bash -- 85x24

	npm install tailwindcss --save-dev
    npx tailwindcss init

If you want to setup Tailwind within a create-react-app React project, the installationg instructons are here: https://tailwindcss.com/docs/guides/create-react-app

We will need to create a tailwind.config.js file in the root of our project. If the init command was run, this will create the config file.

We need to add the paths to all of our template files with the config file.

	

module.exports = {
	content: ["./src/**/*.{html, js}"],
	theme: {
		extend: {},
	},
	plugins: [],
}

Add the @tailwind directive for each of Tailwind's layers to your main CSS file. In our src directory we may have a CSS file named styles.css

	

@tailwind base;
@tailwind components;
@tailwind utilities;

Run the CLI tool to scan your template files for classes and build your CSS. This helps us to reduce the stylesheet size used in production as only those classes used in our markup will be included. This terminal command will also watch for changes and update the stylesheet if changes are made in our markup templates.

bash -- 85x24

	npx tailwindcss -i ./src/styles.css -o ./dist/output.css --watch

If you're using the Tailwind CLI, you can minify your CSS by adding the --minify flag.

bash -- 85x24

	npx tailwindcss -i ./src/styles.css -o ./dist/output.css --minify

We can now add our compiled CSS stylesheet into the <head> section of our HTML to start using Tailwind.

	

<link rel="stylesheet" href="../dist/styles.css">

Our file structure will look like this:

	

project /
		|_dist /
			|_output.css
		|_src /
			|_index.html
			|_styles.css
		|_node_modules /
		package-lock.json
		package.json
		tailwind.config.js

TIP: use the ! (exclamation) and press the ENTER key to create HTML boilerplate code in VSCode.

Adding Utility Classes

Tailwind CSS uses utility classes added to markup to apply styling.

In VSCode we can install the Tailwind CSS intelliSense extension to assist with coding using Tailwind CSS.

Colours

Tailwind includes a number of colours out-of-the-box.

https://tailwindcss.com/docs/customizing-colors

Setting the background colour:

	

<div class="bg-slate-100"> ... </div>

Setting the foreground (text) colour:

	

<div class="text-slate-100"> ... </div>

Our colours can be customised within our tailwind.config.js file.

	

module.exports = {
	content: ["./src/**/*.{html, js}"],
	theme: {
		colors: {
			// Configure your own colour palette here
			transparent: 'transparent',
			current: 'currentColor',
			'white': '#fff',
			'black': '#000',
			'tahiti': {
				100: '#cffafe',
				200: '#a5f3fc',
				300: '#67e8f9',
				400: '#22d3ee',
				500: '#06b6d4',
				600: '#0891b2',
				700: '#0e7490',
				800: '#155e75',
				900: '#164e63',
			},
		}
		extend: {},
	},
	plugins: [],
}

We can use existing colours also by importing tailwindcss/colors into the config file and choosing the colours you want to use.

	

const colors = require('tailwindcss/colors');

module.exports = {
	content: ["./src/**/*.{html, js}"],
	theme: {
		colors: {
			// Configure your own colour palette here
			transparent: 'transparent',
			current: 'currentColor',
			white: colors.white,
			black: colors.black,
			gray: colors.gray,
		}
		extend: {},
	},
	plugins: [],
}

If a one-off custom colour is needed, we can use the arbitrary value notation.

	

<div class="bg-[#1da1fs] text-white"> ... </div>

Theme Configuration

Within the config file, we can define our project's colour palette, fonts, breakpoints, border radius, and much more to create our project theme.

	

module.exports = {
	theme: {
		screens: {
			sm: '480px',
			md: '768px',
			lg: '976px',
			xl: '1440px',
		},
		colors: {
			'blue': '#1fb6ff',
			'purple': '#7e5bef',
			'pink': '#ff49db',
			'orange': '#ff7849',
			'green': '#13ce66',
			'yellow': '#ffc82c',
			'gray-dark': '#273444',
			'gray': '#8492a6',
			'gray-light': '#d3dce6',
		},
		fontFamily: {
			sans: ['Graphik', 'sans-serif'],
			serif: ['Merriweather', 'serif'],
		},
		extend: {
			spacing: {
				'128': '32rem',
				'144': '36rem',
			},
			borderRadius: {
				'4xl': '2rem',
			}
		}
	}
}

Plugins

Plugins allow the extension of Tailwind to add new features that have yet to be added to the core.

Any plugin required can be installed with npm and then added to the config file.

	

module.exports = {
	// ...
	plugins: [
		require('@tailwindcss/typography'),
		require('@tailwindcss/forms'),
		require('@tailwindcss/line-clamp'),
		require('@tailwindcss/aspect-ratio'),
	],
}

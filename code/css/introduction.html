Introduction to CSS (Cascading Style Sheets) to style web pages and web applications.

Introduction to CSS

If we use a CSS property or value that a browser does not understand, the browser will ignore it. If the browser does not understand that line of CSS, it just skips it and gets on with the next thing it does understand.

Feature Queries

We can detect support for new CSS properties using feature queries.

Testing for support is the simplest case, we can use @supports and then test for a CSS property and value. The content inside the Feature Query will only run if the browser responds with true.

	

@supports (display: grid) and (shape-outside: circle()) {
	.item {
		/* CSS from browsers which support the grid layout and CSS shapes */
	}
}

We can also ask the browser if it does not support a feature. In this case, the code inside the feature query will only run if the browser indicates it has no support.

	

@supports not (display: grid) {
	.item {
		/* CSS from browsers which do not support the grid layout */
	}
}

CSS feature queries are not supported in Internet Explorer (IE), but are supported in most modern browsers.

CSS: content-visibility

Replace the use of display: none; in your CSS code.

The content-visibility property is new to CSS. Firsty, using display: none; in our CSS will hide an element and its inner properties. However, those elements will still be downloaded and rendered by the browsers, despite being invisible. This can take up time and CPU that could be used elsewhere.

With content-visibility: hidden;, Chromium browsers will spend the time (main thread) on other tasks, leading to better First Contentful Paint (FCP) metrics when considering page speed performance. Moreover, images that are set with this property won't also be downloaded.

	

.element {
	content-visibility: hidden;
	display: none;
}

This property will only work in Chrome, Edge and Opera. IE, Safari and Firefox do not support this property.

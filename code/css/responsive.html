Responsive Grid & Frameworks

Guide to building a Responsive Grid/Framework for responsive, mobile-first layouts.

Introduction

Many web pages are based on a grid-view, which means that the page is divided into columns. Using a grid-view is very helpful when building web pages. It makes it easier to place elements on the page.

A responsive grid-view often has 12 columns with a total width of 100%. This will shrink and expand as we resize the browser window.

Building a simple responsive grid

We must first ensure that all HTML elements have the box-sizing property set to border-box. This makes sure that the padding and border are included in the total width and height of the elements.

	

*,
*::before,
*::after {
	box-sizing: border-box;
}

We must then calculate the percentage for one column: 100% / 12 columns = 8.33%

We can then make one class for each of the 12 columns:

	

.col-1 {width: 8.33%;}
.col-2 {width: 16.66%;}
.col-3 {width: 25%;}
.col-4 {width: 33.33%;}
.col-5 {width: 41.66%;}
.col-6 {width: 50%;}
.col-7 {width: 58.33%;}
.col-8 {width: 66.66%;}
.col-9 {width: 75%;}
.col-10 {width: 83.33%;}
.col-11 {width: 91.66%;}
.col-12 {width: 100%;}

Many developers jump to a web design framework for their responsive grid needs. We can code a responsive grid ourselves.

Let's start with 12 "cards":

	

<div class="cards">
	<div class="card">ONE</div>
	<div class="card">TWO</div>
	<div class="card">THREE</div>
	<div class="card">FOUR</div>
	<div class="card">FIVE</div>
	<div class="card">SIX</div>
	<div class="card">SEVEN</div>
	<div class="card">EIGHT</div>
	<div class="card">NINE</div>
	<div class="card">TEN</div>
	<div class="card">ELEVEN</div>
	<div class="card">TWELVE</div>
</div>

We are coding mobile-first, so we assume the user is using a small screen and the cards will take up a whole row each (one column). We can make the cards responsive for large screen sizes using @media queries:

	

@media (min-width: 600px) {
	.cards { grid-template-columns: repeat(2, 1fr); }
}

If the screen is larger than 900px, we can make it 3 columns:

	

@media (min-width: 900px) {
	.cards { grid-template-columns: repeat(3, 1fr); }
}

Once the parent has display: grid, all the magic comes from adjusting grid-template-columns: repeat(NUM_COLUMNS, 1fr) for different media breakpoints: @media (min-width: SOME_WIDTH).

In some cases, we maybe able to avoid media queries by using minmax().

Add the following line of code to the .cards selector in the CSS and remove the @media queries:

	

.cards {
	grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
}

The columns will automatically break when each one gets below 300px. We no longer need the @media queries in our CSS code.

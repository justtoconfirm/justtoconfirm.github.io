Install & Setup React

React is an open-source, front-end JavaScript library for building user interfaces and components.

Requirements

We will need to have the following installed on our machine to install and run React:

  • Node 14.0.0 or later
  • npm 5.6 or later

You can also install nvm (Node Version Manager) or nvm-windows for Windows machines to switch Node versions between projects.

Install React

React can be installed using create-react-app and this will setup the development environment but this will not handle backend logic or databases.

This will create the frontend build pipeline using Babel and Webpack.

We use npx to install React. This is a package runner tool that comes with npm version 5.2 or later and will install the latest version of React.

bash -- 85x24

	npx create-react-app APP-NAME-HERE

Install React - Setup TypeScript

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript.

To start a new create-react-app project with TypeScript, you can run:

bash -- 85x24

	npx create-react-app APP-NAME-HERE --template typescript

	# or

	yarn create react-app APP-NAME-HERE --template typescript

To add TypeScript to an existing create-react-app project,first install it:

bash -- 85x24

	npm install --save typescript @types/node @types/react @types/react-dom @types/jest

	# or

	yarn add typescript @types/node @types/react @types/react-dom @types/jest

Next, rename any JavaScript file to be a TypeScript file - for example, src/index.js to src/index.tsx - then restart the development server.

  • @types/react - type definitions for React
  • @types/react-dom - type definitions for the React DOM library

What is TypeScript?

TypeScript is a programming language developed and maintained by Microsoft. It is a superset of JavaScript. It helps you to catch mistakes early through a type system and to make development more efficient.

  • TypeScript is known as an Object-Oriented Programming Language (OOPL), JavaScript is a scripting language.
  • TypeScript has a feature known as Static typing but JavaScript does not have this feature.
  • TypeScript gives support for modules, JavaScript does not support modules.

TypeScript helps spot common errors as you type code.

The tsconfig.json file contains configuration for the typescript compiler.

TypeScript files have a file extension of .tsx

Imports

We need to import the React library to make JSX compile.

TypeScript supports the ES module notation. You can import React with:

	

import * as React from 'react';

A cleaner way to write an import is like this:

	

import React from 'react';

We can also import type definitions and React features in a destructured way:

	

import React { Component, FunctionComponent } from 'react';

Start the development server

Once React has been installed we can use npm to start the local development server. This will run on port 3000 by default on the localhost - http://localhost:3000/

bash -- 85x24

	cd APP-NAME-HERE
	npm start

Code examples

Here's the src/App.tsx code:

	

import React from 'react';
import logo from './logo.svg';
import './App.css';

function App() {
	return (
		<div className="App">
			<header className="App-header">
				<img src={logo} className="App-logo" alt="logo" />
				<p>
					Edit <code>src/App.js</code> and save to reload.
				</p>
				<a
					className="App-link"
					href="https://reactjs.org"
					target="_blank"
					rel="noopener noreferrer"
				>
					Learn React
				</a>
			</header>
		</div>
	);
}

export default App;

Here's the src/index.tsx code:

	

import React from 'react';
import ReactDOM from 'react-dom';
import './index.css';
import App from './App';
import reportWebVitals from './reportWebVitals';

ReactDOM.render(
	<React.StrictMode>
		<App />
	</React.StrictMode>,
	document.getElementById('root')
);

// If you want to start measuring performance in your app, pass a function
// to log results (for example: reportWebVitals(console.log))
// or send to an analytics endpoint. Learn more: https://bit.ly/CRA-vitals
reportWebVitals();

The public/index.html file is where the app renders using the root element.

	

<div id="root"></div>

The root DOM node is managed by React DOM. Applications built with React usually have a single root DOM node. We use the ReactDOM.render() method to render the React application into the root DOM.

Hello World

The smallest React example can be created in the src/index.tsx file and it looks like this:

	

import React from 'react';
import ReactDOM from 'react-dom';

ReactDOM.render(
	<h1>Hello World</h1>,
	document.getElementById('root')
);

Here's the updated code to display a simple Hello World message on screen.

Here's the src/App.tsx file:

	

import React from 'react';

const App = () => {
	return (

		<h1>Hello World</h1>

	);
}

export default App;

Here's the src/index.tsx file:

	

import React from 'react';
import { render } from 'react-dom';
import App from './App';

render(
	<React.StrictMode>
		<App />
	</React.StrictMode>,
	document.getElementById('root')
);

The public/index.html file has not changed.

Embedding Expressions in JSX

In the example below, we declare a variable called name and then use it inside JSX within src/index.tsx by wrapping it in curly braces:

	

import React from 'react';
import ReactDOM from 'react-dom';

const name = 'Dan';

ReactDOM.render(
	<h1>Hello {name}</h1>,
	document.getElementById('root')
);

By default, React DOM escapes any values embedded in JSX before rendering. Everything is converted to a string before being rendered helping to prevent XSS (cross-site-scripting) attacks.

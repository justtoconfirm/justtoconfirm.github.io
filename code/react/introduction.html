Install & Setup React

React is an open-source, front-end JavaScript library for building user interfaces and components.

What is React?

React is an open-source, front-end JavaScript library for building user interfaces using components. It is maintained by Meta (formally Facebook).

JSX (JavaScript XML)

React uses an XML-like syntax called JSX (JavaScript XML).

JSX is a syntax extension for JavaScript tht was written to be used with React. JSX is not valid JavaScript so browsers cannot read it. If a file contains JSX then the code will need to be compiled. A JSX compiler will translate the JSX syntax into regular JavaScript code. Often a tool such as Babel will be used to do this.

When coding in JavaScript, it's always best to include semicolons (;) at the end of statements as to not always rely on Automatic Semicolon Insertion (ASI).

React DevTools

React DevTools (also known as React Developer Tools), is a browser extension for Chrome that will allow working with React to be much easier. Once installed, you will see a tab for React when you open your browser's developer tool inspector. The Elements tab will show the actual DOM (Document Object Model) output.

Requirements

We will need to have the following tools installed on our machine to install and run React:

  • Node v14.0.0 or later
  • npm v5.6 or later

Setup Node Version Manager (NVM)

You can also install nvm (Node Version Manager) or nvm-windows for Windows machines to switch Node versions between projects.

Install React

React can be installed using create-react-app. This will setup the development environment and create the frontend build pipeline using Webpack and Babel.

We use npx to install React. This is a package runner tool. This will install npm and the latest version of React.

bash -- 85x24

	npx create-react-app <APP-NAME-HERE>

Start the development server

Once React has been installed we can use npm to start the local development server. This will run on port 3000 by default as localhost - http://localhost:3000/

In the terminal, navigate to the root of the project folder and run the following script:

bash -- 85x24

	npm start

Use this command to start a JavaScript or TypeScript React application.

Import React

We need to import the React library to make JSX compile.

	

import React from 'react';

We can also import type definitions and React features in a destructured way:

	

import React { Component, FunctionComponent } from 'react';

TypeScript supports the ES module notation. We can import React with:

	

import * as React from 'react';

Hello World

The smallest, and simplest, React example can be created in the src/index.js file to render Hello World on the page:

	

import React from 'react';
import ReactDOM from 'react-dom';

ReactDOM.render(
	<h1>Hello World</h1>,
	document.getElementById('root')
);

Within the public/index.html file, you will find the root element, also called the root DOM node, managed by React DOM. Applications built with React usually have a single root DOM node. We use the render() method of ReactDOM to render the React application into the root element.

	

<div id="root"></div>

Embedding Expressions

In the example below we are extending our Hello World example to output the value of a variable, called name, inside our JSX code by wrapping it in curly braces. Open the src/index.js file and update the code:

	

import React from 'react';
import ReactDOM from 'react-dom';

const name = 'Dan';

ReactDOM.render(
	<h1>Hello {name}</h1>,
	document.getElementById('root')
);

By default, React DOM escapes any values embedded in JSX before rendering. Eveything is converted into a string before being rendered helping to prevent XSS (cross-site-scripting) attacks.

Building Components

React applications are built using Reusable components. Components let you split the UI (User Interface) into independent, reusable pieces of code. Conceptually, components are like JavaScript functions. They accept inputs called props (properties) and return elements onto the screen.

Components will have the file extension of .js or .jsx and should start with an uppercase letter for the name.

The simplest way to create a React component is to write a JavaScript function. Functional components in React are simple and purely functional, often written as JavaScript functions.

Within our components, we have to import React to make the JSX code compile.

Here is a simple React component template written as a function. This will be saved in the root of our project folder as src/Welcome.js.

	

import React from 'react';

function Welcome() {
	return <h1>Hello World</h1>;
}

export default Welcome;

We can also write this function using the ES6 arrow function syntax:

	

import React from 'react';

const Welcome = () => {
	return <h1>Hello World</h1>;
}

export default Welcome;

Within the src/index.js file, we can import our Welcome component:

	

import React from 'react';
import ReactDOM from 'react-dom';
import Welcome from './Welcome';

ReactDOM.render(
	<Welcome />,
	document.getElementById('root')
);

We can amend this code further:

	

import React from 'react';
import { render } from 'react-dom';
import Welcome from './Welcome';

render(
	<Welcome />,
	document.getElementById('root')
);

Props

Props (properties) allow us to pass input (data) into our React components. We can demonstrate this by extending our Welcome component - src/Welcome.js - to have a prop called name that will allow us to pass in a name of a user when we render the component.

	

import React from 'react';

const Welcome = (props) => {
	return <h1>Hello {props.name}</h1>;
}

export default Welcome;

We can pass a value into this prop and into the component when we render the component within our application. Update the src/index.js file like so:

	

import React from 'react';
import ReactDOM from 'react-dom';
import Welcome from './Welcome';

ReactDOM.render(
	<Welcome name="Dan" />,
	document.getElementById('root')
);

As we only have the one prop inside of our src/Welcome.js component, we can remove the braces like so:

	

import React from 'react';

const Welcome = props => {
	return <h1>Hello {props.name}</h1>;
}

export default Welcome;

We may also want to destructure the props so we don't have to use props.name, we can just use name when we want to output the value:

	

import React from 'react';

const Welcome = ({ name }) => {
	return <h1>Hello {name}</h1>;
}

export default Welcome;

A default value can also be set inside of our component. If no value is passed into the prop when our component is rendered, the default value will be used. For this example, we will set the default prop value to be Pearl:

	

import React from 'react';

const Welcome = ({ name = 'Pearl' }) => {
	return <h1>Hello {name}</h1>;
}

export default Welcome;

Components can often have more than one prop. Let's update our Welcome component to have another prop:

	

import React from 'react';

const Welcome = ({ name = 'Pearl', age }) => {
	return <h1>Hello {name}, you are {age} years old.</h1>;
}

export default Welcome;

We can also pass props into our component using an object:

	

import React from 'react';

const Welcome = props => {

	const {
		name = "Pearl",
		age
	} = props;

	return <h1>Hello {name}, you are {age} years old.</h1>;
}

export default Welcome;

Now when we import and use the component, we can pass in more than one value to be rendered.

	

import React from 'react';
import ReactDOM from 'react-dom';
import Welcome from './Welcome';

ReactDOM.render(
	<Welcome name="Dan" age="34" />,
	document.getElementById('root')
);

Adding Attributes

We can use the className prop to apply a class attribute to a React component. We cannot use class as it's considered a keyword.

Here, we add a class of title to our component using the className prop:

	

import React from 'react';

const Welcome = ({ name = 'Pearl', age }) => {
	return (
		<h1 className="title">Hello {name}, you are {age} years old.</h1>;
	);
}

export default Welcome;

If we want to add the for attribute to a component, such as a label, we can use the htmlFor prop:

	

<label htmlFor="name">Name</label>

Composing Components

Components can refer to other components in their output. This allows us to use the same component multiple times.

In this example, we will extend the code inside of src/index.js to include a new component called App which is basically just a function, and this will render the Welcome component twice.

	

import React from 'react';
import ReactDOM from 'react-dom';
import Welcome from './Welcome';

const App = () => {
	return (
		<div>
			<Welcome name="Dan" age="36" />
			<Welcome name="Archie" age="25" />
		</div>
	);
}

ReactDOM.render(
	<App />,
	document.getElementById('root')
);

Fragments

To avoid any JSX syntax errors, if we are composing components, we need to wrap a parent element around the components used. In the earlier example, this was a <div> element, but we can use Fragments (<> & </>) that will prevent adding anything extra to the DOM.

	

import React from 'react';
import ReactDOM from 'react-dom';
import Welcome from './Welcome';

const App = () => {
	return (
		<>
			<Welcome name="Dan" age="36" />
			<Welcome name="Archie" age="25" />
		</>
	);
}

ReactDOM.render(
	<App />,
	document.getElementById('root')
);

Separating Components

It's often best to separate components into individual files to make it easier to manage the code in your project.

Move the code for the App component into a file called src/App.js

	

import React from 'react';
import Welcome from './Welcome';

const App = () => {
	return (
		<>
			<Welcome name="Dan" age="36" />
			<Welcome name="Archie" age="25" />
		</>
	);
}

export default App;

The App component will still import the src/Welcome.js component as before.

We can the import the App component into the src/index.js file.

	

import React from 'react';
import ReactDOM from 'react-dom';
import App from './App';

ReactDOM.render(
	<App />,
	document.getElementById('root')
);

In a basic React application file structure, our files would look like this inside the src directory:

	

project /
	|__src /
		App.js
		index.js
		Welcome.js

It is often better to also move components into individual directories to make the folder structure cleaner. We have now renamed the Welcome component to use index.js

	

project /
	|__src /
		|__components /
			|__Welcome /
				index.js
		App.js
		index.js

Then update the path inside the App component following this change.

	

import React from 'react';
import Welcome from './components/Welcome';

const App = () => {
	return (
		<>
			<Welcome name="Dan" age="36" />
			<Welcome name="Archie" age="25" />
		</>
	);
}

export default App;

TypeScript

TypeScript is a superset of JavaScript developed and maintained by Microsoft. TypeScript helps you to catch mistakes, or errors as you type code, early through a type system and this helps to make development more efficient.

TypeScript projects may feature a tsconfig.json file containing the configuration for the typescript compiler.

TypeScript files have the file extension of .tsx

  • TypeScipt is known as an Object-Oriented Programming Language (OOPL). JavaScript is a scripting language.
  • TypeScript has a feature known as Static typing but JavaScript does not.
  • TypeScript gives support for modules, JavaScript does not support modules.

Setup TypeScript with React

To setup TypeScript with create-react-app:

bash -- 85x24

	npx create-react-app <APP-NAME-HERE> --template typescript

	# OR

	yarn create react-app <APP-NAME-HERE> --template typescript

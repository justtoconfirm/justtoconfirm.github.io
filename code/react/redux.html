Redux

Redux is a state container for JavaScript and TypeScript applications.

What is Redux?

Redux is a state container for JavaScript and TypeScript applications. React components have state at component level. With Redux, the entire state of our application is managed in a single "global" object.

We can setup Redux with our React application, or we can use the Redux Toolkit, which avoids the boilerplate code that's needed when installing Redux. A small React application may not need Redux to manage state and may instead benefit from using the Context API to manage state across our application.

Common concerns with Redux that Redux Toolkit helps to solve:

  • Configuration of a Redux store is too complicated.
  • Packages need to be added to get Redux to do anything useful.
  • Redux requires lots of boilerplate code.

The benefits of using React Toolkit:

  • Easier to setup (less dependencies).
  • Reduction of boilerplate code.
  • Dependencies - Redux Thunk (redux-thunk), Redux DevTools (redux-devtools-extension) are built in, so we no longer need to install as packages.
  • Ability to use direct state mutation sinch React Toolkit uses the Immer library under the hood.

Redux Toolkit is the recommended method to setup Redux within a React application. It is intended to be the standard way to write Redux logic.

Redux Toolkit (also known as "RTK" for short) includes the Redux core, as well as other key packages that are essential for building Redux applications, such as Redux Thunk.

If we want to test our application with an API, we can use JSON Placeholder API.

Terminology

Redux has a number of key terms and definitions to become familiar with:

  • Actions
  • Reducers
  • Store
  • Dispatch
  • Connect

Actions

An action sends data from your application to the Redux store. An action is an object with two properties, type and payload (optional). The type is generally an uppercase string that describes the action. The payload is additional data that maybe passed.

Reducers

A reducer is a function that takes two paramaters, state and action. A reducer is immutable, meaning it cannot be changed, and always returns a copy of the entire state. A reducer will typically consist of a switch statement that does through all the possible action types.

Store

A store holds the whole state tree of our application. The only way to change the state inside the store is to dispatch an action on it. A store is just an object with a few methods. To create the store, we use the createStore method. With React Toolkit we only need to use configureStore instead of createStore to create the Store object.

Dispatch

Dispatch is a method of the Redux store. This is the only way to trigger a state change. With React Redux, our components can never access the store directly, connect() does it for us. We call store.dispatch to dispatch an action. Dispatch is a method available on the store object that accepts an object which is used to update the Redux state.

connect()

The connect() function is the typical way to connect React and Redux. A connected component is sometimes referred to as a container.

Getting Started with React

For ease of getting started with React, use create-react-app:

bash -- 85x24

	npx create-react-app <APP-NAME-HERE>

	cd <APP-NAME-HERE>

Install React & Redux

We can first create a React application using create-react-app and install Redux Toolkit when creating a new application:

bash -- 85x24

	# Redux Toolkit and JavaScript 
	npx create-react-app <APP-NAME-HERE> --template redux

	# Redux Toolkit and TypeScript
	npx create-react-app <APP-NAME-HERE> --template redux-typescript

If we have an existing React application and want to add Redux Toolkit, we can use npm or yarn:

bash -- 85x24

	# npm  
	npm install @reduxjs/toolkit react-redux

	# yarn
	yarn add @reduxjs/toolkit react-redux

The @reduxjs/toolkit package wraps around the core redux package and contains API methods and common dependencies that are useful for building a Redux application.

Should we not want to use Redux Toolkit in our React application, we can install the dependencies as packages using npm:

bash -- 85x24

	npm install redux react-redux redux-thunk redux-devtools-extension
  • Redux - the core library
  • React Redux - React bindings for Redux, connects React and Redux
  • Redux Thunk - Asynchronous middleware for Redux
  • Redux DevTools - connects the Redux app to Redux DevTools

Create the Redux Store

The store is the object that holds the application's "global" state. This creates a Redux store and automatically configures the Redux DevTools extension so we can inspect the store during development.

configureStore() wraps createStore to provide configuration options. This enables use of the Redux DevTools extension and we can add whatever middleware we supply, includes redux-thunk by default.

We can start by creating an empty Redux store and exporting it.

Create a file named src/app/store.js and import the configureStore API from Redux Toolkit.

	

// src/app/store.js (Redux Toolkit)

import { configureStore } from '@reduxjs/toolkit'

export default configureStore({
	reducer: {}
});

In TypeScript this file would be named src/app/store.tsx.

The code below is used to create the store if we're not using React Toolkit. Here we're using the createStore method from Redux:

	

// src/app/store.js

import { createStore } from 'redux'

const store = createStore()

export default store

Connect Redux Store to React App

Once the store is created, we can make it available to our React components. By putting a React-Redux <Provider> around our application in the src/index.js file. We must import the Redux store we created and pass the store as a prop.

	

// src/index.js

import React from 'react';
import ReactDOM from 'react-dom;
import App from './App';
import { store } from './app/store';
import { Provider } from 'react-redux';

ReactDOM.render(
	<Provider store={store}>
		<App />
	</Provider>,
	document.getElementById('root')
);

Slices

Slices automatically generates reducers, action types, and action creators in a single folder named slices. This removes the need to deal with reducers and actions as separate files.

We no longer need to return the entire state when using a slice, we just mutate (change) the state within our slice.

To create a slice we import the createSlice API from Redux Toolkit.

	

import { createSlice } from '@reduxjs/toolkit'

The createSlice() method helps us to create a slice of the Redux Store when using React Toolkit. This method aims to reduce the boilerplate required to add data to Redux. Internally, it uses createAction and createReducer.

We must first import the createSlice() method from the React Toolkit library, We then use the createSlice() method to create our slice.

	

const exampleSlice = createSlice({
	
	// ...

})

Redux Toolkit allows us to write "mutating" logic in reducers. It doesn't actually mutate the state because it uses the Immer library which detects changes to a "draft state" and produces a brand new immutable state based off those changes.

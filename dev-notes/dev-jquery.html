jQuery

jQuery is a lightweight JavaScript library - within a single JavaScript file - to simplify the use of JavaScript. jQuery takes common tasks - such as DOM manipulation, AJAX calls and animations - that may require many lines of JavaScript code, and wraps them into methods you can call with a single line of code. jQuery also has plugins for adding further functionality.

There are two versions of jQuery available:

  • Production version - this is for the live/production website and has been minified/compressed.
  • Development version - this is for testing and development. Is uncompressed and readable.

Adding jQuery to HTML

To include the jQuery library within HTML, the single JavaScript file needs to be referenced within a <script> tag before the closing </body> element of the HTML code:

				

<!-- Include jQuery v3.2.0 (minified) as an example -->
<script src="jquery-3.2.0.min.js"></script>

</body>

If we want to include other JavaScript files containing external jQuery code, JavaScript code and/or jQuery plugins, they should be placed under the jQuery <script> tag.

				

<!-- Include jQuery v3.2.0 (minified) as an example -->
<script src="jquery-3.2.0.min.js"></script>
<!-- Example jQuery plugin -->
<script src="jquery-plugin.js"></script>
<!-- External custom JavaScript/jQuery file -->
<script src="scripts/my-script-file.js"></script>

</body>

If you don't want to download jQuery, you can include it from a CDN (Content Delivery Network). Many users have already downloaded jQuery from a CDN when visiting another website. As a result, it will be loaded from the cache when they visit your website. This improves the performance of the website and leads to faster loading times. Most CDN's will also make sure that once a user requests a file, it will be served from the server closest to them, this also helps improve loading times.

To include jQuery from a CDN, use the following examples:

				

<!-- jQuery v3.2.0 from Google CDN -->
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.2.0/jquery.min.js"></script>

<!-- OR, jQuery v3.2.0 from Microsoft CDN -->
<script src="https://ajax.aspnetcdn.com/ajax/jQuery/jquery-3.2.0.min.js"></script>

</body>

When installing or using a CDN to use jQuery, always remember to check the latest version and browser support. The example version of jQuery 3.2.0 is used only as an example within the code snippets.

Syntax

The jQuery syntax is developed to select HTML elements and perform actions on them. Here is the simple syntax jQuery uses:

				

// Example syntax
$(selector).action()

// Hide the current element
$(this).hide()
// Hide all <p> elements
$("p").hide()
// Hide all elements with the 'test' class
$(".test").hide()
// Hide the element with an id of 'test'
$("#test").hide()

jQuery uses CSS syntax to select elements.

  • $ - used to define/access jQuery.
  • ("p") - the selector used to query (find) the HTML element.
  • hide( ) - action to be performed on the element(s).

Selectors

jQuery selectors are one of the most important parts of jQuery. Selectors allow you to select and manipulate HTML elements using their: name, id, classes, types, values or attributes.

All selectors in jQuery start with the dollar sign $

Element selector

The jQuery element selector allows elements to be selected based on the element name. For example, all <p> elements on the page can be selected and hidden when the button is clicked:

				

$(document).ready( function () {
	
	$("button").click( function () {
		// Hide all <p> elements in the HTML
		$("p").hide();
	})

});

Class and ID selectors

As with the element selector, we can also select elements with a specific class or id attribute. Once again, the elements that have been selected will be hidden when the button is clicked.

				

$(document).ready( function () {
	
	$("button").click( function () {
		// This is a class selector
		$(".js-hide-element").hide();
		// This is an id selector
		$("#js-hide-div-one").hide();
	})

});

The HTML has been included to better demonstrate this.

				

<body>

	<div class="js-hide-element">This is hidden on button click</div>
	<div class="js-hide-element">This is hidden on button click</div>
	<div class="another-class">Not hidden on button click</div>

	<div class="another class" id="js-hide-div-one">This is hidden on button click</div>

</body>

To identify that we are using attributes to apply jQuery/JavaScript to the HTML elements, it is recommended to use the js- prefix. That way, we can use a seperate attribute (class/id) to add CSS styling. This allows us to change the styling of a HTML element without having to change the JavaScript code, which would be the issue if we use the same class/id for both CSS and JavaScript.

				

<div class="css-selector js-selector"> ... </div>

Document Ready Event

The Document ready event prevents any jQuery code from running before the HTML document has finished loading and is ready.

It is good practice to wait for the document to be fully loaded before working with it.

				

$(document).ready( function () {
	
	// jQuery methods go here...

});


// OR, Shorter method to use the document ready event:
$( function () {
	
	// jQuery methods go here...

});

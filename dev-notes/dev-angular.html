Angular

Angular (commonly referred to as Angular2) is a TypeScript based, open-source, front-end web application framework, created by Google. Angular is a complete rewrite from the same team that built AngularJS

Angular CLI

The Angular CLI is a Command Line Interface tool that can create a project, add files and perform a variety of development tasks. To begin using Angular, you need to set up your development environment first.

You will need Node and NPM (Node Package Manager) installed. Versions no less than Node v6.9.x and NPM 3.x.x should be used.

Then install the Angular CLI globally.

			

$ npm install @angular/cli -g

Create a new project

Now that the Angular CLI has been installed, a new project can be created. The following command will generate a new project and a skeleton Angular application.

The my-app section of the command, is the name of the application. This can be changed to anything you feel is appropriate. When the command is used, the required NPM (Node Package Manager) packages will be installed.

			

$ ng new my-app

A folder will be created containing the application.

Run the application

Once the setup/installation is complete, we can run the application. This will launch the server using the ng serve command, watches the files and rebuilds the application if those files change.

The --open (can simply be -o) will automatically open the web browser to display the application running.

			

$ cd my-app
$ ng serve --open

The application will open in the browser on: http://localhost:4200/

The message app works! will be visible in a browser window, if successful.

Angular basics

When using the CLI, Angular created a component. This is called the root component and is named app-root. This can be found in: src/app/app.component.ts. This is a TypeScript file.

				

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'My First Angular App';
}

Open this component file and change the title property. The browser will automatically reload the changes with the revised title text.

				

export class AppComponent {
  title = 'My First Angular App';
}

The component template file code, looks like this:

				

<h1>
  {{title}}
</h1>

As you can see, the {{title}} value from the title property, is displayed between <h1> and </h1>. If we change the CSS and apply it to the <h1> element, the styling of our title text will change.

The app files are all located in the src folder. All Angular components, templates, styles, images and any other files your application needs, goes here.

To change the styling of this title text, open src/app/app.component.css and provide some CSS styling. This will be empty by default.

				

h1 {
	color: #369;
	font-family: Arial, Helvetica, sans-serif;
	font-size: 18px;
}

Components

Angular applications are made up of components. A component is the combination of an HTML template and a component class that controls a portion of the screen.

Every component begins with an @Component decorator function. This takes a metadata object that describes how the HTML template and component class work together.

Within the @Component decorator function, you may find a selector and template property.

The selector property tells Angular to display the component inside of a custom tag in the index.html file.

In the example below, the custom tag will be <app-root>.

				

@Component({
	selector: 'app-root'
})

This can then be added to the index.html file that will be used to output our application. The selector tag will be used within the HTML as the root of the application (where the app will be loaded).

				

<body>

	<app-root>The content of the App will be loaded here.</app-root>

</body>

To add content to the application, we can use the template property. This is added to the @Component decorator function. The content on the index.html page should now read 'Hello World'.

				

@Component({
	selector: 'app-root',
	template: '<h1>Hello World</h1>'
})

Don't forget to include the import and export statements - as with the example above - within the component, otherwise this won't work.

Now when we run the application, Hello World will be displayed as we are displaying whatever has been added to the template property. If we want to place a component in a separate file (such as src/app.components.html), we change the template property to the templateUrl property instead.

				

@Component({
	selector: 'app-root',
	templateUrl: './app.component.html',
})

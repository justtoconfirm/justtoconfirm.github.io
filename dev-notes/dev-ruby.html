Ruby

Ruby is a dynamic, open source, object-oriented programming language designed and developed in the mid 1990's by Yukihiro "Matz" Matsumoto in Japan.

Ruby files use the file extension .rb

Install

If you're using a Mac or Linux machine, Ruby should already be pre-installed. Open the command line or terminal application and enter the following command to confirm the version of Ruby installed:

			

$ ruby -v

Some versions of Ruby pre-installed on Mac, may have limited permissions to allow you to install gems or update Ruby. It is therefore recommended to install Ruby anyway using one of four methods:

  • rvm
  • rbenv
  • Homebrew (Mac only)
  • Build from source

On a Windows machine, Ruby will need to be downloaded manually using tools such as RubyInstaller.

RubyGems comes with Ruby 1.9.3 and higher, so once Ruby has been installed, you can verify that RubyGems has been installed by confirming the version number.

			

$ gem -v

Ruby "Gems"

Sometimes called RubyGems or simply "gems" for short, refers to a collection of Ruby files, or a Ruby library, that performs a certain task. This also refers to the publishing system that allows these 'packages' to be installed and published. The libraries this system publishes are called "gems".

RubyGems is integrated into Ruby 1.9 and later. The code in a gem is a pre-packaged bundle of code to allow features and additional functionality to be added. The versioning of gems is based on Semantic Versioning, and a gem is said to have been "cut" when a new version becomes available.

The gem command allows you to use RubyGems. When combined with the install command, a gem can be downloaded and installed (with their dependencies and/or relevant documentation).

			

$ gem install <gem name>

In the example above, <gem name> is the actual name of the gem you want to install.

Syntax conventions

The Ruby language has specific syntax conventions that make writing and reading Ruby code easier for all. Please follow these conventions when coding in Ruby:

  • Tab function should be set to 2 spaces and indenting should be set to use spaces.
  • Comments should start with # at the beginning of the line.
  • Methods, variables or files should always use snake_case formatting for naming conventions (all lowercase and separate words with underscores).
  • A value that does not change - such as a constant variable - is a constant and should be denoted with all uppercase letters.
  • Declare a class name using CamelCase formatting. CamelCase uses no spaces and capitalizes the first letter of each work, for example, MyFirstClass

Like Python, Ruby does not use semi-colons (;) to end statements.

Comments

Comments are programmer-readable annotations that are ignored at runtime. Their purpose is to make source code easier to understand.

The # character is used to add a single line comment.

				

# This is a single line comment
puts "Hello World"

Multiple-line comments can be added by using =begin and =end syntax (known as block markers).

				

=begin
This is a 
multiline comment.
=end
puts "Hello World"

Execute Ruby files

To execute a Ruby file from the command terminal, use the following command:

			

# Example command to execute a Ruby file
$ ruby <filename_here>.rb

$ ruby my_file.rb

Ruby source files don't need to be compiled before being executed. The Ruby interpreter compiles and executes the Ruby files at runtime.

Hello World

An example of how to recreate the famous "Hello World" code example in Ruby. Begin by opening an empty file and save the file as hello.rb. Place the following line of code into this file:

				

puts "Hello World"

From the command line, type the following command to execute the Ruby code from the source file:

			

# The output should be: Hello World
$ ruby hello.rb

If we want to use a method, we can use the following example:

				

# Method
def hello_world
	puts "Hello World"
end

# Call the method
hello_world()

And once again, use the same command to execute the code:

			

$ ruby hello_world.rb

Methods

You may often have a piece of code that needs to be executed many times. In Ruby, this is called a method (similar to a function in other languages). Before a method can be used, it needs to be defined using the keyword def followed by the name of the method.

At the end of the method, the keyword end is used.

				

def method_name
	# method body goes here
end

A method can be called by using the name given.

				

# Call method
method_name()

In the method definition, we can add parameters. Parameters are used when you have data outside of a method's scope, but you need access to it within the method. If the method does not need access to any outside data, you do not need to define any parameters.

Arguments are pieces of information that are sent to a method to be modified to return a specific result. We pass arguments to a method when we call it.

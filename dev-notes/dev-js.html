JavaScript

JavaScript (JS) was created in 1995 by Brendan Eich at Netscape. It is a high-level, dynamic programming language used to make web pages interactive and to build web applications. JavaScript is part of the ECMAScript language specification.

JavaScript is the most widely used programming language on the web.

A high-level programming language, such as JavaScript, is closer to a naturally spoken language, whereas low-level languages are closer to machine language in syntax.

JavaScript files usually have the file extension of .js

The DOM Parser takes HTML code and creates a structured page that we visually see. CSS parser takes CSS code and renders the document layout correctly. Layout/Rendering engine in the browser includes both to render the page. The JavaScript engine such SpiderMonkey, V8, and Nitro - takes the JavaScript syntax and convert into readable code. This is client side, the files are downloaded and then processed on the client's computer.

JIT (Just In Time) compiler, like Google's V8 engine, is what converts the JavaScript code into machine language.

The console is a text interface to input commands and view outputs. JavaScript has a console within the browser inspector (developer tools). Commands entered are passed to the JIT compiler, compiled, run, and the output returned back to the console. The console will also output any errors and warnings.

Performance

Loops are often used in programming. Each statement in a loop, is executed for each iteration of the loop. Statements or assignments that can be placed outside of the loop will make the loop run faster. The code below accesses the length property outside of the loop making it perform better.

				

var myList = [ 'string1', 'string2', 'string3' ];
var i;
var len = myList.length;

// For loop used to list all items in the array
for ( i = 0; i < len; i++ ) {
	console.log('Item in array: ' + myList[i]);
}

Accessing the HTML DOM is very slow when using JavaScript. If you expect to access a DOM element multiple times, access it once, and use it as a local variable.

				

var obj;

// Access the element within the HTML DOM
obj = document.getElementById("element");
// Set the text within the element
obj.innerHTML = "Hello World";

It is recommended to not declare variables if they are not needed and you don't intend to store values.

				

// Variable does not need to be used in this example
var fullName = firstName + ' ' + lastName;
document.getElementById("element").innerHTML = fullName;

// A much improved example - no variable needed
document.getElementById("element").innerHTML = firstName + ' ' + lastName;

External JavaScript & HTML

External JavaScript files are practical as the same code can be used in multiple locations. To use an external JavaScript/Script file, the name of the file is added to the src attribute of the HTML <script> tag.

Find further notes on HTML here.

It is recommended to place scripts at the bottom of the <body> element just before the closing </body> element to allow the parser to load and display the HTML/CSS (Cascading Style Sheets) before loading the JavaScript. If the parser encounters a script file, it will stop loading the webpage until the script has been downloaded.

This method has it's own issue. The browser cannot download the script until the entire document has been parsed. The abilty to download the script as soon as possible, is very important for performance.

				

<script src="scripts/js-file.js"></script>

</body>

Today, modern browsers support the async and defer attributes on scripts tags. These attributes will tell the browser that it is safe to continue parsing while the JavaScript is being downloaded.

				

<script src="scripts/js-file.js" async></script>
<script src="scripts/js-file-two.js" defer></script>

</body>

Scripts with the async attribute are executed asynchronously. The script is executed as soon as it is downloaded without blocking the browser.

Scripts with the defer attribute are executed in order of how they are added to the HTML, but this also does not block the browser. Defer scripts are only executed after the entire document has been loaded.

90% of browsers support the use of the async attribute, 92% support the use of defer.

Strict mode

Strict mode can be applied on entire scripts by using the strict statement before any other statement (usually placed as the first line in a JavaScript file). Strict mode is only enabled in scripts where it is defined. This is not shared between different script files.

It was introduced in ECMAScript 5 (ES5) and is supported in the following browsers:

  • IE10 and greater.
  • Firefox from version 4.
  • Chrome from version 13.
  • Safari from version 5.1.
  • Opera from version 12.
				

"use strict";		// now applies for the rest of the script

All code written inside ES2015 (ES6) modules and classes is strict by default.

With script mode, for example, you cannot use undeclared variables, so a variable would always need to use the var keyword to declare a variable or an error will be thrown.

Script mode basically catches things that are most likely valid, but could cause errors.

Variables

Data or information that needs to be temporarily stored can use a variable. Variables are containers that you can use to store values. To allow the variable to be used, it first needs to be declared using the var keyword. Once declared, the variable is given a name - called the identifier.

				

var myVariable;

After declaring the variable, you can then give it a value. Until a variable has a value, it is said to be undefined. A value is assigned to a variable using the assignment operator =. The example below shows a value being assigned to a variable:

The value of a variable can be changed:

Variables declared outside of a function are global variables, as are variables that are declared without the var keyword, even if they are created within a function.

Global variables last for as long as your application is active. Local variables are created when the function is invoked, and deleted once the function is finished.

Random number

To create a random number and store the value within a variable, the following code can be used:

				

var randomNumber = Math.floor(Math.random() * 100) + 1;
// Output the random number
console.log(randomNumber);

Let's break this code down a little.

				

Math.random();

The above line of code will return a random number between 0 and 1. If we want to generate a random number between 0 and 50, we simply just need to multiply to result by 50.

				

// Random number between 0 and 50
Math.random() * 50;

The issue here is that the answer is most likely to be a decimal number, and we would like to return whole numbers. The Math.floor JavaScript function, built-in to JavaScript, can be used to take a decimal number that is returned, and round it down to the nearest whole number.

				

// Random number is now a whole number
Math.floor(Math.random() * 50);

Math.random will generate a random number between 0 and 1, we then multiply this value by 50, so we have a number between 0 and 50, before using Math.floor to round the number down to become a whole number.

To output the value, we can either use a variable - as with the first example - or place within a console.log.

				

// Random whole number between 0 and 50
console.log(Math.floor(Math.random() * 50));

// Random whole number between 0 and 100
console.log(Math.floor(Math.random() * 100));

Array

Arrays are JavaScript's way of making lists. These lists can store different data types in an ordered way, meaning the position of each list item is numbered by JavaScript.

Arrays are created with brackets [ ]

				

var myList = [ 'string1', 'string2', 'string3' ];
console.log(myList);		// (3) ["string1", "string2", "string3"]

Each item in an array has a numbered position. We can access an item using it's index number. JavaScript counts from 0, not 1, so the first item in an array will be at position 0. This is because JavaScript is zero-indexed.

If we wanted to get the first item in the array, we would use the index value of 0:

				

var myList = [ 'string1', 'string2', 'string3' ];
var myListItem = myList[0];

console.log(myListItem);	// string1

It is often convenient to know how many items are inside of an array. We can use the built-in JavaScript length property for this.

				

var myList = [ 'string1', 'string2', 'string3' ];

console.log(myList.length);		// 3

JavaScript has a number of built-in functions for arrays that help us to perform common tasks. First, the push function allows us to add items to the end of an array.

				

var myList = [ 'string1', 'string2', 'string3' ];
myList.push( 'string4', 'string5' );	// Add two items to the end of the array

console.log(myList);		// (5) ["string1", "string2", "string3", "string4", "string5"]

Now that we can push items onto an array, let's remove the last item on the end of the array using the pop function.

				

var myList = [ 'string1', 'string2', 'string3' ];
myList.pop();	// Remove last item from the end of the array

console.log(myList);		// (2) ["string1", "string2"]

An array can be sorted alphabetically using the sort method.

				

var fruits = [ 'Banana', 'Orange', 'Apple', 'Mango' ];
fruits.sort();		// Sort the elements of the fruits array

Loops are especially helpful when we have an array. We can use a loop to perform an action on each item within the array, or log all items that are currently within the array.

				

var myList = [ 'string1', 'string2', 'string3' ];

// For loop used to list all items in the array
for ( var i = 0; i < myList.length; i++ ) {
	console.log('Item in array: ' + myList[i]);
}

// If we want to list the items in reverse
for ( var i = myList.length -1; i >= 0; i-- ) {
	console.log('Item in array: ' + myList[i]);
}

We can also use a for loop to compare values in more than one array. This can be done by using a for loop within another for loop. Each time the outer for loop runs once, the inner for loop will run completely.

				

var myList = [ 'string1', 'string2', 'string3' ];
var myListTwo = [ 's1', 's2', 's3', 'string1' ];

for ( var i = 0; i < myList.length; i++ ) {
	console.log( myList[i] );

	for ( var j = 0; j < myListTwo.length; j++ ) {
		console.log( myListTwo[j] );

		if ( myList[i] === myListTwo[j] ) {
			console.log('Match: ' + myList[i]);
		}
	}
}

When running the code, the Match of 'string1' will be highlighted within the console, as it can be found in either array.

The reverse JavaScript function can be used to reverse the order of items inside of an array.

				

var myList = [ 'string1', 'string2', 'string3' ];
myList.reverse();

console.log(myList);	// (3) ["string3", "string2", "string1"]

Arrays can also be concatenated using the concat method.

				

var array1 = [ 1, 2 ];
var array2 = [ 3, 4, 5 ];
var array3 = array1.concat( array2 );	// Returns a new array

console.log(array3);	// (5) [1, 2, 3, 4, 5]

Control Flow & Conditional statements

In programming, making a decision with code is called control flow. Control flow statements enable JavaScript to make a decision by executing a different code based on a condition.

An if statement can be used for this. If the condition is true, JavaScript will run the code block, if the condition is false, the else statement will be executed and this code block will run.

Code between curly braces are called blocks. If/Else statements have two code blocks.

				

var needTea = true;

// If the condition is 'true'
if ( needTea ) {
	console.log('I\'m happy to be drinking tea.');
} else {
	console.log('Still no tea.');
}

Conditional statements are made even more powerful with comparison operators.

  • Less than: <
  • Greater than: >
  • Less than or equal to: <=
  • Greater than or equal to: >=
  • And - 'both must be true': &&
  • Or - 'either can be true': ||
  • Not - 'opposite of what it really is': !
  • Not equal to - 'not equal each other': !== 
				

var stopLight = 'green';
var pedestrians = false;

if ( stopLight === 'green' && pedestrians === false ) {
	console.log('Go');
} else {
	console.log('Stop');
}

If/Else statements check the how true or false the variable is, so there is no need to check for this.

				

var needTea = true;

// We DO NOT need to do this as the if statement will check true/false
if ( needTea === true ) {
	console.log('I\'m happy to be drinking tea.');
}

// This is CORRECT
if ( needTea ) {
	console.log('I\'m happy to be drinking tea.');
}

If and Else statements are great if we have a few different conditions to check, but for multiple conditions, we would need to write a lot of JavaScript code making it difficult to read and maintain. For this purpose, we can turn to a switch statement to write more readable and concise code instead.

Switch statement and if/else statements are essentially the same, but a switch statement can be easier for a human to read. To create a switch statement, we use the switch ketword.

				

var shoppingItem = 'papaya';

switch ( shoppingItem ) {
  case 'tomato':
    console.log('Tomatoes are £0.49');
    break;
  case 'lime':
    console.log('Limes are $0.49');
    break;
  case 'papaya':
    console.log('Papayas are $1.29');
    break;
  default:
    console.log('Invalid item');
    break;
}

The switch keyword initiates the statement and is followed by ( ... ) which contains the condition that each case will compare to. The program stops with the break keyword. This keyword will prevent the switch statement from executing and more of it's code. At the end of the switch statement, there is a default condition. If no cases are true, then this part of the code will run.

Conditional statements can be shortened using a Ternary operator. 

				

var animal = 'cat';

/*
If the animal is a 'cat' log 'The animal is a cat'
else log 'The animal is not a cat'
*/
var result = ( animal === 'cat' ) ? 'The animal is a cat' : 'The animal is not a cat';

console.log(result);	// The animal is a cat

While loops

For loops are great, but they have limitations. You have to know how many times you want the loop to run. This is something that can be solved by using a while loop. A while loop begins with the keyword while followed by the condition within the parenthesis. As long as the variable evaluates to true, the block of code will loop.

				

var cards = [ 'Diamond', 'Spade', 'Heart', 'Club' ];
var currentCard = 'Heart';

while ( currentCard !== 'Spade' ) {
	console.log(currentCard);

	var randomNumber = Math.floor( Math.random() * 4 );

	currentCard = cards[randomNumber];
}

// The while loop will continue until a Spade is found
console.log('Spade');

If the currentCard is not 'Spade', then use console.log within the loop to print the value of the currentCard. Then, create a random number between 0 and 3 and put it into a variable named randomNumber. We can then use this randomNumber to reassign currentCard to a new card from the array. Outside of the while loop, a console.log is ued to log when the program has found 'Spade'.

As another example, lets create a coin toss using a while loop:

				

var coinFace = Math.floor( Math.random() * 2 );

while ( coinFace === 0 ) {

	console.log('Heads, flipping the coin again.');
	var coinFace = Math.floor( Math.random() * 2 );

}
console.log('Tails.');

For loops

Loops are a very important concept in programming, allowing you to keep running a piece of code over and over again, until a certain condition is met.

The following example will print out the numbers 1 through to 20 within the console. This is an example of a for loop.

				

for ( var i = 1; i < 21; i++ ) {
	console.log(i);
}

The loop takes three input values as arguments:

  • A starting value - the count is started at 1, but could be any number you like. The variable name of i can be changed (it does not have to be 'i'), but is usually the convention within loops because it is short and easy to remember.
  • Exit condition - the loop will keep going until the value of i is no longer less than 21. When i reaches 21, the loop will no longer run.
  • Incrementor - we have specified i++ which means 'add 1 to i'. The loop will run once for every value of i, until i reaches a value of 21. The value of i is output within the console on every iteration.

Functions

Functions are one of the fundemental building blocks of JavaScript. A function is a JavaScript procedure (a set of statements that perform a task). Functions take data, perform a set of tasks, and then return the result. The beauty of functions is that they allow us to write a chunk of code once that we can reuse over and over without having to write the same code multiple times.

To create a function, it must first be declared using the function keyword.

				

function myFunc () {

	// Code is placed here

}

Defining a function does not execute it, but names the function and specifies what the function is to do when it is called. To call the function, follow the example code below:

				

myFunc();

A simple example can be found below. This example will display an alert message when the function is called.

				

function helloWorld () {
	
	// Show alert when function is called
	alert("Hello World");

}

// Call the helloWorld() function
helloWorld();

Functions must be in scope when they are called, but the function declaration can appear below the function call.

An example of a simple function which takes two numbers as arguments passed into the num1 and num2 parameters (within the function parenthesis), multiplies the values together, stores the result in a variable and returns the result using the return statement.

				

function multiply ( num1, num2 ) {

	var result = num1 * num2;
	return result;

}

When we call the function, the two values we want to multiply are passed as arguments into the multiply function and the result will then be returned.

				

multiply(4, 2);		// Multiply 4 and 2. Result: 8

The return statement tells the browser to return the variable (result) outside of the function so it is available to use. This is necessary because a variable defined inside of a function is only available inside of that function. This is called variable scoping. The return statement simply specifies that the value be returned by the function.

We can also include conditional statements within functions.

				

var computerOn = false;

function pressPowerButton () {
	
	if ( computerOn ) {
		console.log('Computer turning off.');
		computerOn = false;
	} else {
		console.log('Computer turning on.');
		computerOn = true;
	}

}

pressPowerButton();		// Output: Computer turning on.
pressPowerButton();		// Output: Computer turning off.

This code turns the computer on if it's off, and off if the computer is on. This block of code within the function is executed twice, making it reusable.

When functions return their value, we can use them together and inside one another. If we need to calculate temperature in Celsius and then Fahrenheit, we could create two functions like so:

				

/*
This function is used to calculate the fahrenheit
We pass this function into the getFahrenheit function
*/
function multiplyByNineFifths ( celsius ) {
	
	return celsius * ( 9 / 5 );

}

// This is the function we call
function getFahrenheit ( celsius ) {
	
	// Use the multiplyByNineFifths function and return the result
	return multiplyByNineFifths ( celsius ) + 32;

}

// Output: The temperature is: 59F
console.log('The temperature is: ' + getFahrenheit(15) + 'F');

We can use functions to section off small bits of logic or tasks that we can then call when we need to. Functions can help take large problems and break them into small manageable chunks.

An example of how functions can be used to section off small bits of logic:

				

var orderCount = 0;

We can use functions to return more than one value using an array. The code example, below, calculates both the area and the volume of a shape. The area of the shape is stored in a variable called area and the volume is stored in a variable called volume. Both variables are then placed into an array called sizes which is returned to the code that called the function. This allows both values to then be used.

				

function calculateSize ( width, height, depth ) {

	var area = width * height;
	var volume = width * height * depth;
	// Array named sizes created to contain the area and volume values
	var sizes = [area, volume];

// Array index is [0] so area will be calculated

// Array index is [1] so volume will be calculated
var getVolume = calculateSize(3, 2, 5)[1];

// Output values in the variables within the console

console.log(getVolume);				// 30

Return values are values returned by the function when it completes.

				

// Calculate the value squared
function squared ( num ) {

	return num * num;

}

// Calculate the value cubed
function cubed ( num ) {

	return num * num * num;

}

Anonymous Function

If you're writing a one-off chunk of JavaScript that never needs to be referred elsewhere, it's wise to use anonymous scope so the code will not be accidentially referenced elsewhere.

A simple example of an anonymous function that will be called on page load:

				

( function () {

	...	

})();

A more detailed example:

				

// Call this function once, then throw it away
( function () {
	
	var x = 123;
	console.log(x);		// 123

})();

// This will error as it's outside the function
console.log(x);

The empty brackets after the function causes the function to automatically be called (invoked) on the page load. This is known as an immediately-invoked function expresssion (IIFE).

This self-executing function is called on the page load automatically.

The error that will appear if this code is run:

			

Uncaught ReferenceError: x is not defined

Events

A function will not do anything until it is called. To call a function on a button click, we need to use an event. Events are actions that happen in the browser. Example of events can include: a button being clicked, a page loading or video loading etc.

To listen for events occuring we use event listeners. The block of code that then runs in response to this event firing, is called the event handler.

				

buttonName.addEventListener( 'click', functionName );

Above, we have an event listener added to a button. This takes two input values, called arguments. The first argument is the type of event we are listening for (click) as a string. The second argument is the function or code that we would like to run when the event occurs. In this example, the function functionName will be called.

You do not need to specify the parenthesis when calling a function within an event listener.

The following code example will show an event listener being used to display an alert message when the button is clicked.

				

<body>

	<button class="alert-btn">Alert</button>

<!-- Include the JavaScript file -->
<script src="scripts/alert.js"></script>

</body>
				
			
				

// Find the first element with the class alert-btn
var alertBtn = document.querySelector( '.alert-btn' );
// Attach the event listener to the button to call the showAlert function when clicked
alertBtn.addEventListener( 'click', showAlert );

// Function that is called

Object Oriented Programming (OOP)

Objects in JavaScript can be likened to objects in the real world. A car is an example of a real world object that has properties, used to define the object's characteristics such as: name or colour, and methods such as: accelerate or brake.

Objects are very important when it comes to programming. You live in an object-oriented world.

Objects group together a set of properties and methods to create a model. In JavaScript, properties can be seen to be similar to variables, methods are like functions, within an object.

Syntax

The way a language is written, it's style, is called the syntax and is defined by the creators of the language. Every language has its own API (Application Programming Interface). It's a collection of functions and properties we can use.

We have the window object in JavaScript. If you take a look at this in our console by typing window, we can see that our window object has many functions. These are referred to as methods when they are within an object. We also have a number of properties we can also use. Properties are like variables within objects. These were setup by the JIT compiler. This is the JavaScript API in a nutshell.

APIs can exist within other APIs. When using the jQuery library, jQuery attaches itself onto the window object, and when viewed within the console, you will see the $ sign showing that jQuery has indeed attached itself to the window object (when we type 'window' into the console). jQuery has its own API, so when you look in the console at the window object, the window object will contain the jQuery API and all it's methods and properties that jQuery provides. This is what's known as an embedded API / extension API or extended API.

jQuery needs the native JavaScript API to define it's own API for it to work. This is known as an API extension as it's adding onto the another API (Native JavaScript API). Embedded APIs are usually libraries, as they are adding (attaching themselves) to the existing API.

Creating an object within the window object - everything created in JavaScript goes inside the window object - with my own methods and properties, I am essentially creating my own API (just without a name). You could create your own library using this process.

				

window {
	MyAPI: {
		getSomething: function () {},
		getAnother: function () {},
		finish: function () {}
	}
}
				
			
				

window {
	$jQuery: {
		find: function () { ... },
		two: function () { ... }
	}
}

No matter how many browser windows/tabs you open, each window has it's own window object and each window object is isolated from the other.

				

Native API {
	get: function () { ... },
	set: function () { ... },
	return: function () { ... },

		myAPI: function () { ... },

Objects: Constructor notation

The previous object was created using Literal notation. This is fine should you need just the one object, but not useful if we want to create multiple objects. The Constructor notation allows a JavaScript function to be used to make a 'template' for creating multiple objects. Instances of this object can then be created using the keyword of new, followed by a call to the function to create a new object.

The name of the Constructor function usually begins with an uppercase letter.

				

function Hotel ( name, rooms, booked ) {

	this.name = name;
	this.rooms = rooms;
	this.booked = booked;

	// Method to check the rooms available in the Hotel
	this.checkAvailable = function () {

		return this.rooms - this.booked;

	};

}

An instance of this object can now be created. This will inherit the properties and methods within the Hotel object. Values for each hotel will be passed into the object using arguments when the function is called. The keyword of this is used instead of the object's name, to indicate that the properties and methods belong to the Hotel object.

Lets now create a hotel named 'The Grand Hotel' as an instance of the Hotel object:

				

var grandHotel = new Hotel("The Grand Hotel", 104, 98);

We can create a second hotel as another instance of our Hotel object.

				

var parkHotel = new Hotel("Park Hotel", 98, 94);

Each time the Hotel function is called, the values are passed as arguments into the object parameters and then assigned to the properties. Both instances can then use the checkAvailable() method within the object to confirm the number of rooms the hotel has available for bookings.

				

// Properties of the Grand Hotel
var grandHotelName = grandHotel.name;
var grandHotelRooms = grandHotel.rooms;

// Method to check the rooms available at the Grand Hotel
var grandAvailableRooms = grandHotel.checkAvailable();

ES6

ES6 - also known as ECMAScript 2015 / ES2015 - is the latest version of JavaScript (ECMAScript standard), that was ratified in June 2015. ES6 is a significant update to the language, and the first major update since ES5 was standardized in 2009. ES6 contains a number of new features and implementation of these new features of ES6 in major JavaScript engines for use in browsers is underway now, however, not all browsers support ES6. To use ES6 currently, we must use tools such as Babel to transpile the ES6 code we write into ES5 JavaScript code so that all browsers can use the code we write.

If you would like to setup Babel, the installation and setup instructions can be found here.

Variables

You can create variables using var, let, or const in ES6. Variables do not have a type and as long a they are not constants, you can change their data.

				

// This is how you define a variable. 'x' will be 'undefined'
var x;

// Declare a constant - the convention is to us CAPS for constants
const FOO = 42;

// Variable declared using 'let'
let bar = 'Hello';

let and var are pretty similar when it comes to declaring variables, but unlike var, let allows you to declare a variable that are limited in scope to the block, statement, or expression on which it is used. The var keyword defines a variable to an entire function regardless of block scope.

				

var a = 42;
if (true) {
	var b = 7;
	let c = 123;
}

// We can access the 'a' variable
console.log(a);

// We can also access the 'b' variable
console.log(b);

/* 
The variable of 'c' cannot be accessed
This is because we are declaring it using 'let' inside of the 'if' block.
It is only, therefore, accessible within the 'if' block itself.
*/
console.log(c);

Const is slightly simpler in that you can use it for a variable that has a value that is to remain the same.

				

const PI = 3.14;

// You cannot assign a new value to a const - this will cause an error
PI = 3;

Functions

There are a few ways to define functions in JavaScript. The common uses are:

				

function sum ( a, b ) {
	
	return a + b;

}

// Declare function using a variable
var sum = function ( a, b ) {
	
	return a + b;

}

// Using the ES6 arrow function
let sum = ( a, b ) => a + b;

The Arrow function => is sometimes called the "fat arrow" function and is a concise way of writing function expressions in ES6. The expression after the arrow leads to what is being returned. In the example above, this is a + b.

You can then call the function:

				

let result = sum( 40, 2 );

// OR, without using a variable
sum( 40, 2 );

As an example, lets run the ES6 code using Babel and view the ES5 JavaScript output. You will notice that Babel will add "use strict" at the top of the file being output once the ES6 code has been transpiled.

				

// Arrow function
let sum = ( a, b ) => a + b;

/* 
Call the function and pass the values as arguments into the function parameters 'a' and 'b'
The variable will then hold the result
*/
let result = sum( 40, 2 );

console.log(result);

The ES5 that is created once the ES6 code above is transpiled:

				

"use strict";

var sum = function sum( a, b ) {
	
	return a + b;

};

var result = sum( 40, 2 );

console.log(result);

When the code is run within the console, the value of 42 should be output.

Classes

JavaScript classes were introduced in ES6 and are syntactical sugar over JavaScript's existing prototype-based inheritance. The class syntax is not introducing a new object-oriented model to JavaScript, but it does create a much simpler syntax for creating objects.

Classes are special functions and has two components: class expressions and class declarations.

To define a class, we use a class declaration. This uses the classkeyword followed by the name of the class.

				

class ClassName {
	...
}

Another way to define a class is to use a class expression. Class expressions can be named or unnamed. The name given to a named class expression is local to the class's body.

				

// Unnamed class expression
var VariableName = class {
	...
}

// Named class expression
var VariableName = class ClassName {
	...
}

Lets create a simple example of a class in ES6. This will use the class declaration syntax.

This is very similar to how we use Constructor notation in JavaScript.

				

class Person {
	constructor ( name ) {
		this.name = name;
	}

	getName () {
		return this.name;
	}
}

// Create a new instance of the 'Person' class
var p = new Person("Dan");

// Output: Dan
console.log(p.getName());

The constructor method is a special method for creating and initializing an object created with a class. There can only be one constructor within a class. A SyntaxError will be thrown if more than one is added. The constructor will take the value(s) passed as arguments when the function is called, and allow them to be used within the class - as with the name example (above).

Modules

JavaScript, by default, lacks the ability to create modules to create modular code and seperate the code into seperate files to allow it to be easy to maintain and reusable. There are two main standards which let developers use modules in JavaScript, they are CommonJS and Asynchronous Mobile Definition (AMD). By using ES6, modules will are brought into JavaScript as standard. As most browsers are still to support ES6, we still will need to transpile ES6 modules into ES5 JavaScript code.

Each ES6 module is defined in it's own file. The functions or variables defined in a module are not visible outside of it, unless you explicitly export them. This means that you can write code in your module and only export what should be accessed by other parts of your application.

To export your module, use the keyword of export. Similarly, to consume exported modules, you use the import keyword.

Next, let's create a file named utility.js for the module.

				

function generateRandom() {

	return Math.random();

}

function sum ( a, b ) {

	return a + b;

}

export { generateRandom, sum }

The export keyword on the last line of the file, exports the two functions (as listed in curly braces).

In another file where we want to import the functions we have exported from the utility.js module, we can now import them using the code below:

				

import { generateRandom, sum } from 'utility';

// Logs a random number
console.log(generateRandom());
// Output: 3
console.log(sum( 1, 2 ));

XMLHttpRequest

XMLHttpRequest is a JavaScript object that provides client functionality for transferring data between a client and a server. It provides an easy way to retrieve data from a URL - such as a third-party API - without having to perform a full page refresh. This enables a web page to update just part of the page without disrupting what the user is doing.

XMLHttpRequest is used heavily with AJAX.

The XMLHttpRequest API is often called XHR. This is a very useful JavaScript object that allows us to make network requests to retrieve resources from a server via JavaScript.

To create a request, we need to create a new request object instance from the XMLHttpRequest constructor.

				

// URL to make the request to - example
var requestURL = 'https://mdn.github.io/learning-area/javascript/oojs/json/superheroes.json';
var request = new XMLHttpRequest();

A new request is then created using the open method.

				

request.open( 'GET', requestURL );

The two parameters are as follows:

  • GET - The HTTP method used to make the network request. GET can be used to retrieve some simple data.
  • URL - The URL to make the request to. This can be a JSON file.

Next, we are setting the responseType to be JSON so that the server will know we are requesting a JSON object to be returned. The request is then sent using the send method.

				

request.responseType = 'json';
request.send();

XMLHttpRequest makes sending HTTP requests very easy. Simply create an instance of the object, open a URL, and send the request.

AJAX

AJAX (Asynchronous JavaScript And XML) is not a programming language but is a combination of:

  • The XMLHttpRequest object within the browser used to request external data from a web server.
  • JavaScript and the HTML DOM, to display or use the data.

Ajax applications may use XML, JSON or plain text to transport data.

Ajax allows web pages to be updated asynchronously by exchanging data with a web server behind the scenes. This means that it is possible to update parts of a web page without the need to reload the whole page.

This is how Ajax works:

  • An event occurs on a web page (page load, button clicked ...).
  • An XMLHttpRequest object is created.
  • The XMLHttpRequest object sends a request to a web server.
  • The server processed the request and then sends the response back to the web page.
  • This response is read by the JavaScript and the JavaScript then updates the page.

All modern browsers support the XMLHttpRequest object. This object is used to exchange data with a web server behind the scenes.

Fetch API

JavaScript has a built-in new standard for making requests to get data that replaces the previous XMLHttpRequest API. We can use the Fetch API, which is much cleaner, to make server requests easily.

To use the Fetch API, we simply call fetch with the URL you want. By default, the Fetch API uses the GET method.

Support for Fetch API is not present in IE (including IE11) and much of Safari currently, but Edge, Chrome and Firefox do provide support. This is still very much an experimental implementation.

A simple call using Fetch API would look like this:

				

// Call the fetch function, passing the URL of the API as a parameter
fetch( url )
.then( function() {
	
	// Your code for handling the data you get from the API

})
.catch( function() {

	// This is where you run code if the server returns any errors
	
});

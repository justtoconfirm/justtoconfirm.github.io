Babel

Babel is a transpiler for JavaScript best known for it's ability to turn ES6 JavaScript into ES5 JavaScript code that runs in all browsers.

Install Babel

Begin by creating an empty folder/directory where Babel can be installed. From inside this project folder, create a src and dist folder. The src folder will contain our ES6 JavaScript code and Babel will output ES5 JavaScript code within the dist folder.

Our example project folder will be named babel-project.

			

$ mkdir babel-project
$ cd babel-project
$ mkdir src
$ mkdir dist

Next, we want to create a package.json file so we can install Babel locally using NPM (Node Package Manager).

			

$ mkdir init

Once the package.json file has been created, Babel can be installed using NPM. We will begin by installing babel-cli (Babel Command-Line Interface) followed by the ECMAScript 2015 preset, which will allow us to write ES6 JavaScript code.

			

$ npm install babel-cli --save-dev
$ npm install babel-preset-es2015 --save-dev

Both NPM packages will be listed as devDependencies within the package.json file. Open the package.json file and add the following task within the scripts property:

				

{
  "name": "babel",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "babel": "babel --presets es2015 src/main.js -o dist/main.bundle.js"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "babel-cli": "^6.24.1",
    "babel-preset-env": "^1.4.0"
  }
}

To run Babel and compile code from ES6 JavaScript into ES5 JavaScript, use the following NPM command within the root of our project folder:

			

$ npm run babel

This command will run the babel command listed within the scripts property of the package.json file.

The ES6 file to be compiled is named main.js and is placed within the src directory/folder of our project. Once compiled, a file named main.bundle.js will be created in the dist directory/folder containing the compiled ES5 JavaScript code.

If you want to compile more than one JavaScript file and bundle into a single file, the package.json file can be updated to use a glob rather than a specific filename.

				

"babel": "babel --presets es2015 src/*.js -o dist/main.bundle.js"

Including JavaScript with HTML

ES6 JavaScript code cannot currently be used with some current browsers, so Babel can be used to compiled the code into ES5 for use in all browsers. Once compiled into ES5, we can include the JavaScript code within HTML using a standard <script> tag.

				

<script src="dist/main.bundle.js"></script>

Development Server

We can also use a lightweight web server to run the application locally during development. Use NPM to install http-server. You would use a local web server if you want to test an application that needs to be loaded using the http protocol and will not work if loaded using the file protocol.

			

$ npm install http-server --save-dev

Once installed, update the package.json file to allow the server to be run.

				

{
  "name": "babel",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "babel": "babel --presets es2015 src/main.js -o dist/main.bundle.js",
    "start": "http-server"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "babel-cli": "^6.24.1",
    "babel-preset-env": "^1.4.0"
  }
}

To run the server, use the following NPM command in the terminal:

			

$ npm run start

Open the browser and point the URL to: http://localhost:8080 to view the server. To change the port being used, update the package.json file:

				

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "babel": "babel --presets es2015 src/main.js -o dist/main.bundle.js",
    "start": "http-server -p 9000"
},

The above example would change the port of the server to 9000

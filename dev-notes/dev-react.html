React

React (sometimes also called React.js or ReactJS), is an open-source JavaScript library for building user interfaces. It was developed by Facebook. It is used in the creation of interactive and reusable UI (User Interface) components. React can be used on both the client side and server side.

React is only the view layer.

React gives you a template language and functions to render HTML. This is all React will output.

React allows developers to create large applications that use data which can change over time, without reloading the page.

React corresponds only to the View layer in the Model-View-Controller (MVC) template. It can be used alongside other JavaScript libraries or frameworks in MVC.

Install React

Using the NPM (Node Package Manager) Create React App package, is the easiest way to start building a React single page application (SPA). Begin by opening a terminal and enter the following commands. The my-app part of the command is the name of your application.

			

$ npm install -g create-react-app
$ create-react-app my-app

$ cd my-app
$ npm start

This process does not handle backend logic or databases, just the frontend of the build pipeline. Any backend can then be used. Once the process has been completed using the command above, a directory containing the application named my-app is created.

create-react-app uses the Babel and Webpack build tools, so you can use ES6 JavaScript syntax.

ES6 support in React was introduced in version 0.13.0. ES6 code is transpiled into ES5 for use in browsers using the Babel transpiler.

When you use the command npm start, a local server is started and the application will be visible using the URL: http://localhost:3000, although the browser will be opened by default.

When your ready to deploy your React application to production, run the following command. This will build your application for production inside the build folder within your project directory.

			

$ npm run build

Alternative installation

Another way to install and setup React, is to install each component required individually. Begin by using NPM (Node Package Manager) to install React.

The npm init command will create the package.json file. We will then install the react and react-dom NPM packages.

			

$ npm init
$ npm install react react-dom --save-dev

The React documentation recommends using React with Babel to let you use ES6 and/or JSX in your code. ES6 is the modern JavaScript syntax and includes new features to make development easier. JSX is an extension to the JavaScript language that works well with React.

It is recommended to install babel-preset-react and babel-preset-es2015 and to create a .babelrc config file to use Babel.

Babel is a compiler and lets you write modern JavaScript code that still works in older browsers.

A modern build pipeline typically consists of: Package manager, bundler and compiler.

JSX

JSX is a syntax extension of JavaScript and is neither a string nor HTML. JSX is used to describe what the UI should look like. React elements are produced when we use JSX.

JSX is closer to JavaScript than HTML. React DOM uses the camelCase property for naming conventions instead of HTML attribute names. For example, class becomes className in JSX, and tabindex becomes tabIndex. JSX is not a requirement for using React

Hello World

In this example, the smallest React example is created to output Hello World.

				

import React from 'react';
import ReactDOM from 'react-dom';

ReactDOM.render(
	<h1>Hello World</h1>,
	document.getElementById('root')
);

This file is saved as index.js within the src directory. When we run React using NPM, Hello World will be output on the webpage.

			

$ cd my-app

# This is used to start a React application
$ npm start

The index.js file will be the homepage of this React application.

Adding CSS

Create a simple CSS stylesheet within the src directory of the my-app React application. This will be called style.css.

				

body {
	color: #00d8ff;
}

We can then include the stylesheet within the index.js file. This will change the colour of the text within our page.

				

import React from 'react';
import ReactDOM from 'react-dom';
import './style.css';

Import & Export

Importing and exporting modules or components, is a big part of JavaScript. ES6 introduced the import and export keywords.

To import a module or component, you can use the import statement. In React, this will look like the following syntax:

				

import React from 'react';
import ReactDOM from 'react-dom';

An export statement may look like this:

				

// App is the name of the class we want to export
export default App;

The default keyword is used, in the example above, when the module is being exported. This is telling the JavaScript engine that the function or variable should be included as default. The default module can be imported using any variable name. For example, if we wanted to import the module as SomeExampleClass, we could do something like this:

				

import SomeExampleClass from './App';

Exporting without the default keyword means we will have to use braces to import the file. Say we want to export a constant, we could add another export statement to the file.

				

export default class App {
	// ...
}

export const DIRECTION_IN = 'DIRECTION_IN';
export const DIRECTION_OUT = 'DIRECTION_OUT';

The first line of the file will export the App class directly. Now, we can import both the App class and the DIRECTION_OUT constant, using braces.

				

import App, {DIRECTION_OUT} from './App';

The as keyword can also be used to rename the variable/constant:

				

import App, {DIRECTION_OUT as APP_DIRECTION_OUT} from './App';

Example

To keep things simple, an example has been created to show how components can be created and used with React. Inside the src folder of our my-app project, created earlier, open an empty file. This will be our main index file that React will use to create our application.

				

import React from 'react';
import ReactDOM from 'react-dom';

function AppContainer () {
	return (
		<div>
			<Header />
			<h1>Hello World</h1>
		</div>
	);
}

function Header () {
	return (
		<header>
			<p>This is a header</p>
		</header>
	);
}

ReactDOM.render(
	<AppContainer />,
	document.getElementById('root')
);

Next, we need to create a simple HTML page where we can render our React application. This will be placed within a public directory.

				

<!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="utf-8" />
	<title>React application</title>
</head>
<body>

<!-- Where our React application will be rendered -->
<div id="root"></div>

<!-- Need to include the JS file -->
<script src="../src/index.js"></script>

</body>
</html>

The "root" attribute used within the index.html file, is called a "root" DOM node because everything inside it will be managed by React DOM. Applications built with just React usually have a single root DOM node. This does not have to be called "root" to work.

We may also want to create components. Create a components directory within the src folder of our project and add the following file:

				

import React from 'react';

function Box () {
	return (
		<div className='box'>
			<p>This is a component</p>
		</div>
	);
}

export default Box;

The index.js file needs to be updated to include the Box component.

				

import React from 'react';
import ReactDOM from 'react-dom';

import Box from './components/Box';

function AppContainer () {
	return (
		<div>
			<Header />
			<h1>Hello World</h1>
			<Box />
		</div>
	);
}

function Header () {
	return (
		<header>
			<p>This is a header</p>
		</header>
	);
}

ReactDOM.render(
	<AppContainer />,
	document.getElementById('root')
);

If the application is not currently running, you can open the command terminal, cd into the my-app directory and run the React application to see the result.

			

$ cd my-app
$ npm start

The file structure should look like this:

			

my-app /
	.gitignore
	node_modules /
	package.json
	public /
		index.html
	README.md
	src /
		index.js
		components /
			Box.js

The simplest way to define a component is to write a JavaScript function. Component names should always start with an uppercase letter.

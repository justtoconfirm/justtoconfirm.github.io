React

React (sometimes also called React.js or ReactJS), is an open-source JavaScript library for building user interfaces. It was developed by Facebook. It is used in the creation of interactive and reusable UI (User Interface) components. React can be used on both the client side and server side. This JavaScript library allows you to turn a complex UI into simple and reusable components.

React allows developers to create large applications that use data which can change over time without the need to reload the page. React also provides a template language and functions to render HTML.

React is only the view layer. ReactDOM is a standalone library that is used to render React components on the DOM. React will only output HTML.

React only corresponds to the View layer in the Model-View-Controller (MVC) template. React can also be used alongside other JavaScript libraries or frameworks.

Install React

Using the NPM (Node Package Manager) Create React App package, is the easiest way to start building a React single page application (SPA). Begin by opening a terminal and enter the following commands. The my-app part of the command is the name of your application.

			

// If you need to install the create-react-app package
$ npm install -g create-react-app

// Create a new React application if create-react-app is already installed
$ create-react-app my-app

$ cd my-app
$ npm start

This process does not handle backend logic or databases, just the frontend of the build pipeline. Any backend can then be used. Once the process has been completed using the command above, a directory containing the application named my-app is created.

create-react-app uses the Babel and Webpack build tools, so you can use ES6 JavaScript syntax.

ES6 support in React was introduced in version 0.13.0. ES6 code is transpiled into ES5 for use in browsers using the Babel transpiler.

When you use the command npm start, a local server is started and the application will be visible using the URL: http://localhost:3000, although the browser will be opened by default.

When your ready to deploy your React application to production, run the following command. This will build your application for production inside the build folder within your project directory.

			

$ npm run build

The React documentation recommends using React with Babel to let you use ES6 and/or JSX in your code. ES6 is the modern JavaScript syntax and includes new features to make development easier. JSX is an extension to the JavaScript language that works well with React.

It is recommended to install babel-preset-react and babel-preset-es2015 and to create a .babelrc config file to use Babel.

Babel is a compiler and lets you write modern JavaScript code that still works in older browsers.

A modern build pipeline typically consists of: Package manager, bundler and compiler.

Simple Application

We are going to create a very simple application. When the create-react-app package is used to create a new application, the file structure will look like this:

			

.gitignore

	favicon.ico
	index.html
src /
	App.css
	App.js
	App.test.js
	index.css
	index.js
	logo.svg
package.json
README.md

The "root" attribute (root DOM node) within the index.html file is managed by the React DOM and is where the application will be rendered within the HTML (public/index.html). This does not have to be called "root", but we will keep it as such for now.

Once installed, the files that make up the application look like this:

				

<!doctype html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="shortcut icon" href="%PUBLIC_URL%/favicon.ico">
    <!--
      	Notice the use of %PUBLIC_URL% in the tag above.
      	It will be replaced with the URL of the `public` folder during the build.
      	Only files inside the `public` folder can be referenced from the HTML.

      	Unlike "/favicon.ico" or "favicon.ico", "%PUBLIC_URL%/favicon.ico" will
      	work correctly both with client-side routing and a non-root public URL.
      	Learn how to configure a non-root public URL by running `npm run build`.
    -->
    <title>React App</title>
</head>
<body>
    <div id="root"></div>
    <!--
      This HTML file is a template.
      If you open it directly in the browser, you will see an empty page.

      You can add webfonts, meta tags, or analytics to this file.
      The build step will place the bundled scripts into the <body> tag.

      To begin the development, run `npm start`.
      To create a production bundle, use `npm run build`.
    -->
</body>
</html>
				
			
				

body {
	margin: 0;
	padding: 0;
	font-family: sans-serif;
}
				
			
				

import React, { Component } from 'react';
import logo from './logo.svg';
import './App.css';

class App extends Component {
	render() {
    	return (
      		<div className="App">
        		<div className="App-header">
          			<img src={logo} className="App-logo" alt="logo" />
          			<h2>Welcome to React</h2>
        		</div>
        		<p className="App-intro">
          			To get started, edit <code>src/App.js</code> and save to reload.
        		</p>
      		</div>
    	);
  	}
}

export default App;
				
			
				

import React from 'react';
import ReactDOM from 'react-dom';
import App from './App';

it('renders without crashing', () => {
  	const div = document.createElement('div');
  	ReactDOM.render(<App />, div);
});

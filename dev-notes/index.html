HTML

HTML stands for HyperText Markup Language and describes the structure of web pages using HTML elements which act like building blocks. HTML elements are represented by tags.

Introduction

An example of a basic HTML structure. This file needs to be saved with the file extension of .html

The structure of HTML can be explained:

  • <!DOCTYPE html> - Browser instruction about the version of HTML the document is written in.
  • <html> - HTML root element.
  • <head> - Element that contains meta information about the document.
  • <title> - This element specifies the title for the document.
  • <body> - Contains the visible page content (what you want to display on the page).

The HTML document itself begins with <html> and ends with </html>. The visible part of the HTML document is between <body> and </body>.

An example HTML structure can be found below:

				

<!DOCTYPE html>
<html>
<head>
	<title>HTML page</title>
</head>
<body>

<!-- Webpage content goes here -->

</body>
</html>

<!DOCTYPE> declaration

The <!DOCTYPE> declaration must be the very first thing in your HTML document, before the <html> tag. The declaration is not an HTML tag, but an instruction to the browser about what version of HTML the page is written in.

In HTML5 there is only one declaration:

				

<!DOCTYPE html>

Character encoding

The charset attribute specifies the character encoding for the HTML document.

This should be placed as the first meta tag in the HTML document.

				

<!DOCTYPE html>
<html>
<head>
	
	<meta charset="utf-8" />

	<title>HTML page</title>
</head>
<body>

Viewport Meta tag

On desktop browsers, the viewport is always a fixed number of pixels wide and the content is rendered into it. Safari on iOS introduced mobile viewports that are virtual. Even though the screen maybe 320 pixels in width, the browser creates the virtual viewport that's larger and renders the content there. The content is then zoomed out until it fits into the actual physical pixels available on the device's screen.

The viewport meta tag allows the mobile browser to specify what this virtual viewport should be.

				

<!DOCTYPE html>
<html>
<head>

	<meta name="viewport" content="width=device-width, initial-scale=1" />

	<title>HTML page</title>
</head>
<body>

The width property controls the size of the viewport. It can be set to a specific value of pixels (width=600) or to the special value of device-width which is the width of the screen in CSS pixels at a scale of 100%.

The initial-scale property controls the zoom level when the page is first loaded. The maximum-scale, minimum-scale, and user-scalable properties control how users are allowed to zoom the page in or out.

Setting the initial-scale to equal 1, seems to be necessary to make the virtual viewport adapt to landscape mode in iOS.

This meta tag needs to be added to your HTML to make the page mobile responsive. Without this meta tag being added, the page would be rendered, but the user would have to zoom in.

Language attribute

The lang attribute specifies the language of the element's content. In HTML5, the lang attribute can be used on any HTML element.

To specify the language of the document, it can be added to the <html> tag.

Here we specify the language as English:

				

<html lang="en">

Adding CSS to HTML

CSS (Cascading Style Sheets) describes how HTML elements are to be displayed. CSS is often kept in an external/separate file to allow the look of the entire web site to be created using a single file.

To add CSS to HTML, we add a link to it within the <head> section of the HTML code. In the example, we are linking to a CSS file named stylesheet.css within the styles directory.

Find further notes on CSS here. 

				

<head>
	
<link rel="stylesheet" href="styles/stylesheet.css" />

</head>

Adding JavaScript to HTML

JavaScript external files should always be placed before the closing </body> tag within the HTML code. In this example, the myfile.js JavaScript file within the scripts directory is added to the HTML code.

The JavaScript notes can be found here

				

<body>

<script src="scripts/myfile.js"></script>

</body>

HTML Structure

Here is a full example showing the basic structure of the HTML document:

				

<!DOCTYPE html>
<html lang="en">
<head>

	<meta charset="utf-8" />
	<meta name="viewport" content="width=device-width, initial-scale=1" />
	<title>HTML page</title>
	<link rel="stylesheet" href="styles/stylesheet.css" />

</head>
<body>

<!-- Webpage content goes here -->

<script src="scripts/myfile.js"></script>

</body>
</html>

Elements

HTML elements usually consist of a start tag and an end tag. The HTML element is everything from the start tag to the end tag.

				

<!-- Example HTML element -->
<start-tag>Content here</end-tag>

<p>Example paragraph</p>

Elements with no content are called empty elements. Empty elements do not have an end tag. <br> is an empty element without a closing tag.

Empty elements can be "closed" by using a backslash within the tag itself: <br />.

HTML5 does not require empty elements to be closed. If you want stricter validation, or you need to make the document to be readable by XML parsers, you can use this method.

				

<!-- This is acceptable -->
<br>
<!-- This is the strict version and still works -->
<br />

Attributes

Attributes provide additional information about HTML elements. All HTML elements can have attributes. To specify an attribute, it should always be added to the start/opening tag. Attributes are normally in name/value pairs.

				

<start-tag attribute="value">Content here</end-tag>

<!-- lang attribute added -->
<html lang="en">
<!-- This link has the href attribute -->
<a href="https://www.google.co.uk">This is a link</a>

<!-- class attribute added to the element so we can use CSS class -->
<div class="css-class"> ... </div>
<!-- ID attribute added to the element so we can use CSS id -->
<div id="css-id"> ... </div>

<!-- js hook attribute added to the element so we can use JavaScript -->
<div class="js-class"> ... </div>

It is recommended to use JavaScript hooks to identify JavaScript when using HTML attributes. The attribute should always be prefixed with js-. This allows the element to be styled using CSS without the need to change the JavaScript code also if the styling should change.

Multiple attributes can be added to an element.

				

<!-- multiple attributes can be added to HTML elements -->
<div id="css-id" class="css-class css-class-two js-class"> ... </div>

The HTML5 standard does not require quotes around attribute values, but it is recommended.

Comments

Comment tags are used to insert comments into the HTML source code. Comments are not displayed by the browser, but they can help to document your HTML code.

HTML comments start with an exclamation mark ! in the opening tag, but not in the closing tag.

				

<!-- Comment Here -->

Links

Links are defined with the <a> tag. The href attribute specifies the destination web address/URL of the link and the link text will be the visible label given to the link. Clicking on the link text will open the specified address.

Links can be used to open external URL/full web addresses or pages within the current website.

Links should be added between the opening <body> and closing </body> tags.

				

<body>

	<!-- Open Google -->
	<a href="http://www.google.co.uk">Google</a>

	<!-- Open page2.html within the current website -->
	<a href="page2.html">Open Page 2</a>

</body>

Should you want to open the link in a new browser tab/window, the target attribute can be used.

				

<body>

	<!-- Open Google in a new browser window/tab -->
	<a href="http://www.google.co.uk" target="_blank">Google</a>

</body>

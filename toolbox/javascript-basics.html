JavaScript Basics

Statements

A script is a series of instructions that a computer can follow one-by-one. Each individual instruction, or step, is known as a statement. In JavaScript, statements should end with a semicolon. The semicolon also tells the JavaScript interpreter when an instruction, or step, is over indicating that it should move to the next step in the script.

Each statement should start on a new line as this makes your code easier to read and follow.

    

var today = new Date();     // This is a statement

document.write(today);      // This is a statement

Statements surrounded by curly braces are within a code block. The closing curly brace of a code block is not followed by a semicolon.

A code block is a number of statements grouped together.

    

{
    // Statements are grouped here, this is a code block
}

As this example shows:

    

var today = new Date();
var currentHour = today.getHours();
var greeting;

// This is a code block
if (currentHour > 12) {
    greeting = "Good Afternoon";
} else {
    greeting = "Welcome";
}

document.write(greeting);

Comments

You should write comments to explain what your code does. Comments help to make your code easier to read and understand to you and others who maybe reading your code. Comments are not processed by the JavaScript interpreter.

A single-line comment is anything that follows the two forward slash characters // on the same line. This type of comment is often used for short descriptions of what the code is doing.

    

// This is a single-line comment

To write a comment that stretches over more than one line, use a multi-line comment, starting with the /* characters and ending with the */ characters.

    

/* This is a multi-line comment
over many lines. */

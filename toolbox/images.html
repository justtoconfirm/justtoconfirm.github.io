Images

Images can improve the design and the appearance of a web page. In HTML, images are defined with the <img> tag. The tag itself is empty and contains attributes only without a closing tag.

    

<img src="image.jpg" alt="An example alt text">

The Alt attribute

The Alt attribute - also called an "alt tag" - is the written copy that appears in place of an image if it fails to load on a user's screen. This text helps screen-reading tools to describe images to visually impaired users and allows search engines to better crawl and rank the page content as a consideration of SEO (Search Engine Optimisation).

The example, below, shows an example image with an alt attribute but this is considered as being "okay" as it's not very descriptive. More detail can be said about the image.

    

<img src="pancakes.jpg" alt="Pancakes">

This example is much better as it's far more descriptive.

    

<img src="pancakes.jpg" alt="Stack of pancakes with maple syrup">

Further examples may include:

    

<!-- Okay text -->
<img src="escalator.jpg" alt="Man on escalator">

<!-- Good text -->
<img src="escalator.jpg" alt="Man walking on an escalator">

<!-- Recommended text -->
<img src="escalator.jpg" alt="Man wearing a backpack walking down an escalator">

It is recommended to not include "image of" or "picture of" within your alt attribute text as it's assumed your text refers to an image, so there's no need to specify that.

Adding dimensions to images

When considering performance, adding dimensions to your images is recommended. Dimensions allow the page to be laid out with the appropriate space for the image before the image itself has been downloaded. This avoids a layout shift as the image is downloaded and is something Chrome has recently started measuring in the Cumulative Layout Shift (CLS) metric.

Adding width and height attributes to your <img> markup is now recommended. Without dimensions, this may render in two stages. The first stage as the HTML is downloaded and the second stage once the image is downloaded. This would cause the main content to jump down after the image is downloaded and the space needed to display the image can be calculated. This is very disrupting to the user.

An image example with width and height dimensions:

    

<img src="image.jpg" width="600" height="240" alt="An example alt text">

How CSS Interacts with dimensions

Dimensions such as widths and heights on an image can cause issues when you try to alter them using CSS. For example, if you want to limit your images to a certain width you might use the following CSS:

    

img {
    max-width: 100%;
}

This will override the width of the image and constrain it when necessary. If you have explicitly set the height on the image tag, then we are not overriding that (only the width) and you will end up with a stretched or squashed image as we have no longer maintain the aspect ratio of the image.

This can easily be fixed by adding the height property with the value of auto to your CSS so the height attribute from the HTML is overridden too.

    

img {
    max-width: 100%;
    height: auto;
}

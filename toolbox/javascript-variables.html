JavaScript Variables

A script will have to temporarily store bits of information to do its job. This information can be stored in variables.

A variable is a good name for this concept because the data stored in a variable can change (or vary) each time the script runs.

When you write JavaScript, you have to tell the interpreter every individual step you want it to perform. Variables are used to "remember" the values you need the script to use. You can compare variables to short-term memory, because once you leave the page, the browser will forget the information held in variables within your script.

JavaScript is an untyped language meaning a variable can hold a value of any data type. Unlike many other programming languages, you don't need to declare the data type (string, number, boolean) as JavaScript automatically handles this during execution of the code.

Declaring a Variable

Before you can use a variable, you need to announce that you want to use it. This involves creating the variable and giving it a name. This is known as declaring the variable.

To begin, the keyword of var is used to let the JavaScript interpreter know that you are about to declare a variable.

In order to use the variable, you must give it a name. This is sometimes called the identifier. In our example, the variable is called quantity. If the name is more than one word, it is usually written in camelCase. This means the first word is lowercase and any subsequent words have their first letter capitalized.

Rules for naming variables:

  • The name must begin with a letter, dollar sign ($), or an underscore (_) and must not start with a number.
  • You must not use a dash (-) or a period (.) in a variable name.
  • All variables are case sensitive, so score would be different to Score, but it's considered bad practice to create two variables that have the same name.
  • If your variable name is made up of more than one word, use camelCase. For example, firstName rather than firstname.
  • The name of the variable should describe the information/data being stored.
  • Avoid the use of reserved or keywords.

Below, we have declared a variable named quantity. This is an undefined variable as it has no value.

    

var quantity;

Assigning Values

Once you have declared a variable, you can now tell it what information you would like it to store. This is where we assign a value to the variable.

Here, we are setting a value of '3' to the quantity variable we declared. Where possible, the variable's name should describe the data being stored. The equals sign (=) is an assignment operator and is used to assign a value to a variable.

    

var quantity = 3;

Data Types

JavaScript distinguishes between numbers, strings, and true or false values known as Booleans. JavaScript also has other data types include: Arrays, objects, undefined, and null.

For tasks that involve counting or calculations, you will use numbers, usually between 0-9. Large numbers, such as 5272, should have no comma between the values and you can also use decimal or negative values.

    

var price = 3;
var quantity = 5272;

The strings data type consists of letters and other characters and is enclosed within a pair of single or double quotes. Strings are frequently used when working with text and are often used to add content into a page as they can also contain HTML markup.

    

var firstName = "Dan";
var lastName = "Jackson";

Sometimes you will want to use double or single quote markes within a string. You can use a technique called escaping. This is done by using a backward slash (or backslash). This tells the interpreter that the following character is part of the string rather than at the end of it.

    

var message = "<a href=\'sale.html\'>25% off Sale</a>";

A Boolean value can only have a value of true or false and can often be considered as being on or off.

    

var inStock = true;
var shipping = false;

Using Shorthand to Declare Variables

Developers often use shorthand to create variables. While shorthand may save you a little bit of time typing code, it can make the code harder to read. Here are a few examples:

    

// Three variables declared on the same line, then values assigned to each
var price, quantity, total;
price = 5;
quantity = 3;
total = price * quantity;

// Three variables declared and assigned values on the same line
var price = 5, quantity = 3;
var toda = price * quantity;

Changing the value of a Variable

Once you have assigned a value to a variable, you can then change what is stored in the variable later in the same script. Once created, you do not need to use the var keyword to assign a new value, you just use the variable name and the assignment operator.

    

var inStock = true;
var shipping = false;

// Some processing happens here and values need to change

inStock = false;
shipping = true;

Variables and Memory

Each variable you declare takes up memory. The more variables a browser has to store, the greater the memory needed to run your script causing the script to execute more slowly and your web page to take longer to respond. The browser has to remember global variables for as long as the web page using them is loaded. Local variables are only remembered for the period of time that a function is being executed.

A variable references a value stored in memory. This same value can be referenced by more than one variable. Here, the value 15 is stored in memory and the two variables are referencing this same value.

    

var height = 15;
var width = 15;

Global variables are added to the JavaScript Window object. To view the Window object, open the browser console within the Developer Tools and enter the Window command:

    

window

It is often best to avoid using too many variables if you can. Avoid creating new variables if you don't intend to save values. Often, we may come across code examples like this:

    

var fullName = firstName + " " + lastName;
document.getElementById("userName").innerHTML = fullName;

This can be simplified with fewer variables to be written like this:

    

document.getElementById("userName").innerHTML = firstName + " " + lastName;

It's also recommended to keep the number of elements in the DOM small as this will improve page loading and speed up the rendering of the page. Every attempt to search the DOM will benefit from fewer elements and accessing the DOM is very slow when compared with other JavaScript objects.

If you expect to access a DOM element more than once, then use it as a local variable.

    

var obj;

obj = document.getElementById("welcomeMessage");
obj.innerHTML = "Hello World";

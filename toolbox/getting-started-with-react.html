Getting Started with React

What is React?

  • React is an open-source JavaScript library created by Facebook.
  • Used to build front end user interfaces (UI).
  • React is the view layer of an MVC (Model View Controller) application.

React allows components to be created, which are similar to reusable HTML elements, to quickly build user interfaces. Data within React is stored and handled using state and props.

React Developer Tools

There is a browser extension called React Developer Tools that will make working with React much easier. Download React DevTools for Chrome, or for the browser you prefer to use.

Once installed, when you open DevTools, you'll see a tab for React. This will allow you to inspect components as they're written. You can still go to the Elements tab to see the actual DOM (Document Object Model) output.

Install React

Node Package Manager (NPM) can be used to install React. Begin by creating a new directory where the React application will be built and setup a package.json file. Open your terminal and run the following commands:

    

$ mkdir react-application
$ cd react-application

$ npm init -y

We now want to install the following packages:

  • react - the top-level API of the React library.
  • react-dom - DOM specific methods used by React.

React and ReactDOM were split into two packages from version 0.14. ReactDOM is where you will find the render method.

Once again, run the following commands in your terminal to install react and react-dom as dependencies:

    

$ npm install react react-dom --save

At this stage you may need to setup Webpack and Babel to get React working.

JSX: JavaScript & XML

In many React component or code examples, you'll see what looks like HTML. This is not quite HTML but is JSX, which stands for JavaScript XML. With JSX, we can write what looks like HTML but we can also create and use our own XML-like tags.

JSX is actually closer to JavaScript than HTML. There are a few key differences to note when writing it:

  • className is used instead of class for adding CSS classes as class is a reserved keyword in JavaScript.
  • Properties and methods in JSX are camelCase - onclick will become onClick.
  • Self-closing tags must end in a slash - e.g. <img />

Building Components

Almost everything in React consists of components. Most React apps have many small components and everything loads into the main App component. Components often also get their own file.

As we are using Webpack, it's often best to place all React components within the src directory.

Begin by creating a file named index.js inside your project folder and within this file import React and ReactDOM.

    

import React from 'react';
import ReactDOM from 'react-dom';

Alternatively you can choose to import only the method you need from the ReactDOM library by changing the import statement. Here we only want to import the render method.

    

import React from 'react';
import { render } from 'react-dom';

We need to now add a render method to the index.js file. The render method will display a message onto the page.

    

RenderDOM.render(<h1>Hello React</h1>, document.getElementById('app'));

If the import statement is amended to import the render method, then the code example above can be changed.

    

render(<h1>Hello React</h1>, document.getElementById('app'));

The full code for the index.js file is here:

    

import React from 'react';
import { render } from 'react-dom';

render(<h1>Hello React</h1>, document.getElementById('app'));

To display the application in the browser, we need a simple HTML file. Create an empty HTML file in your project directory named index.html and add the following HTML code:

    

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1">
        	
    <title>React App</title>
</head>
<body>

    <!-- Root element -->
    <div id="app"></div>

<script src="main.js"></script>

</body>
</html>

When you run the Webpack build a main.js file will be created. This needs to be included within your HTML file for the application to render. This will be the bundled JavaScript file and should be located within the distribution (dist) directory.

Within your project folder, the file structure should look something like the example below. The .gitignore, node_modules and package.json files have been included.

    

project-folder /
    .gitignore
    node_modules /
    package.json
    package-lock.json
    dist /
        index.html
        main.js
    src /
        index.js

Let's create a component named App that we can then import into the index.js file. Most React applications have many small components and everything can load into the main App component. The App component will be created within it's own file inside the src directory.

This component is a class component but can be written as a simple/functional component also.

Create a new file named App.js and add the code example below.

    

import React, { Component } from 'react';

class App extends Component {
    render() {
        return (
        	<div>
        		<h1>Hello React</h1>
        	</div>
        )
    }
}

export default App;

The component needs to be exported so it can be imported into other components. This newly created App component then needs to be imported into the index.js file and rendered. Update the index.js file so it looks like the example below.

    

import React from 'react';
import { render } from 'react-dom';
import App from './App';

render(<App />, document.getElementById('app'));

When you run the build and view the app, nothing should change and 'Hello React' should render on the page.

The file structure in your project should look like this:

    

project-folder /
    .gitignore
    node_modules /
    package.json
    package-lock.json
    dist /
        index.html
        main.js
    src /
        App.js
        index.js

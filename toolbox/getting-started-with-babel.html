Getting Started with Babel

Install Babel

Use Node Package Manager (NPM) to install the packages required to setup Babel.

  • @babel/core - this is the main dependency, including Babel.
  • @babel/preset-env - the default Babel preset used to transform ES6 into ES5 syntax and to optionally configure browser polyfills.
  • babel-loader - a Webpack loader that hooks Babel into Webpack.

Open your terminal within your project and install the packages as devDependencies.

    

$ npm install @babel/core @babel/preset-env babel-loader --save-dev

Should you need to use Babel within your React project, you should also install the following package:

    

$ npm install @babel/preset-react --save-dev

Babel Config File

To allow Babel to work, a config file is required. This config file is named .babelrc and should be placed within the root of your project folder. Once the .babelrc file is created, it maybe hidden in your project folder. Use CTRL + SHIFT + . (dot), to see this hidden file.

    

$ touch .babelrc

In your code editor, open the .babelrc config file and add the following code:

    

{
    "presets": [ "@babel/preset-env" ]
}

To setup Babel for use with a React application, the @babel/preset-react package is required. You will also need to update the Webpack config file to apply Babel to Webpack.

The .babelrc config file should be updated to include the @babel/preset-react package should you be building a React application. This will allow your React components to be written in ES6 JavaScript syntax.

    

{
    "presets": [ 
        "@babel/preset-env",
        "@babel/preset-react" 
    ]
}
